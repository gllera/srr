# CLAUDE.md

## Project

SRR Frontend ‚Äî single-page RSS reader. Zero runtime deps. Parcel + TypeScript + plain CSS.

## Commands

`npm run dev|build|test|lint|format` ‚Äî dev server, production build (`../dist/srrf/`), vitest+jsdom, ESLint, Prettier.

## Architecture

Entry: `src/index.html` ‚Üí `src/styles.css` + `src/js/app.ts`. Bundler: Parcel 2. `SRR_CDN_URL` is replaced at build time via `parcel/resolve-cdn-url.js`. Resolution: env var ‚Üí `cdn-url:` from `$SRR_CONFIG_INLINE` (raw YAML) ‚Üí `cdn-url:` from `$SRR_CONFIG` (file path, default `$XDG_CONFIG_HOME/srr/srr.yaml`) ‚Üí fallback `http://localhost:3000`.

Dependency chain: `app ‚Üí nav ‚Üí data ‚Üí {idx, cache}`, `app ‚Üí fmt`, `app ‚Üí dropdown ‚Üí {data, nav}`, `app ‚Üí gestures ‚Üí {nav, dropdown}`, `{idx, data, sw, types.d} ‚Üí format.gen` (generated, see below). All in `src/js/`, strict mode.

| Module | Role |
|---|---|
| `format.gen.ts` | **Generated ‚Äî do not edit.** The TS side of the writer‚Üîreader contract: format constants (`IDX_PACK_SIZE`, `IDX_HEADER_SIZE`, `FETCHED_AT_BLOCK`, `DELTA_FETCHED_MAX`, `LATEST_KEEP`, ‚Ä¶) and wire interfaces (`IArticleWire`/`IFeedWire`/`IChannelWire`/`IDBWire`) emitted from the backend Go declarations by `srr gen-ts`. Regenerate with `make generate`; `make verify` fails when stale. |
| `idx.ts` | Binary idx pack parsing. Imports the format constants from `format.gen.ts`; re-exports `IDX_PACK_SIZE`. Exports `IdxPack`/`IdxHeader` interfaces, `makeIdxPack()` (header decoded at construction, entries lazily in `parse()`), `parseIdxHeaders()` (idx/h<N>.gz summary decode), `findPackForBlocks()` (pack-level timestamp binary search). |
| `data.ts` | CDN data layer: fetches `db.gz`, then the idx header summary (`idx/h<hdrs>.gz`) + latest idx pack only; finalized idx packs load lazily on first touch (`fetchIdxPack`, promise-deduped slots) with eager fetch-all as the fallback when `hdrs` lags or the summary 404s. Fetches JSONL data packs on demand. Exports `db`, `init`, `loadArticle(chronIdx)`, async `getChannelId(chronIdx)`/`findChronForTimestamp(ts)`/`findLeft`/`findRight`/`countLeft`, sync `countAll` (latest pack + its cumulative header), `groupChannelsByTag()`. Re-exports `IDX_PACK_SIZE`. |
| `nav.ts` | Navigation state machine: hash routing (`#pos[!tokens]`), traversal, filtering. Returns `IShowFeed`. Uses `pushState`/`replaceState`. Tokens are channel IDs or tag names. Exports `filter`, `fromHash`, `left`, `right`, `first`, `last`, `switchFilter`, `goTo`, `cycleFilter`, `getFilterEntries`, `getCurrentFilterKey`, `isSingleFilter`, `pruneSeen`, `unreadCount`, `peek`. `unreadCount(ch)` = the channel's articles strictly after its `srr-seen` position (`recordSeen` bumps the shown article's own channel+tag keys on every view, filtered or not, so reading via [ALL] clears counts); both sides of the subtraction are idx-derived (`countAll` ‚àí `countLeft(seen+1)`) so db.gz `total_art` drift can't skew it, and it returns -1 ("unknown, not zero") for channels never seen on this device so a fresh localStorage doesn't badge every channel with its full history. `peek(span=10)` = the headlines around `pos` under the current filter (`IPeekItem[]`, chron order: up to `span` matches each side plus the shown article, title falling back to "(untitled)", `when` to fetched_at); it walks `findLeft`/`findRight` and `loadArticle`, so the common case (neighbors in the resident idx pack + LRU'd data pack) costs zero fetches. |
| `cache.ts` | `makeLRU<T>(maxSize)`: LRU cache via Map insertion order. Used by `data.ts` for data-pack caching. |
| `fmt.ts` | Pure utilities (no DOM imports): `sanitizeHtml`, `timeAgo`, `formatDate`, `URL_DENY`, `imgProxy(url, prefix)`, `getImgProxy()`, `setImgProxy(value)`, `isValidProxy(value)` (empty or `https?://`-prefixed; the dropdown editor validates before storing ‚Äî `setImgProxy` stays a dumb setter), `extractImageUrls(html)` (regex-scrapes `<img src>` ‚Äî both quoted and unquoted values, the latter to catch `#minify`'s quote-stripped CDN URLs ‚Äî and returns only http(s) URLs; used by nav.ts prefetch), `collapseBrokenMedia(e)` (delegated error handler: hides ‚Äî `.srr-broken` ‚Äî the `<img>`/`<video>` whose resource failed, targeting the `<video>` host for `<source>` failures; old articles outlive their external images. The error event doesn't bubble, so app.ts registers it on `.srr-content` with `capture: true`). `sanitizeHtml` strips dangerous elements/attributes for defense-in-depth and resolves the proxy prefix once per call; on every `<img>` it forces `loading=lazy` + `decoding=async` (a long article's images must not compete with first render or the neighbor prefetch) and `referrerpolicy=no-referrer` (links already carry `rel=noreferrer`; without it each image fetch tells the feed host where the reader lives ‚Äî the leak the optional proxy exists to prevent; `<video>` takes no referrerpolicy attribute). Relative URLs: `<img src>`/`<video src>`/`<video poster>`/`<a href>` values that are *relative references* (no URL scheme ‚Äî the self-hosted `assets/` keys plus any other relative URL the feed carried, including protocol-relative `//host`) resolve against the pack base (`new URL(SRR_CDN_URL, location.href)` ‚Äî same as data.ts's `DB_URL`, no `data.ts` import to avoid its eager `db.gz` fetch) and bypass the image proxy, dropped (attribute removed) when the resolved URL escapes that base (bounds-checked ‚Äî a `//host` or sub-path `../` traversal off-origin is an info-leak vector); external http(s) image/poster URLs keep the proxy path, external video URLs and anchor hrefs pass through (a link is navigation, not an auto-loaded resource). `imgProxy(url, prefix)` requires the caller to pass the resolved prefix ‚Äî production callers hoist `getImgProxy()` out of per-image loops. Effective prefix comes from localStorage `srr-img-proxy` (set via channel dropdown image-proxy icon); empty/absent = passthrough (no proxy by default). |
| `dropdown.ts` | Toolbar dropdowns (channel menu + headlines peek): own DOM lookups + open/close state. Exports `closeAllDropdowns()`, `showChannelMenu(currentTag, guard)` and `showPeekMenu(guard)`. `toggleDropdown` enforces one open menu at a time (opening either closes the other and resets editor/fill state). Keyboard contract for `role="menu"`: a document-level **capture**-phase keydown handler (capture + stopPropagation because app.ts binds the same arrows to filter cycling on the document bubble path) roves focus with ArrowUp/Down (wrapping), Home/End, and activates with Space (Enter is native anchor behavior); it skips rows hidden in collapsed tag groups, leaves inline-editor `<input>`s alone, and acts only on the module's own still-attached menus (a re-imported test instance must never double-handle). `closeAllDropdowns` hands focus back to the menu's button when focus was inside. The headlines peek (`showPeekMenu`, anchored on the counter button, `l` shortcut) lists `nav.peek()` rows newest-first like a feed ‚Äî title over "channel ¬∑ age" meta, current row `.srr-active` + `aria-current` and scroll-centered, click = `nav.goTo(chron)`; rows fill async after the menu opens (`fillPeek`, freshness-token guarded like `fillUnread`) so a pack-boundary fetch never delays the menu. The currently-shown article's tag and the `guard` mutex are passed in from `app.ts` to keep state ownership clear. The image-proxy icon expands an inline editor row in place of the chip row (input + save/clear; Enter commits after `isValidProxy`, Escape cancels) ‚Äî the row stops click/mousedown propagation so app.ts's window-level "any click closes dropdowns" handler never fires, and the input's Escape stops propagation so the document-level handler doesn't close the menu mid-edit. The calendar icon expands a date editor the same way: a native `<input type=date>` (min = `first_fetched` day, max = today; starts empty so `change` fires once on a complete date) ‚Äî change/Enter commits (a `done` flag dedupes the change+Enter pair), jumping via `findChronForTimestamp` to local midnight of the picked day; commit calls `closeAllDropdowns()` itself since clicks inside the editor never bubble to the window close-handler. Feed-error badges: a channel row whose feeds carry a non-empty `ferr` gets an `.srr-err-dot` prefix (so does the tag header grouping it, since the group may be collapsed) and the row's `title`/`aria-label` carry the error text ‚Äî the evidence already ships in db.gz, this only renders it.. Unread badges: after the menu renders, an async pass (`fillUnread`, freshness-token guarded like the nav prefetch) calls `nav.unreadCount` per channel row and appends `.srr-unread` counts (capped "999+") to rows with unread > 0, plus the sum of known children onto each tag header (inserted before the collapse toggle); unknown (-1) and zero stay unbadged. Badges are float-right in block rows (prepended so a long title ellipsizes before clipping them) and flex-positioned in tag headers. |
| `gestures.ts` | Touch and scroll handlers: one-finger swipe = prev/next, two-finger vertical swipe = cycle filter, scroll-down hides toolbar. Exports `setupGestures({ prev, next, toolbar, guard })`. |
| `app.ts` | UI orchestrator: DOM lookups, render, channel-label refresh, error popup, keyboard handler, `guard()` mutex, init. Async handlers always go through `guard()`. Position persisted to localStorage. |
| `types.d.ts` | Ambient types: `IDB`, `IChannel`, `IFeed`, `IArticle`, `IShowFeed`, `IPeekItem`. Wire shapes are aliased/extended from `format.gen.ts` via `import()` type references (a top-level import would make the file a module and un-global the names); `IDB`/`IChannel` add the client-side normalizations `data.ts init()` applies (`seq` required, `channels` non-null, `.id` stamped). |

CSS: native nesting, `srr-` prefix on all classes, dark mode via `prefers-color-scheme`.

## Data Structures

See root `CLAUDE.md` Data Contract for db.gz, IChannel, IArticle, pack format, CDN layout, and chronIdx.

Frontend-specific additions:
- `channels` in `IDB` is `Record<number, IChannel>` (JSON object keyed by channel ID); defaults to `{}` if absent. `channel.id` is populated from object keys at init.
- **IArticle**: `{ s, a, p, t, l, c }` ‚Äî chan_id, fetched_at, published, title, link, content. Loaded from JSONL data packs.
- **IShowFeed**: `{ article, has_left, has_right, filtered, channel, countRight }` ‚Äî `countRight`: always `number` (never null); count of filtered articles strictly after `pos`.
- Dev: `../packs/` sibling directory served on port 3000 with CORS.

## Key Behaviors

**guard() mutex** (app.ts): all async UI actions go through it. Drops concurrent calls. On error ‚Üí popup with retry.

**Eager fetch**: `data.ts` starts `fetch("db.gz")` at module load (before `init()` call).

**Init**: `data.init()` loads db.gz, the idx header summary `idx/h<hdrs>.gz` (when `db.hdrs === numFinalizedIdx()`), and the latest idx pack ‚Äî O(1) requests at any store size. Finalized idx packs are fetched lazily by chronIdx addressing when navigation touches them; `idxHeaders` (one `IdxHeader` per pack, latest last) powers sync counting (`countAll`), pack-skip deltas (`packHasCandidate`), and the pack-level timestamp search without any fetch. When `hdrs` lags or the summary fetch fails, init falls back to the old eager fetch-all path and peels each pack's own header (`hdrs` stays optional on the wire type ‚Äî an absent value simply never equals `numFinalizedIdx()`). Latest packs are generation-named `L<seq>.gz` (`db.seq`, normalized `??= 0` for the omitempty empty-store case); every pack name is write-once, so all pack fetches use `cache: "force-cache"`. A non-OK response on a latest pack means this tab's db.gz predates the backend GC grace window ‚Äî `assertPackOk` triggers one `location.reload()` (sessionStorage-guarded against loops, guard cleared after a successful `init()`) and always throws so callers never touch the body.

**Article loading**: `loadArticle(chronIdx)` resolves pack+offset via binary search on `IdxPack.bounds`, fetches and parses the JSONL data pack (LRU-cached by pack ID, max 20 packs).

**Neighbor prefetch** (nav.ts): after each `left()`/`right()`, the neighbor lookup (`findLeft`/`findRight` ‚Äî may lazily fetch an idx pack) runs un-awaited off the critical path: its promise lands in `nextLeft`/`nextRight` (consumed by the next keypress; a rejection clears the slot instead of failing the committed navigation) and on resolution `schedulePrefetch(target)` warms the neighbor's images on `requestIdleCallback` (`{timeout:500}`; WebKit has no rIC, so a `setTimeout(‚Ä¶, 200)` fallback keeps iOS/Safari prefetching instead of stalling at every data-pack boundary) ‚Äî it `loadArticle()`s the neighbor, runs `fmt.extractImageUrls()` over its content, and for each URL creates an `Image` with `fetchPriority="low"`/`decoding="async"` and `img.src = imgProxy(...)`. `abortPrefetch()` cancels in-flight loads by setting `img.src=""` (WHATWG image-update steps) and dropping the refs. The `currentPrefetch` array (also the freshness token: a pending idle callback bails when `my !== currentPrefetch`) bounds memory to one neighbor at a time. Edits to nav must preserve this abort/freshness discipline.

**Content fade-in**: double `requestAnimationFrame` for opacity transition.

**Service worker** (`src/sw.ts`, registered best-effort from `app.ts` with `{type: "module"}` so it can import `LATEST_KEEP` from the generated contract ‚Äî module-SW support predates the SW's `DecompressionStream` requirement in every engine, so the module type never limits compatibility): offline-capable cache. Lives at `src/` root (not `src/js/`) so Parcel emits it at the deployment root, giving it a default scope over the whole env (`/srr/`, `/srr.tmp/`, ‚Ä¶). Buckets keyed by mutability: `srr-assets-v1` (content-hash `assets/‚Ä¶` ‚Äî immutable, cache-first), `srr-packs-v3` (write-once pack names ‚Äî finalized `idx|data/<n>.gz`, latest generations `idx|data/L<seq>.gz`, and idx header summaries `idx/h<N>.gz` ‚Äî cache-first with revalidate-on-miss; only `db.gz` is network-first ‚Üí cache fallback; offline consistency is structural since a cached db.gz of seq N can only pair with the write-once `L<N>`), `srr-shell-v1` (navigation network-first + content-hashed JS/CSS cache-first; runtime-cached, no precache manifest ‚Äî keeps the SW hand-written and zero-dep), `srr-meta-v1` (two synthetic entries: the last-seen db.gz `gen` and `seq`). **Manifest tracking** (`checkManifest`): on each successful online `db.gz` fetch the SW gunzips a clone (raw gzip bytes, no `Content-Encoding` ‚Äî same manual `DecompressionStream` as `data.ts`) and reads `gen` + `seq` (absent == 0). A `gen` change (any change, not just increments) purges the whole packs bucket (in-place store rebuild ‚Äî same names, new bytes); a `seq` change (gen unchanged) prunes only cached `L<g>` generations with `g < seq - LATEST_KEEP` (imported from `format.gen.ts` = the backend's `latestKeep` GC grace window, so the SW never drops a generation the store still serves ‚Äî an offline device may rely on it) plus cached idx summaries `h<g>` with `g < hdrs - LATEST_KEEP` (pruned on the seq signal rather than a meta key of their own: hdrs can advance without a seq bump on a zero-article migration/retry cycle, but that strands at most one stale ~1KB name until the next article-producing fetch). Both awaited *before* the db.gz response resolves, so the purge is race-free against the page's subsequent idx/data requests. Best-effort: a parse failure never blocks serving db.gz. The operator bumps `srr gen --bump` after an in-place store rebuild; the `srr-packs-vN` constant now only marks cache-format changes. **Cache bound** (`enforceCacheBounds`): after each successful online `db.gz` fetch (via `event.waitUntil`, off the critical path ‚Äî and never offline, so a device can't lose a pack it can't refetch) each finalized pack series is capped at `PACK_KEEP` (100) entries, lowest-numbered evicted first (the numeric names encode chron age, so no access-time bookkeeping), and the assets bucket at `ASSET_KEEP` (500), oldest-cached first (Cache key insertion order; `cacheFirst` never re-puts on a hit). `db.gz` and the `L<seq>`/`h<N>` names are exempt ‚Äî offline consistency depends on them and checkManifest owns their pruning. Evicting a pack still being read just costs one CDN refetch on the next miss. Needs a secure context (HTTPS or localhost): production CF HTTPS qualifies, an `http://` LAN-IP deploy does not ‚Äî registration just no-ops there and the app runs off the network. Coverage: `e2e/browser/reader.e2e.test.ts` ("serves the reader offline after one online visit", "purges stale finalized packs when db.gz gen changes", "bounds the pack and asset caches, evicting oldest entries first").

**Installable (PWA)** ‚Äî `src/manifest.webmanifest` (linked from `index.html`) + the SW's fetch handler satisfy the browser install criteria. `start_url`/`scope` are `"."` (deployment-relative) so the same bundle installs under `/srr/` or `/srr.tmp/`. Icons: `src/icon.svg` is the single source of truth ‚Äî the same RSS glyph as the inline favicon, recolored white on the brand-orange tile and inset into the maskable safe zone. The raster fallbacks are generated from it with `rsvg-convert -w N -h N src/icon.svg -o src/icon-N.png` (192, 512, and a 180 `apple-touch-icon.png`); the manifest also references `icon.svg` directly as a scalable `any` icon. Regenerate the PNGs whenever `icon.svg` changes. Coverage: `e2e/browser/reader.e2e.test.ts` ("exposes a valid, installable web app manifest").

## State Machines (nav.ts)

State: `pos` (chronIdx), `filter` (object with `active`, `channels`, `chanTotal`, `tokens`, `matches()`, `clear()`, `set(tokens)`).

**`filter`** ‚Äî active when `tokens` non-empty:
- `filter.clear()`: empties tokens, repopulates `channels` from all channels with `total_art > 0`
- `filter.set(tokens)`: resolves tokens (numeric channel IDs or tag names) into `channels` map (`chan_id ‚Üí add_idx`); falls back to `clear()` if no token resolves
- `last(token?)`: no arg = use current filter; `""` = clear filter; otherwise `filter.set([token])`; always jumps to last matching article
- `goTo(idx)`: navigate to chronIdx; if filter active and target doesn't match, snap forward to next match; falls back to `last()` if none
- `switchFilter(token)`: sets filter to token (or clears if `""`); resumes at last seen position for that channel/tag if valid, otherwise jumps to `first()`
- `cycleFilter(dir)`: steps through `getFilterEntries()` by `dir` (+1/-1), calls `switchFilter()`
- `getFilterEntries()`: returns `["", "tagName", ..., "channelId", ...]` built via `data.groupChannelsByTag()`
- `getCurrentFilterKey()`: returns `""`, the single token, or `""` for multi-token filters (URL-only edge case)
- Navigation uses `findLeft`/`findRight` ‚Äî async (may lazily fetch an idx pack); finalized packs without a filter match are skipped via summary header deltas, no fetch. `filter.clear()`/`set()` stay synchronous via `data.countAll`
- Hash: `#pos[!tokens]` ‚Äî `!` segment, `+`-separated tokens, each `encodeURIComponent`-wrapped to survive special chars in tag names.

**Time-range jumps** (dropdown.ts channel menu): "8h"/"16h"/"1d"/"7d" chips compute `Math.floor(Date.now()/1000) - seconds` (unix seconds; the chips carry `t:<seconds>` data-values ‚Äî `t:28800`/`t:57600`/`t:86400`/`t:604800`), look up `data.findChronForTimestamp(ts)`, and call `nav.goTo(chron)` so the user lands at the article from that point and can navigate right. The calendar chip opens the date editor for arbitrary archive dates ‚Äî the same `findChronForTimestamp` ‚Üí `goTo` path with `ts` = local midnight of the picked day (O(1) requests at any depth, courtesy of the resident summary headers).

## Test Patterns

`src/js/nav.test.ts` ‚Äî large nav suite. `src/js/data.test.ts` ‚Äî pure-function cases only. `src/js/idx.test.ts` ‚Äî idx binary-parsing unit tests. `src/js/fmt.test.ts` ‚Äî sanitizeHtml / collapseBrokenMedia / timeAgo / formatDate / isValidProxy tests. `src/js/cache.test.ts` ‚Äî LRU cache tests. `src/js/dropdown.test.ts` ‚Äî image-proxy inline-editor, date-jump, feed-error-badge, unread-badge, menu-keyboard-navigation, and headlines-peek tests (mocks `./data`/`./nav`; seeds the dropdown DOM skeleton, then `vi.resetModules()` + dynamic import because the module captures DOM refs at load ‚Äî each import adds another document-level keydown listener, which is why the handler only acts on its own attached menus and the keyboard/peek describes call `closeAllDropdowns()` in `afterEach`).

**nav.test.ts**:
- **Mock**: `vi.hoisted()` + `vi.mock("./data", ...)` with same shape as data.ts exports. Mocks `getChannelId`, `loadArticle`, `groupChannelsByTag`, `findLeft`, `findRight`, `countLeft`, `findChronForTimestamp`.
- **Reset**: `beforeEach` resets data state, calls `nav.filter.clear()`, and re-spies `history.pushState`/`replaceState`.
- **Helpers**: `makeArticle(overrides)`, `makeChannel(overrides)` ‚Äî factory with defaults. `setupIndex(entries)` ‚Äî populates `db.channels` and wires `getChannelId`/`loadArticle` mocks.
- **Hash checks**: spy on `history.pushState`/`replaceState` (note the spy accumulates across tests in the same describe).

**data.test.ts**: mocks `./data` with inline reimplementations of `findChronForTimestamp` and `groupChannelsByTag` driven by writable `db`/`fetchedAts` state ‚Äî data.ts's module-load `fetch` would otherwise fire under jsdom.

**E2e (`e2e/`)** ‚Äî writer‚Üîreader contract: the unit tests above mock `./data`; the e2e suite runs the REAL `srrb` binary to write packs from canned feeds (`e2e/harness.ts` `srr()` + in-process `feedServer()`, `e2e/fixtures.ts`), then reads them back with the REAL frontend code. Two layers + their own configs (excluded from `npm test` via `vitest.config.ts` `test.exclude`):
- **contract** (`e2e/contract/`, `vitest.contract.config.ts`, jsdom, in `make verify`): `mountReader()` installs a `fetch` shim mapping CDN URLs ‚Üí store files (raw `.gz` bytes, no `Content-Encoding` ‚Äî data.ts decompresses via `DecompressionStream`), `vi.resetModules()` + dynamic-imports the real `data.ts`/`nav.ts` (its module-load `db.gz` fetch must hit the shim, so stub-before-import), then asserts every chronIdx round-trips, pack splits, dedup/seq-generation advance, latest-pack GC grace window, and nav filtering. Cross-checks `srr inspect --validate`. `summary.e2e.test.ts` is the one synthetic-store exception: it hand-builds a 100k-article store (two finalized idx packs + `idx/h2.gz`) so the real reader proves the summary fast path (O(1) boot, lazy pack fetch, header-delta skip) and the eager fallback within the verify time budget ‚Äî real ‚â•50k writer coverage lives in the backend Go tests.
- **browser** (`e2e/browser/`, `vitest.browser.config.ts`, Puppeteer, opt-in via `make test-browser`/`test-e2e`): `serve.ts` global-setup builds the real bundle with relative `SRR_CDN_URL=/packs/` and serves it + a per-run pack dir from one origin; scenarios drive headless Chrome (render, keyboard nav, deep-link, tag filter, headlines peek). Reuses the Chromium under `~/.cache/puppeteer/` (`puppeteer` pinned to 25.0.2). Gotcha: set `Connection: close` + `server.closeAllConnections()` or `server.close()` stalls on Chrome keep-alive sockets.
- Content that must force data-pack splits has to be incompressible (`fixtures.ts` seeded alphanumeric) ‚Äî packs roll on COMPRESSED size (`db_pack.go` `data.Len() >= PackSize<<10`).

## Deployment

GitHub Actions (`release.yml` `pages` job): version tags (`v*.*.*`) or manual. Deploys to GitHub Pages. Reads `SRR_CONFIG_INLINE` (a `ci` environment secret carrying YAML config) and extracts `cdn-url:` from it at build time.
