# CLAUDE.md

## Project

**SRR Backend** ‚Äî Static RSS Reader Backend. Go CLI that fetches RSS/Atom/RDF feeds into gzip-compressed pack series (idx/, data/). Backends: local filesystem, S3, SFTP.

## Commands

`go build -o srr .`, `go test ./...`, `go test -run TestName .`, `go test -v ./store/`. Release: `CGO_ENABLED=0 go build -ldflags "-s -w"`. No Makefile/linter/Dockerfile.

## Architecture

- **`main.go`** ‚Äî CLI via `alecthomas/kong` + YAML config. Source: `$SRR_CONFIG_INLINE` (raw YAML content, takes precedence) ‚Üí `$SRR_CONFIG` (file path) ‚Üí `$XDG_CONFIG_HOME/srr/srr.yaml`. `Globals` struct for flags. Command groups: `chan` (alias `ch`): `add`, `upd`, `rm`, `ls`, `show`, `edit`, `apply`, `import`; `art` (alias `a`): `fetch`, `ls`; `pipe`; `preview` (alias `p`); `config` (alias `c`); `inspect` (alias `i`); `version`.
- **`cmd_inspect.go` / `cmd_inspect_check.go` / `cmd_inspect_report.go`** ‚Äî `InspectCmd` (`srr inspect`): pack consistency checker. Split across three files by responsibility: command entry + fetchers (`cmd_inspect.go`: `openFetcher`/`httpFetcher`/`loadCore`/`loadDataPack`), validation checks invoked by `--validate` (`cmd_inspect_check.go`), single-mode reports for `--chron`/`--filter`/`--list-tags`/`--from-hash` (`cmd_inspect_report.go`). The idx parse + addressing mirror itself lives in `idx_read.go` (shared with `srr art ls`). Modes: `--chron N` (single-entry inspection w/ recovered fetched_at vs stored), `--validate` (full sweep: bounds-vs-data, db-meta cross-check, chanCounts/fetchedAts continuity across pack boundaries, unknown chan_ids, latest-pack file presence, idx header-summary coverage/content vs `hdrs`, search coverage/consistency via `checkSearch` ‚Äî overclaim, latest tail count vs `srcht`, shard sizes, every title's grams probe positive in its shard bloom, summary == concat of shard headers), `--filter <tag|chan_id>` (filter resolution + count, mirrors frontend filter math), `--from-hash '<hash>'` (replays `nav.fromHash` end-to-end), `--list-tags` (mirrors `groupChannelsByTag`). `--url` for HTTP CDN; otherwise uses `--store`. Diagnoses the `(reading 's')` crash class in frontend `nav.ts` (`showFeed` accesses `data.db.channels[article.s]`).
- **`cmd_fetch.go`** ‚Äî `signal.NotifyContext` for graceful shutdown. `errgroup` (`golang.org/x/sync/errgroup`) with `SetLimit(globals.Workers)` and a `sync.Pool` for feed buffers. Articles sorted by published time (ascending) before storage. Order: `PutArticles` ‚Üí `SyncIdxSummary` ‚Üí `SyncSearch` ‚Üí `Commit` ‚Üí GC sweeps (all sync/GC steps warn-only; each GC sweep runs only when its counter ‚Äî `Seq`/`HdrPacks`/`SearchPacks` ‚Äî advanced this run, since most cycles fetch nothing and would just re-delete an already-empty window; a crash-leaked name is still swept by the next advancing run). `--interval` / `SRR_FETCH_INTERVAL` duration flag runs fetch in a loop. Error summary (fetched/failed counts) logged at end.
- **`cmd_chans.go`** ‚Äî `AddCmd` (`srr chan add`, strict-create; flags `-t/--title`, `-u/--url` (repeatable), `-g/--tag`, `-p/--parsers`, `-i/--ingest`), `UpdCmd` (`srr chan upd ID`, channel-level fields plus `-u/--url` (REPLACE), `--add-url` (idempotent APPEND), `--rm-url` (strict REMOVE); the three URL flags are mutually exclusive), `RmCmd` (`srr chan rm`), `LsCmd` (`srr chan ls`, filter by `-g/--tag`, yaml/json output), `ShowCmd` (`srr chan show ID`), `ApplyCmd` (`srr chan apply` reads channel JSON from `--file PATH` or stdin; accepts a single object or an array; whole-input atomic; create when `id` absent, full-replace update when `id` present, error if `id` references a missing channel), `EditCmd` (`srr chan edit ID` opens `$VISUAL`/`$EDITOR`/`vi` on the channel JSON; no-change is a no-op; id changes refused; tempfile preserved on parse or apply error). Per-URL state (`ETag`, `Watermark`, `BoundaryGUIDs`) is preserved across `chan upd -u ‚Ä¶`, `chan upd --add-url ‚Ä¶`, `chan apply` updates, and `chan edit` by URL match. Empty-string convention on `-t`/`-g`/`-p`/`-i` clears the field where allowed (title cannot be cleared).
- **`cmd_art.go`** ‚Äî `ArtCmd` (`srr art ls`): lists stored articles newest-first with cursor pagination (`-b/--before`), filter by channel ID (`-i`) or tag (`-g`), with `-l/--limit` (default 50). Outputs JSON. `readAllIdx` walks the packs parsed by the shared `idx_read.go` reader (`loadIdxPacks` + `getPackRef`), filtering by `db.Channels()`/`AddIdx`; `loadContent` parses JSONL data packs to fill in the per-article `ArticleData` (title, link, published, content) from the data pack.
- **`cmd_import.go`** ‚Äî OPML import with hierarchical ID selection (`-a` all, `-i` specific). OPML group hierarchy auto-resolves to hierarchical tags; `-g/--tag` overrides. `-t/--title` triggers merge mode (every selection collapses into one channel with N feeds; `-t ""` or `-t` without selection errors). `-p/--parsers` and `--ingest` stamp every imported channel (merged or per-leaf). `-n/--dry-run` lists resulting channels without importing. `applyImportDefaults` centralises post-walk Tag/Pipe/Ingest stamping.
- **`opml.go`** ‚Äî OPML XML parsing. `ParseOPMLTree` builds `OPMLNode` tree from file. `normalizeGroupName` converts group names to tag-safe identifiers.
- **`cmd_preview.go`** ‚Äî `preview` subcommand: dispatches a URL through the ingest engine (`-i/--ingest` picks the strategy, otherwise falls back to the db.gz root `Ingest` then `#rss` via `ingest.Select`), runs module pipeline (`-p`), serves rendered articles via local HTTP server (`-a/--addr`). Opens the DB read-only to read the root default.
- **`cmd_ingest.go`** ‚Äî `IngestCmd` (`srr ingest [strategy]`): top-level command for the db.gz root `ingest` field. Mirrors `srr pipe`: no args prints, `""` alone clears, otherwise replaces.
- **`channel.go`** ‚Äî `Channel` (Title, Tag, Pipe, `Ingest`, `Feeds []*Feed`) and `Feed` (URL, ETag `etag`, LastModified `last_modified`, Watermark `wm`, BoundaryGUIDs `bg`, FetchError `ferr`). `Channel.Fetch` resolves the effective pipeline once via `resolvePipe(rootPipe, ch.Pipe)` and the effective ingest strategy once via `ingest.Select(ch.Ingest, rootIngest)`, then iterates feeds sequentially sharing the buffer pool slot; per-feed errors record into `Feed.FetchError` while items from successful feeds still commit. Also hosts the `resolvePipe` helper and the `#base` token constant for root-pipe inheritance. Pure type + orchestration; per-feed HTTP/dedup logic lives in `feed.go`.
- **`cmd_pipe.go`** ‚Äî `PipeCmd` (`srr pipe [pipe...]`): top-level command for the db.gz root `pipe` field. No args prints; `""` alone clears; otherwise replaces. Also hosts `filterPipe` (drops empty strings; returns nil for all-empty input as the CLI "clear" sentinel).
- **`feed.go`** ‚Äî `Feed.fetch` receives an already-resolved `ingestName` from `Channel.Fetch` (resolution lives at the channel level so all feeds in a channel share one strategy and the choice is made once per channel), dispatches I/O+parse through the shared `*ingest.Fetcher` engine, then applies the dedup/watermark/pipeline path on the returned `ingest.Result.Items`. Dedup model per feed: `Watermark` is the max published unix-second ever seen and `BoundaryGUIDs` is the union of (GUIDs at `Watermark` in the most recent fetch) and (dateless GUIDs in the most recent fetch). Repopulated each non-empty fetch from the current response (no carry-over) so its size stays bounded by what the publisher currently exposes; a `NotModified` result or a non-modified response with zero items preserves prior `Watermark`/`BoundaryGUIDs` so the dedup state survives a transient empty channel. An item is new iff its GUID isn't in the prior fetch's `BoundaryGUIDs` AND (`pub == 0` OR `pub >= Watermark`). Item `pub` is clamped to `fetchedAt` so a publisher CMS bug that ships a far-future date can't poison `Watermark`. Within-fetch dedup uses a per-GUID set checked first so duplicate items in one feed response are collapsed and can't pollute `Watermark`/`BoundaryGUIDs`. Trade-off: items at `Watermark` or dateless items that disappear from the feed for one fetch and reappear later are re-ingested as duplicates (snapshot semantics over carry-over).
- **`cmd_gents.go`** ‚Äî `GenTSCmd` (`srr gen-ts`, hidden): generates `frontend/src/js/format.gen.ts` ‚Äî the TS side of the writer‚Üîreader data contract ‚Äî by reflecting over the live Go declarations (the `tsConsts` table references the db.go format constants; `tsTypes` walks the json tags of `ArticleData`/`Feed`/`Channel`/`DBCore`, omitempty ‚Üí `?`, nil-able without omitempty ‚Üí `| null`). `//go:generate go run . gen-ts` (run via `make generate`); `--check` compares instead of writing and is wired into `make verify` (`verify-be`), so editing a format atom without regenerating fails loudly. A struct reachable from `tsTypes` but not listed in it is a hard generator error.
- **`cmd_config.go`** ‚Äî `ConfigCmd` (`srr config`): prints resolved configuration. With no argument, prints all globals and active backend config; with a key argument, prints a single field. Reads from `store.Configs()` for backend-specific sections.
- **`processing.go`** ‚Äî Per-item transformation utilities shared by `Feed.fetch` and `PreviewCmd`: `processItem` runs the module pipeline, enforces GUID/Published immutability, then normalises Title (via `bluemonday` strict policy + whitespace collapse) and strips control chars from Link/Content. Also hosts `validFeedURL` (used by `cmd_chans.go` and `opml.go`).

### Ingest (`ingest/`)

An "ingest strategy" is the I/O+parse layer that produces `mod.RawItem`s from a URL. The default zero-config behaviour is the historical RSS/Atom/RDF path (`#rss`); alternative strategies ‚Äî shipped out-of-repo as external shell commands ‚Äî cover URLs that aren't feeds. Selection precedence per channel: `Channel.Ingest` (most specific) > `DBCore.Ingest` (db.gz root, managed by `srr ingest`) > built-in `#rss`. Empty strings fall through; persistence uses `omitempty`.

Built-in names start with `#` and resolve through the package's `Register` registry (init-time). Anything not starting with `#` is treated as a shell command per the external-ingest protocol. Only `#rss` ships built-in; source-specific strategies live out-of-repo as external commands.

Mirrors `mod/`'s architecture: a `Fetcher` struct (built by `New()`) holds the registered built-in `FetchFunc`s in a name‚Üífunc map; `Fetcher.Fetch(ctx, args, ...)` dispatches by name ‚Äî built-ins for matching keys, shell-command fallthrough for anything else.

- **`main.go`** ‚Äî `Request` / `Result` structs, `FetchFunc` type (`func(ctx, *http.Client, buf, Request) (Result, error)`), factory registration `Register(name, func() FetchFunc)` (the outer factory runs once per `New()` so built-ins can capture per-instance state ‚Äî same pattern as `mod.Register`), `Select(channel, global) string` precedence helper (final `#rss` fallback), the `Fetcher` struct + `New() *Fetcher`, and `Fetcher.Fetch(ctx, args, client, buf, req)` which routes `args` to a built-in if registered or executes it as a shell command otherwise. External wire schema is the JSON encoding of `Request` (stdin) and `Result` (stdout) ‚Äî see struct tags; stderr is passthrough. A non-zero exit code or empty stdout is a hard error (fails just that feed's fetch); stdout is capped at 64 MiB. Items on the wire are `mod.RawItem` records, so external strategies emit `guid` as the FNV-32a hash (uint32) of any stable per-item string, matching the dedup contract used by built-ins. `published` is RFC3339 (null/absent for dateless). The author-facing protocol spec + a reference implementation live in `backend/README.md` (Ingest Strategies); the protocol tests are in `ingest/external_test.go`.

**External file self-hosting.** The ingest engine owns no asset machinery: it just sets the external command's working directory to `Request.AssetDir` (`cmd.Dir`) so the command stashes/references files with relative paths. The caller (`main`) owns the rest. `FetchCmd.fetch` creates **one cache dir shared by every feed this run** = `assetCacheRoot()` (`MkdirAll`, kept across runs), stores it on `fetchRun.cacheDir` (creation is **mandatory** ‚Äî never empty, since a command handed an empty working dir would run in SRR's cwd), and `Feed.fetch` hands it to **every** feed as both `Request.AssetDir` and the upload-step `cacheDir` ‚Äî no strategy gate, so a built-in fetcher may download into it too. Whichever fetcher ran owns the file layout inside the shared dir. The upload itself is the **automatic end-of-pipeline step** inlined in `Feed.fetch` (run after `processItem`, which stays a pure, store-free transform): for each item it calls `mod.RewriteAttrs(content, fn)` (the generic walk over `assetAttrs` ‚Äî img/video `src`/`poster` + `a href`), and `fn` scans for **`#`-markers** ‚Äî a value starting with `#` whose remainder names an existing regular file in `cacheDir` (e.g. `#/photo.jpg`) ‚Äî uploading each through `assetFetcher.UploadCacheRef` (content-hash key, upload-if-absent) and rewriting the marker to the returned key. `UploadCacheRef` classifies its guard failures with sentinel errors: `errNotAsset` (names no regular file ‚Äî an ordinary `#fragment` anchor) and `errEscapesCache` (resolves outside the cache dir ‚Äî attacker-influenced content) are **declined** by `fn` (the marker stays untouched, one containment implementation lives in `UploadCacheRef` alone); any other upload failure (oversize, store error) returns the error from `fn`, which `RewriteAttrs` surfaces so `Feed.fetch` **hard-fails the whole feed** ‚Äî feed state (watermark/dedup/etag) is left untouched, so a transient failure self-heals next fetch while a permanently-rejected asset wedges the feed until fixed. `<video poster>` is **not** a usable target: `#sanitize`'s poster regex (`^(https?://|assets/)`) strips a `#`-poster before the upload step runs. Download dedup is the command's (cache check); upload dedup is SRR's (content hash + store existence check). The cache is disposable, no eviction (ops-managed). A `FetchFunc` owns only I/O and parsing ‚Äî dedup, watermarking, pipeline modules, and storage all stay in `Feed.fetch`/`processItem` and operate uniformly on `Result.Items`. Built-in `FetchFunc`s and the shell-command path must be concurrent-safe (one `*Fetcher` is shared across workers; unlike `*mod.Module` it isn't pooled).
- **`rss.go`** ‚Äî Built-in `#rss` strategy (registered inline in `init`): the zero-config default. HTTP GET with `If-None-Match` / `If-Modified-Since` hints into the shared per-worker buffer, then `ParseFeed`. Returns `NotModified` on 304; populates `ETag` / `LastModified` for the next request. Also houses the streaming RSS/Atom/RDF parser: `ParseFeed(data, fn)`, `ErrStopFeed` sentinel, and the package-private `hash(string)` used for GUID stability (the FNV-32a contract external fetchers replicate).

### Store (`store/`)

Low-level storage interface: `Get`/`Put`/`AtomicPut`/`Rm`/`Close`. Registry selects by URL scheme; local = empty scheme `""`. Config registry: backends call `RegisterConfig` in `init()` with a config struct pointer; `LoadConfigs` reads YAML sections into them.

| Method | Signature | Behavior |
|---|---|---|
| `Get(ignoreMissing)` | returns `io.ReadCloser` | Controls error-on-missing; streaming read |
| `Put(ignoreExisting)` | accepts `io.Reader` | Controls overwrite vs exclusive create; streaming write |
| `AtomicPut` | accepts `io.Reader` | temp-then-rename (local/SFTP); overwrite (S3); streaming write |
| `Rm` | ‚Äî | Silent on missing (local/SFTP warn + return nil; S3 unconditional delete) |

- **`local.go`** ‚Äî Auto-creates subdirs via `os.MkdirAll`.
- **`s3.go`** ‚Äî `IfNoneMatch` precondition headers + CRC32 checksums. `S3Config`: region, endpoint, profile, static credentials.
- **`sftp.go`** ‚Äî Auto-creates subdirs via `client.MkdirAll`. Auth chain: URL password ‚Üí config password ‚Üí config/default private key ‚Üí `~/.ssh/` keys ‚Üí SSH agent ‚Üí error. Host key verification via `~/.ssh/known_hosts` by default (`SFTPConfig.Insecure` to skip).

### Pack Storage (`db.go` + `db_pack.go` + `db_search.go`)

See root `CLAUDE.md` Data Contract for pack format spec (idx/, data/, search/ series), db.gz schema, CDN layout, and file-based locking.

Split across four files by concern:
- **`db.go`** ‚Äî `DB`, `DBCore`, lifecycle (`NewDB`/`Close`/`Commit`), channel CRUD (`AddChannel`/`RemoveChannel`/`ChannelByID`/`Channels`), `withDB`/`withDBCtx` boilerplate wrappers, and shared utilities (`gunzip`, `readGz`, `jsonEncode`). `Commit` serializes `DBCore` via `AtomicPut`. `BumpGen` owns the bump-implies-reset invariant (Gen++ zeroes `HdrPacks`/`SearchPacks`/`SearchTail`; `srr gen --bump` is a thin caller). Also holds the format atom constants including `searchGram`/`searchBloomBytes`/`searchBloomK` (exported to TS as `SEARCH_GRAM`/`SEARCH_BLOOM_BYTES`/`SEARCH_BLOOM_K` by `cmd_gents.go`).
- **`db_pack.go`** ‚Äî Binary idx + JSONL data pack writer. Contains `ArticleData` and `Item` types, the `pack` struct (`newPack`/`writeIdx`/`writeIdxHeader`/`writeArticle`), `loadPack`/`savePack`/`expectedLatestIdxSize`, and the `PutArticles` driver. Also `dataKeyFor` and `parseDataPack` (used by `cmd_art.go` and `cmd_inspect.go` for read-side access), and the idx header-summary machinery: `summaryKey`/`numFinalizedIdx`/`idxKeyAndSize`/`readPackHeader` (generalized header reader, takes a size param), `saveSummary` (publishes a count-named concatenation of finalized-pack headers; shared by `SyncIdxSummary` and `SyncSearch`), `SyncIdxSummary` (rebuilds `idx/h<N>.gz` from the finalized packs' headers whenever `HdrPacks` lags ‚Äî covers normal finalization, pre-summary store migration, and post-`gen --bump` resets; sets `HdrPacks` only after the save succeeds), and `GCSummaries` (sweeps superseded `h<g>` names, same grace window as `GCLatest`).
- **`db_search.go`** ‚Äî Search pack writer and folding contract. `SearchEntry` type (`s`/`w`/`t` JSONL wire) with its read-side decoder `parseSearchEntries` (used by `srr inspect`'s `checkSearch` and the tests ‚Äî the wire format has one owner). `foldSearchText` (NFD ‚Üí strip Mn ‚Üí lowercase ‚Üí œÇ‚ÜíœÉ ‚Üí non-letter/number = word break, space-joined; mirrored by `search.ts fold()`). `bloomBits`/`bloomAdd`/`bloomHas` (FNV-1a-64 double-hash, 4 probes). `walkArticles` (streams `[from, to)` chron range from idx+data packs). `SyncSearch(ctx, batch)` (called by `cmd_fetch` between `SyncIdxSummary` and `Commit` with the same sorted `[]*Item` it gave `PutArticles`, warn-only): reconciles `search/` with the store ‚Äî reads back `search/L<Seq-1>` (trusted iff entry count == `SearchTail`), fills the missing chron range from `batch` when it covers the gap exactly (the common cycle ‚Äî no pack re-read seconds after the write) else via `walkArticles`, finalizes bloom-headed shards at `idxPackSize` boundaries (`saveSearchShard` decodes titles for the bloom there, once per 50k articles), writes `search/L<Seq>`, rebuilds the bloom summary from streaming `readPackHeader` reads when `SearchPacks` lags `numFinalized`; sets `SearchPacks`/`SearchTail` only after every save succeeds. `finalizedSearchKey`/`searchSummaryKey`. `GCSearchSummaries` (sweeps superseded `s<g>` names, same grace window as `GCSummaries`). `golang.org/x/text` (unicode/norm) is a direct dependency.
- **`idx_read.go`** ‚Äî the single Go read-side mirror of the binary idx format: `keyGetter` (fetch+gunzip a store key, local or HTTP), `idxPack`/`idxBound`, `parseIdxPack` (byte-for-byte mirror of frontend `idx.ts` `makeIdxPack().parse()`), `getPackRef` (mirrors `data.ts`), `packIdxFor`, `loadIdxPacks`. Every read-side command (`srr inspect`, `srr art ls`) goes through this one parser; the only other implementations of the format are the writer (`db_pack.go`) and the frontend reader.

Backend-specific behavior:
- `PutArticles` and `savePack` manage the idx and data series. `SyncSearch` manages the search/ series. Order: `PutArticles` ‚Üí `SyncIdxSummary` ‚Üí `SyncSearch` ‚Üí `Commit`.
- `PutArticles` writes binary idx and JSONL data directly; splits idx packs at every 50,000 articles; sets `FirstFetchedAt` on first run that produces articles.
- Per-entry `delta_fetched_at` is computed against `prevFetchedTS = FirstFetchedAt/28800 + FetchedAtCursor` so the first entry of each batch records the elapsed time since the previous fetch (clamped to 7 bits with carry).
- `ArticleData` struct: `{ ChannelID, FetchedAt, Published, Title, Link, Content }` ‚Äî serialized as JSONL with short keys `s/a/p/t/l/c`.
- `DBCore.Channels` is `map[int]*Channel`; serialized as a JSON object keyed by channel ID. `AddChannel` assigns the first free ID in [0, 255] and returns an error if all 256 slots are taken. `RemoveChannel` uses `delete`.
- `seq` names the latest packs (`idx/L<seq>.gz`, `data/L<seq>.gz`, `search/L<seq>.gz`): each article-producing batch writes generation `seq+1` (idx + data series; SyncSearch adds search), bumps `Seq` only after idx+data saves succeed, and `Commit` publishes it ‚Äî names are write-once, so a generation can never be seen half-written or rewritten. `GCLatest(ctx, keep)` (called from `cmd_fetch` after `Commit`, `latestKeep` = 2, warn-only) deletes expired generations for all three series, sweeping a 4-wide trailing window so crash-leaked packs self-heal.
- `hdrs` names the idx header summary (`idx/h<hdrs>.gz`): `cmd_fetch` calls `SyncIdxSummary` between `PutArticles` and `Commit` (warn-only ‚Äî a failed ~1KB summary write must not discard a durable batch; readers fall back to eager idx loading and the next run retries), then `GCSummaries` next to `GCLatest`. `srr gen --bump` resets `HdrPacks` to 0 (an in-place rebuild invalidates the copied headers).
- `srch`/`srcht` name the search shards and latest tail: `cmd_fetch` calls `SyncSearch` between `SyncIdxSummary` and `Commit` (warn-only); `GCSearchSummaries` runs next to `GCSummaries`; `GCLatest` also sweeps `search/L<g>` generations. `srr gen --bump` resets both `SearchPacks` and `SearchTail` to 0.
- `withDB(locked, fn)` / `withDBCtx(ctx, locked, fn)`: most cmd `Run()` methods are now a single `withDB` call wrapping the work.

### Module System (`mod/`)

Pipeline per-channel during fetch. Factory pattern: `New()` returns a fresh stateful processor; each built-in registers a `func() Processor` factory that runs once per `New()` so it can capture per-instance state (compiled policies, a shared HTTP client). A built-in processor is `mod.Processor` = `func(context.Context, Params, *RawItem) error` (the ctx is the per-fetch context, threaded through `Module.Process`); most built-ins ignore the ctx, but network-bound ones like `#readability` use it for cancellation/deadline.

**File layout** ‚Äî `main.go` holds the engine (registry + `Module`/`Process` dispatch), the `RawItem` wire types, and the external/shell-module exec. Support helpers carry a `helper_` filename prefix (`helper_params.go`, `helper_assets.go`, `helper_ssrf.go` ‚Äî the latter exports `SafeTransport`, the SSRF-guarded `*http.Transport` used by `#readability`) to set them apart from the built-in module files, which are named for the module itself (`sanitize.go`, `minify.go`, `readability.go`, each with a matching `_test.go`).

**Module parameters** ‚Äî a pipeline token may carry `key=value` options after the name, e.g. `#readability timeout=30s maxbody=16MiB`. A bare key with no `=` (e.g. `verbose`) is shorthand for `verbose=true`, so boolean flags need no value. `Module.Process` splits the token on whitespace; when the first field is a registered built-in it parses the rest into `mod.Params` (see `helper_params.go`) and dispatches internally, otherwise the original string runs as a shell command (so shell commands with spaces/`=` are untouched). Built-ins read options via typed accessors with a default for the absent case ‚Äî `Params.Duration(key, def)`, `Params.Bytes(key, def)` (sizes accept `B`/`KB`/`MB`/`GB` decimal or `KiB`/`MiB`/`GiB` binary; bare = bytes), `Params.Bool(key, def)` (a bare flag reads as true; explicit values take the `strconv.ParseBool` set) ‚Äî and call `Params.only(allowed...)` to reject unknown keys. A malformed value or unknown key is a hard error (loud config feedback), distinct from a module's per-article fail-open behaviour; modules that take no options reject any param. Parameters are parsed per-item but the cost is negligible. External (shell) modules don't get parsed params ‚Äî they receive their args on the command line.

- `#sanitize` ‚Äî bluemonday, content-only.
- `#minify` ‚Äî tdewolff/minify, content-only.
- `#readability` ‚Äî for feeds that syndicate only a teaser/summary: HTTP-GETs the item's `Link` (browser-like UA, default 20s timeout, 8 MiB body cap) and replaces `Content` with the article's main body extracted via `go-shiori/go-readability`. Tunable per pipeline position via `timeout=` (Go duration) and `maxbody=` (byte size), e.g. `#readability timeout=30s maxbody=16MiB`; the timeout is enforced via a per-call `context.WithTimeout` so the shared client can carry different budgets. **Fail-open**: empty/invalid link, non-2xx, fetch/parse error, or empty extraction all leave `Content` untouched and return nil (logged at WARN) ‚Äî one bad article never fails the fetch (but a bad `timeout=`/`maxbody=` value, or an unknown param, is a hard error). Place it **before** `#sanitize`/`#minify` (e.g. channel pipe `["#readability", "#base"]`) so the fetched HTML gets clamped to the allowed element set. Network-bound, so unlike the other built-ins it honours the fetch `context` (cancellation/deadline).
- External modules: `/bin/sh -c`, stdin/stdout JSON (`RawItem`), stderr passthrough. The author-facing protocol spec + a reference implementation live in `backend/README.md` (External mod protocol); the protocol tests are in `mod/main_test.go`.

#### Asset self-hosting (`assetFetcher.UploadCacheRef` + `RewriteAttrs`)

SRR self-hosts files an ingest strategy leaves in the run's shared cache dir: it stores them under a content-hash key (`assets/<2-hex>/<16-hex><ext>`) and rewrites the in-content reference to that key. This is **not** a pipeline module ‚Äî it is the automatic end-of-pipeline step inlined in `Feed.fetch`.

- Implementation `assetFetcher` (`assets.go`): `UploadCacheRef(ctx, cacheDir, localname)` resolves `localname` inside `cacheDir` (guards: containment via `EvalSymlinks`+`Rel`, reject non-regular/symlink, `SRR_MAX_MEDIA_SIZE` cap), keys by `contentHashKey` (sha256 of the **file bytes**), and `Put`s only if the key isn't already in the store (`Backend.Get` existence check). It does no outbound HTTP ‚Äî the file is already on disk (an ingest strategy downloaded it). Built once per `fetch` run in `cmd_fetch.go` from `db.Backend`, shared across workers (concurrent-safe). The external-ingest upload step (inlined in `Feed.fetch`, see Ingest ¬ß) calls it once per `#`-marker against the run's shared cache dir (`fetchRun.cacheDir` = `assetCacheRoot()`).
- `RewriteAttrs(content, fn)` (`mod/helper_assets.go`): the generic HTML attribute walk over `assetAttrs` (img/video src/poster **plus `a href`**, the generic-file set) behind the upload step. The caller's `fn` returns `(newVal, ok, err)` and owns the `#`-marker upload policy ‚Äî which values are markers, how files are stored, and which failures are fatal (a non-nil `err` aborts the walk and fails the feed) ‚Äî keeping that policy out of `mod`.
- Idempotency: download dedup is the ingest strategy's (its cache check); upload dedup is SRR's ‚Äî `UploadCacheRef` skips the `Put` when the content-hash key is already in the store (`Backend.Get` existence check), so a re-run re-uploads nothing.
- `assets/` is a reserved store prefix in the writer‚Üîreader contract (analogous to `idx/`/`data/`); the frontend (`fmt.ts`) resolves it against the pack base. `S3.Put` stamps `Content-Type` by extension so assets render in-browser; local/SFTP rely on the static server's mime-by-extension.
- `GUID` and `Published` are immutable for all modules (built-in or external; change = error). Enforced in `processItem` after each pipeline step ‚Äî the captured value before the step is compared to the post-step value, attributing changes to the offending module.
- `RawItem.Raw` is set by `feed.go` to the parsed entry as `mod.RawFeedItem` (`map[string][]mod.RawField`); modules can type-assert it for typed access. External (shell) modules see the same data as JSON via the short keys `@`/`$`/`+` (text/attrs/children).

### Pipe Hierarchy

Two levels store an optional mod pipeline: root (`DBCore.Pipe`) and channel (`Channel.Pipe`, JSON `pipe`). Resolved once per channel at the start of `Channel.Fetch` via `resolvePipe` in `channel.go`:

- An empty channel `Pipe` (nil/absent or empty slice) inherits the root pipe; only a non-empty `Pipe` overrides.
- The `#base` token in a channel override expands inline to the root pipe; can appear multiple times. Non-token entries pass through verbatim.
- CLI: `srr pipe` (root), `srr chan add -p` / `srr chan upd -p` / `srr chan import -p` (channel). `-p ""` clears (reverts to inherit-root). The `-p` flag is **repeatable, one step per flag** and carries `sep:"none"` so kong never comma- or space-splits it ‚Äî a step keeps its spaces (module parameters) and any commas (e.g. `-p "#readability timeout=30s maxbody=16MiB" -p "#base"`). `srr pipe` (root) is a variadic positional, one arg per step (also `sep:"none"`).
- `NewDB` substitutes `defaultRootPipe()` (`["#sanitize", "#minify"]`) when the loaded `DBCore.Pipe` is nil ‚Äî fresh DBs and existing DBs predating this feature both pick up the default. Persisted on the next `Commit`.

## Conventions

- **Imports**: stdlib ‚Üí external ‚Üí internal, blank-line separated, alphabetical within groups
- **Errors**: Always wrap with `fmt.Errorf("context: %w", err)`. Single sentinel: `ErrStopFeed` (feed.go). No custom error types.

## Constraints (preserve when editing)

- File-based DB lock (`.locked`) with `--force` override
- Env vars prefixed `SRR_`
- `ErrStopFeed` sentinel is part of the `parseFeed` callback contract for early-exit (currently unused in production code; kept for the API)
