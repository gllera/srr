# CLAUDE.md

## Project

**SRR Backend** ŌĆö Static RSS Reader Backend. Go CLI that fetches RSS/Atom/RDF feeds into gzip-compressed pack series (idx/, data/). Backends: local filesystem, S3, SFTP, HTTP.

## Commands

The root `Makefile` is the unified entry point for both halves of the monorepo; prefer it over raw `go` commands:

- `make build-be` ŌĆö build the binary to `dist/srr`
- `make test-be` ŌĆö `go test ./...`
- `make vet-be` ŌĆö `go vet ./...`
- `make format-be` / `make format-check-be` ŌĆö `gofmt -w .` / gofmt gate (the latter runs inside `make verify-be`)
- `make lint-be` ŌĆö `golangci-lint run ./...` (gate-clean; runs inside `verify-be` ŌĆö config in `.golangci.yml`: std-error-handling preset, errcheck + SA5011 off in tests)
- `make generate` / `make generate-check` ŌĆö regenerate / freshness-check `frontend/src/js/format.gen.ts` (`srr gen-ts`)
- `make verify-be` ŌĆö vet + gofmt check + lint + build + test + generate-check
- `make release VERSION=ŌĆ”` ŌĆö cross-compile all platforms (`CGO_ENABLED=0 -ldflags "-s -w"`)

Direct equivalents when working inside `backend/`: `go build -o srr .` (the Makefile artifact is `dist/srr`), `go test ./...`, `go test -run TestName .`, `go test -v ./store/`.

## Architecture

- **`main.go`** ŌĆö CLI via `alecthomas/kong` + YAML config. Source: `$SRR_CONFIG_INLINE` (raw YAML content, takes precedence) ŌåÆ `$SRR_CONFIG` (file path) ŌåÆ `$XDG_CONFIG_HOME/srr/srr.yaml`. (Remote URL config sources and a `srr config <key> <value>` file updater were built then deliberately reverted 2026-07-04 ŌĆö `SRR_CONFIG` is a local path and `srr config` is read-only.) `Globals` struct for flags. Command groups: `feed` (alias `f`): `add`, `upd`, `rm`, `ls`, `show`, `edit`, `apply`, `import`, `export`; `art` (alias `a`): `fetch`, `ls`; `asset`: `heal` (`cmd_asset.go` ŌĆö overwrite an existing `assets/` object's bytes in place, the operator repair path for a published-broken asset: key grammar enforced via `assetKeyRe` (renamed from `healKeyRe` ŌĆö now also the validation gate for expiration's asset harvesting, see `db_expire.go`), must already exist unless `--create` (re-create a referenced-but-deleted object), Content-Type from `--content-type` or the key extension; a deliberate hashŌēĀbytes break since immutable articles reference the key forever); `recipe`; `syndicate`; `gen`; `preview` (alias `p`); `config` (alias `c`); `inspect` (alias `i`); `frontend` (alias `fe`): `update`; `version`; `serve`; hidden `gen-ts`. The optional top-level `secrets:` YAML section (a flat env-var nameŌåÆvalue map) is parsed by `parseSecrets` (`secrets.go`) and pushed via `mod.SetSecrets` into `mod.SubprocessEnv`, which merges it into every external ingest/mod command's environment (secrets win over the ambient env; NOT applied to asset-process/asset-peek). An empty name or a name containing `=` is a hard error; values stay plaintext in `srr.yaml`.
- **`cmd_inspect.go` / `cmd_inspect_check.go` / `cmd_inspect_report.go`** ŌĆö `InspectCmd` (`srr inspect`): pack consistency checker. Split across three files by responsibility: command entry + fetchers (`cmd_inspect.go`: `openFetcher`/`httpFetcher`/`loadCore`/`loadDataPack`), validation checks invoked by `--validate` (`cmd_inspect_check.go`), single-mode reports for `--chron`/`--filter`/`--list-tags`/`--from-hash` (`cmd_inspect_report.go`). The idx parse + addressing mirror itself lives in `idx_read.go` (shared with `srr art ls`). Modes: `--chron N` (single-entry inspection: feed_id + resolved data packId/offset + the stored fetched_at), `--validate` (full sweep: bounds-vs-data, db-meta cross-check, feedCounts continuity across pack boundaries, unknown feed_ids, latest-pack file presence, idx header-summary coverage/content vs `hdrs`, meta coverage/consistency via `checkMeta` ŌĆö overclaim, latest tail count vs `mt`, shard sizes, every title's grams probe positive in its shard bloom, summary == concat of shard headers; db-meta expiration checks: `0 Ōēż add_idx Ōēż total_articles` and `0 Ōēż xp Ōēż total_art` range checks plus a live-count cross-check ŌĆö live idx entries at `chron Ōēź add_idx` must equal `total_art ŌłÆ xp` (`feedIDStats`; reuse-proof, since a reused id's legacy entries sit below `add_idx`) ŌĆö replacing the pre-expiration "first idx occurrence < add_idx" issue, which is now expected behavior; the all-time `total_art == idx count` check stays, a known id-reuse limitation, commented in place), `--filter <tag|feed_id>` (filter resolution + count, mirrors frontend filter math ŌĆö sums live counts, `TotalArt ŌłÆ Expired`), `--from-hash '<hash>'` (replays `nav.fromHash` end-to-end), `--list-tags` (mirrors `groupFeedsByTag`; per-tag article totals are live counts too). `--url` for HTTP CDN; otherwise uses `--store`. Diagnoses the `(reading 'f')` crash class in frontend `nav.ts` (`showFeed` accesses `data.db.feeds[article.f]`).
- **`cmd_fetch.go`** ŌĆö `signal.NotifyContext` for graceful shutdown. `errgroup` (`golang.org/x/sync/errgroup`) with `SetLimit(globals.Workers)` and a `sync.Pool` for feed buffers. Articles sorted by published time (ascending) before storage. Order: `PutArticles` ŌåÆ `SyncIdxSummary` ŌåÆ `SyncMeta` ŌåÆ `ExpireArticles` ŌåÆ `SyncOutFeeds` ŌåÆ `Commit` ŌåÆ GC sweeps ŌĆö expiration runs before the out-feed sync so the same cycle's syndication already excludes what just expired; warn-only like the sync steps, and it runs on every cycle including serve's `--interval` loop and the GUI single-feed fetch (`only`-mode) ŌĆö global maintenance, like `SyncIdxSummary`/`SyncMeta` (production `now` = `db.core.FetchedAt`) (all sync/GC steps warn-only; each GC sweep runs only when its counter ŌĆö the TAIL generation `tailGen(seqŌłÆnd)` for `GCLatest` (a delta cycle bumps `Seq` but supersedes nothing), `HdrPacks`/`MetaPacks` for the summary sweeps ŌĆö advanced this run, since most cycles fetch nothing and would just re-delete an already-empty window; a crash-leaked name is still swept by the next advancing run) ŌåÆ the ingest-cache age sweep (`sweepAssetCache`, `--cache-max-age`/`SRR_CACHE_MAX_AGE` default 72h, 0 disables; post-Commit only, deletes cache files whose mtime aged out ŌĆö see "External file self-hosting" for the mtime-as-relevance contract). `--interval` / `SRR_FETCH_INTERVAL` duration flag runs fetch in a loop; the loop body is `FetchCmd.fetchLoop(ctx, client)` (single cycle when interval Ōēż 0, else cycleŌåÆsleepŌåÆrepeat until ctx cancel, returning nil on clean shutdown and logging ŌĆö not propagating ŌĆö a failed cycle), shared with `srr serve --interval`. **Feed-subset filter** (`feedFilter`, embedded in both `FetchCmd` and `ServeCmd` so kong flattens the flags into each): `--tag`/`-g`, `--feed`/`-i`, `--exclude-tag`, `--exclude-feed` ŌĆö all `sep:","` (comma-separated **or** repeated), env `SRR_FETCH_TAG`/`_FEED`/`_EXCLUDE_TAG`/`_EXCLUDE_FEED` (an empty env value is a no-op, kong yields an empty slice), so the same restriction reaches the persistent `srr serve --interval` loop (serve copies its `feedFilter` into the constructed `FetchCmd`). `FetchCmd.selectFeeds(db)` resolves the cycle's feeds: the GUI single-feed `only` path is unchanged (exact ids, hard error on unknown); otherwise `feedFilter.apply` builds the include set = union of tag matches (`matchTag`: **hierarchical** ŌĆö exact, or under the `sel/` subtree, so `news` matches `news/tech` but not the sibling `news2`) and feed-id matches ŌĆö or **every feed when no include selector is given** ŌĆö then subtracts the exclude matches (returned sorted by id) ŌĆö and finally, on the unattended loop path only, the **dormancy backoff** thins the result (`backoffActive`/`targetInterval`/`filterDue`): each feed's poll interval drifts up as `(now ŌłÆ last_new)/8` from the loop `--interval` base (first skip after ~8├Ś base of quiet), capped at `--fetch-backoff-max`/`SRR_FETCH_BACKOFF_MAX` (default 1h; 0 = kill switch, poll everything every cycle), keyed on `LastOK` as the poll clock (`now` = `db.core.FetchedAt`). A feed that produces snaps back to base next cycle (its `LastNew` moves to ~now); failing feeds freeze `LastOK` and are never skipped (backoff must not mask an outage); one-shot runs, the GUI `only` path, and explicit include selectors (`--tag`/`--feed`) always poll at full rate. Warn-only, never abort (safe for a shared per-box config): each selector matching no feed in the store, and an empty final set, log one `slog.Warn`; a zero-feed cycle still runs global maintenance (expiration/sync/GC), like the `only` path. Error summary (fetched/failed counts) logged at end. The end-of-pipeline asset-upload step is parallel too: `Feed.fetchURL` defers self-hosting to `fetchRun.uploadAssets`, which fans out marker-bearing items over a per-feed `errgroup`. The **run-global** asset worker pool lives on the shared `assetFetcher` (`a.sem`, sized by `SRR_ASSET_WORKERS`/`--asset-workers`, default nproc, independent of `--workers`) and is held by the singleflight **leader** job only ŌĆö the goroutine that actually peeks/transcodes/uploads a unique asset ŌĆö so followers coalescing on the same bytes never pin a slot, and total concurrent peek/transcode/upload jobs across all feeds is capped regardless of how assets distribute. The singleflight body runs under the **run/shutdown ctx** (`a.baseCtx`), not the per-feed errgroup ctx, so one feed's cancellation can't poison a follower feed sharing an asset (only run shutdown aborts it; a follower whose own feed cancels bails via the `DoChan` caller-select). Marker-less items (`mod.HasAssetMarkers` is false) skip the fan-out; `FetchCmd.fetch` sizes the pool (cap Ōēź 1). **Live fetch stats** (`progress.go`): when stderr is a terminal, `runFetch` renders an in-place status line ŌĆö `fetching ┬Ę feeds done/total ┬Ę new articles N ┬Ę failed N ┬Ę assets N (N active)`, verb flipping to `saving` for the pack-write half of the cycle ŌĆö via `startFetchProgress` (200 ms ticker; feed workers bump counters where `onFeed` fires, asset job counts live on `assetFetcher.active/done`, bumped by the singleflight leader). The process-wide `status` line coordinates with logging: `main` routes the default slog handler through it (`log.SetOutput(status)`), so a log record erases the line, prints, and redraws it beneath ŌĆö plain `\r` + space padding, no ANSI. On a non-terminal stderr (services, pipes, tests) everything is a no-op/passthrough, so logs stay byte-clean.
- **`cmd_feeds.go`** ŌĆö One feed = one URL. `AddCmd` (`srr feed add`, strict-create; flags `-t/--title`, `-u/--url` (single, required), `-g/--tag`, `-r/--recipe`, `-i/--ingest` + repeatable `-p/--pipe` (feed-level {ingest, pipe} overrides on top of the recipe ŌĆö see Recipes), `-e/--expire-days N` (retention window; 0 = keep forever)), `UpdCmd` (`srr feed upd ID`, feed-level fields ŌĆö incl. `-e/--expire-days`, `-i` (`""` clears) and `-p` (a lone `-p ""` clears) ŌĆö plus `-u/--url` to repoint the source), `RmCmd` (`srr feed rm`), `LsCmd` (`srr feed ls`, filter by `-g/--tag`, yaml/json output), `ShowCmd` (`srr feed show ID`), `ApplyCmd` (`srr feed apply` reads feed JSON from `--file PATH` or stdin; accepts a single object or an array; whole-input atomic; create when `id` absent, full-replace update when `id` present, error if `id` references a missing feed), `EditCmd` (`srr feed edit ID` opens `$VISUAL`/`$EDITOR`/`vi` on the feed JSON; no-change is a no-op; id changes refused; tempfile preserved on parse or apply error). The per-feed fetch state (`ETag`/`LastModified`/`Watermark`/`BoundaryGUIDs`/`FetchError`, plus the `LastOK`/`FailStreak`/`LastNew` vitals) is preserved when the URL is unchanged across `feed upd -u ŌĆ”`, `feed apply` updates, and `feed edit` (`setFeedURL` no-ops on an unchanged URL); changing the URL resets it (a new source shares no dedup/cache history). `feedView` is the flat JSON/YAML shape `{id?, title, url, error?, tag?, recipe?, ingest?, pipe?, no_title?, expire_days?, expired?}` ŌĆö `error` and `expired` are read-only (server-owned: the last `FetchError` and the cumulative expired-entry counter), reported by ls/show/edit but never applied back. Empty-string convention on `-t`/`-g`/`-r`/`-i` clears the field where allowed (title and url cannot be cleared). `normalizeFeed` validates the recipe name exists (eager error on unknown recipe), runs the feed-level pipe through `filterPipe` + `validatePipe(ŌĆ”, allowDefault=true)` (same token rules as a recipe pipe; `#default` allowed), and bounds `expire_days` to `0..36500` (100 years). **Subscribe-time discovery (automatic, networked):** `feed add` and `feed upd -u` (only when the URL actually changes) resolve the URL through `ingest.Resolve` (the `resolveFeedURL` package var ŌĆö a test seam) when the feed's effective ingest strategy is the built-in `#feed` (`resolvesFeed` gates on `ingest.Select(feedIngest, recipe.Ingest, default.Ingest)` == `ingest.Builtin` ŌĆö the feed-level ingest override participates; external-ingest feeds are stored as-is). A homepage URL is repointed to its discovered feed; an unresolvable URL is a **hard error** that rejects the add/repoint. `feed apply` / `feed edit` deliberately stay offline. So `add`/`upd -u` are no longer purely offline ŌĆö they make one network probe.
- **`cmd_art.go`** ŌĆö `ArtCmd` (`srr art ls`): lists stored articles newest-first with cursor pagination (`-b/--before`), filter by feed ID (`-i`) or tag (`-g`), time window by `--since`/`--until`, with `-l/--limit` (default 50). Outputs JSON. `readAllIdx` walks the packs parsed by the shared `idx_read.go` reader (`loadIdxPacks` + `getPackRef`), filtering by `db.Feeds()`/`AddIdx`; `packReader` (one instance per run, shared by the window search and the content fill so no data pack is read twice) resolves an idx entry to its `ArticleData`, caching each JSONL data pack it parses and serving delta-region entries (`packID == deltaPackID`) from the already-parsed chain. **Time window** (`--since`/`--until`, a **half-open `[since, until)`** window ŌĆö `--since` inclusive, `--until` exclusive, so consecutive windows compose without overlap; either usable alone; `since >= until` and an unparseable value are hard errors): the clock is **`fetched_at`, not `published`** ŌĆö one stamp per fetch cycle applied to the whole batch makes it chron-monotone (the same property `ExpireArticles` relies on), so `packReader.findWindow` locates the window with two `sort.Search` binary searches over the entry slice, reading O(log n) data packs instead of the whole series; `published` is not monotone and would force a full scan. Consequence: a backfill's old articles ride their recent ingest time. Values parse via `parseTimeBound` ŌĆö a duration before now (Go units plus the `d`/`w` extensions `dayWeekRe` rewrites to hours, so `1d12h` composes), a bare **local** date `2006-01-02`, or an RFC3339 instant; bare unix seconds are deliberately not a form, and both bounds resolve against one `now`. `Total` counts matches **inside the window**, `--before` is clamped to it (`start = min(beforeIdx, hi)`, scan floors at `lo`) so paging can't walk out. Unlike the content fill ŌĆö which tolerates an entry addressing past its pack by leaving the row blank ŌĆö the search treats that as a hard error: one bogus timestamp would silently relocate the whole window. No short flags for `--since`/`--until`: kong flattens the globals into every command and `-s` (`--pack-size`) / `-u` are taken there.
- **`cmd_import.go`** ŌĆö OPML import with hierarchical ID selection (`-a` all, `-i` specific). Each OPML `xmlUrl` becomes one single-URL feed. OPML group hierarchy auto-resolves to hierarchical tags; `-g/--tag` overrides. Cross-folder URL dedup: a `seen` set on `importWalker` emits a URL once even when it is cross-listed in several folders (the first folder visited ŌĆö case-insensitive name order ŌĆö wins its tag). `-r/--recipe` stamps every imported feed. `-n/--dry-run` lists resulting feeds without importing. `applyImportDefaults` centralises post-walk Tag/Recipe stamping. (No merge mode ŌĆö that was removed with the Feed concept.) **Subscribe-time discovery (automatic, networked):** the shared `resolveImportBatch` pipeline (`applyImportDefaults` ŌåÆ `importRecipes` ŌåÆ up-front `validateRecipeRef`, so a typo'd `--recipe` fails before any URL is probed ŌåÆ `resolveImportFeeds`) is called by both `ImportCmd.Run` and serve's `handleImport`; `resolveImportFeeds` probes every `#feed` feed concurrently (bounded by `globals.Workers`; external-ingest feeds pass through untouched) via the `resolveFeedURL` seam, repointing homepage URLs to their discovered feed. Import is **partial-success**, not atomic on resolution: resolvable feeds are imported, unresolvable ones are skipped and reported (`reportImportFailures` ŌåÆ the test-overridable `stdout`). `--dry-run` probes too, so its preview shows resolved URLs and the skip report. `importRecipes` reads the db.gz recipes map (read-only) to gate the `#feed` decision per each feed's resolved recipe.
- **`opml.go`** ŌĆö OPML XML parsing. `ParseOPMLTree` builds `OPMLNode` tree from file; each `OPMLNode.Feed` is a single-URL `Feed`. `normalizeGroupName` converts group names to tag-safe identifiers.
- **`cmd_export.go`** ŌĆö `ExportCmd` (`srr feed export`): writes the current feeds as OPML 2.0, the inverse of import ŌĆö hierarchical tags become nested outline groups, one leaf outline per feed URL. `buildOPML`/`outlinesOf` are reused by serve's `handleExport` (`GET /api/export`).
- **`cmd_gen.go`** ŌĆö `GenCmd` (`srr gen`): prints the db.gz `gen` counter; `-b/--bump` increments and commits it via `BumpGen` (the SW cache-invalidation key after an in-place store rebuild; also resets `HdrPacks`/`MetaPacks`/`MetaTail` ŌĆö see db.go).
- **`utils.go`** ŌĆö reflection/print helpers for `srr config`: `toKebab`, `fieldName`, `maskSecret`, `printFields`.
- **`cmd_preview.go`** ŌĆö `preview` subcommand: loads a recipe (`-r/--recipe`, default `default`) from db.gz, applies optional ad-hoc overrides (`-p` pipe steps, `-i` ingest strategy) with **feed-level override semantics** ŌĆö `renderPreview` resolves `resolvePipe(resolvePipe(default.Pipe, recipe.Pipe), pipeOverride)` and `previewFetch` resolves `ingest.Select(ingestOverride, recipe.Ingest, default.Ingest)`, so `-p`/`-i` simulate exactly what a feed carrying those overrides would run (`#default` in `-p` expands to the recipe's effective pipe) ŌĆö dispatches through the ingest engine, runs the module pipeline, and serves rendered articles via local HTTP server (`-a/--addr`). Opens the DB read-only.
- **`cmd_serve.go` / `serve_overview.go` / `serve_feeds.go` / `serve_recipes.go` / `serve_syndicate.go` / `serve_fetch.go` / `serve_tools.go`** ŌĆö `ServeCmd` (`srr serve`): a localhost web admin GUI. Serves an embedded `webui/` bundle (`//go:embed`) at `/` and a JSON API under `/api/*`. Each handler opens its own `withDB` scope per request ŌĆö read-only (no lock) for GET, `withDB(true)` (acquireŌåÆcommitŌåÆrelease) for mutations; no persistent lock or DB handle. A Host/Origin loopback allowlist (`hostGuard`) blocks cross-origin access to the mutating API ŌĆö with one carve-out: a non-loopback `Origin` is allowed when the browser-set `Sec-Fetch-Site: same-origin` fetch metadata asserts the request came from the GUI's own page, which is how a Host-rewriting proxy deployment works (cloudflared tunnel + dashboard `httpHostHeader: localhost:8088`, e.g. `srr.llera.eu` ŌĆö without the carve-out every browser mutation 403'd while GETs sailed through, since browsers only attach `Origin` to non-GET requests). The Host check stays unconditional (DNS-rebinding pages are same-origin to the browser but can't present a loopback Host). Binds `localhost:8088` by default (`--addr`/`SRR_SERVE_ADDR`). Optional `--interval`/`SRR_SERVE_INTERVAL` (default 0 = off) runs the same all-feeds fetch loop in-process ŌĆö a background goroutine driving `FetchCmd.fetchLoop` under the server's signal context, so one Ctrl-C/SIGTERM stops both; a running cycle holds the store lock for its duration, so a concurrent GUI mutation gets the same 409 contract as a separate fetch process. Reuses CLI logic via shared helpers: `saveFeed` (feed upsert + subscribe-time discovery), `setRecipe`/`removeRecipe` (also back `srr recipe`), `setOutFeed`/`removeOutFeed` (also back `srr syndicate`), `renderPreview` (also backs `srr preview`), and `FetchCmd.runFetch(ctx, client, onFeed func(feedProgress))` (also backs `srr art fetch`; the unexported `FetchCmd.only []int` field restricts a cycle to those feed ids ŌĆö set by the SSE `/api/fetch?id=N` handler for the GUI's per-row fetch, empty = every feed, unknown id fails the cycle in-band; the SSE stream carries per-feed progress). Lock contention ŌåÆ 409 (`os.ErrExist`); a triggered fetch holds the lock for its duration like the cron loop. `/api/inspect` captures the existing `srr inspect` report. **`GET /api/overview`** (`serve_overview.go`) is the single-read store snapshot every read-tab renders from: feeds projected to the UI list shape (map-key id + friendly `error`, plus writable `expire_days`/`dedup_days`/`dedup_title` and read-only server-owned `expired`, still omitting the per-feed cache/dedup internals bg/etag/wm), the derived tag buckets (live article counts: `TotalArt ŌłÆ Expired` ŌĆö the overview is a display projection), `db.core.Recipes` directly (no defensive copy ŌĆö the map is consumed read-only for JSON marshaling), the syndication `out` list, `gen`/`fetched_at`/`total_art`, the effective store-wide `dedup_days` default (the Tools tab renders/edits it), `cdn_url` (from `globals.CdnURL`, omitted when unset ŌĆö lets the syndicate tab link the live `out/<name>` files), and `version` (the running binary's version ŌĆö the webui header labels itself with it, `#ver` next to the wordmark) ŌĆö one db.gz read per request. (All projection logic is inlined into `getOverview`; the granular `GET /api/feeds`/`/api/tags`/`/api/recipes`/`/api/syndicate`/`/api/gen` read endpoints were removed ŌĆö the webui only ever called `/api/overview`.) Full API surface (`registerAPI`): `GET /api/overview`, `POST /api/feeds`, `POST /api/feeds/apply`, `PUT|DELETE /api/feeds/{id}`, `PUT|DELETE /api/recipes/{name}`, `PUT|DELETE /api/syndicate/{name}`, `GET /api/preview`, `GET /api/resolve` (read-only add-feed probe: ingest-fetch the URL through a recipe ŌåÆ `{url, title, items}` ŌĆö the discovery-resolved URL, the feed's own channel-level title, item count; optional `&ingest=` carries the feed-level ingest override; advisory, save re-resolves), `POST /api/gen/bump`, `PUT /api/dedup` (store-wide default seen.gz dedup horizon ŌĆö the Tools tab's Dedup pool panel, mirroring `srr dedup --days N`; days in [0, 36500], 0 resets to the built-in default, echoes the effective value), `GET /api/export`, `POST /api/import` (import + export live in the Tools tab's OPML panel; the webui only ever calls the `dry_run=1` half: every OPML feed lands in a review sheet ŌĆö per-row include checkbox, editable title/tag/recipe ŌĆö with already-subscribed URLs pre-unchecked and dry-run resolution failures kept as pre-unchecked rows too, error inline, rather than dropped: the probe ran the default recipe's ingest, so an "unresolvable" URL may just need a recipe with an external ingest. The kept rows commit via `POST /api/feeds/apply`, atomic and offline ŌĆö resolved URLs were folded by the dry run, unresolved ones store as-is. The non-dry import-all commit path stays for direct API users), `GET /api/inspect`, `/api/fetch` (SSE; optional `?id=N` restricts the cycle to one feed ŌĆö malformed id ŌåÆ 400 pre-stream, unknown id ŌåÆ in-band error event). Webui (`webui/app.js`): the Feeds tab is a triage-first health console ŌĆö a fifth `stale` grade (fetching fine but no new article in 30 days, `STALE_AFTER`) joins ok/warn/err/idle, board stats are grade-filter toggles, a store-pulse strip alarms when the store-wide `fetched_at` ages past 6h/24h (`storePulse`; ### Ingest (`ingest/`)

An "ingest strategy" is the I/O+parse layer that produces `mod.RawItem`s from a URL. The default zero-config behaviour is the historical RSS/Atom/RDF path (`#feed`); alternative strategies ŌĆö shipped out-of-repo as external shell commands ŌĆö cover URLs that aren't feeds. Selection per feed: `ingest.Select(feed.Ingest, recipe.Ingest, default.Ingest)` (variadic, first non-empty wins) ŌĆö the feed-level override, falling back to the feed's resolved recipe's ingest, falling back to the `default` recipe's ingest, falling back to `#feed`. Empty strings fall through; persistence uses `omitempty`.

Built-in names start with `#` and resolve through the package's `Register` registry (init-time). Anything not starting with `#` is treated as a shell command per the external-ingest protocol. Only `#feed` ships built-in; source-specific strategies live out-of-repo as external commands.

Mirrors `mod/`'s architecture: a `Fetcher` struct (built by `New()`) holds the registered built-in `FetchFunc`s in a nameŌåÆfunc map; `Fetcher.Fetch(ctx, args, ...)` dispatches by name ŌĆö built-ins for matching keys, shell-command fallthrough for anything else.

- **`main.go`** ŌĆö `Request` / `Result` structs, `FetchFunc` type (`func(ctx, *http.Client, buf, Request) (Result, error)`), direct registration `Register(name, FetchFunc)` (built-ins are stateless, so `New()` shares the registry rather than instantiating one per call), `Select(feed, global) string` precedence helper (final `#feed` fallback, exported as the `Builtin` const ŌĆö the token callers compare against to gate subscribe-time discovery), the `Fetcher` struct + `New() *Fetcher`, and `Fetcher.Fetch(ctx, args, client, buf, req)` which routes `args` to a built-in if registered or executes it as a shell command otherwise. External wire schema is the JSON encoding of `Request` (stdin) and `Result` (stdout) ŌĆö see struct tags; stderr is passthrough. A non-zero exit code or empty stdout is a hard error (fails just that feed's fetch); stdout is capped at 64 MiB. Items on the wire are `mod.RawItem` records, so external strategies emit `guid` as the FNV-32a hash (uint32) of any stable per-item string, matching the dedup contract used by built-ins. `published` is RFC3339 (null/absent for dateless). The author-facing protocol spec + a reference implementation live in `backend/README.md` (Ingest Strategies); the protocol tests are in `ingest/external_test.go`.

**External file self-hosting.** The ingest engine owns no asset machinery: it just sets the external command's working directory to `Request.AssetDir` (`cmd.Dir`) so the command stashes/references files with relative paths. The caller (`main`) owns the rest. `FetchCmd.fetch` creates **one cache dir shared by every feed this run** = `globals.CacheDir` (`MkdirAll`, kept across runs; **always set** ŌĆö the kong `${cacheDir}` default covers unset, main()'s post-parse floor covers an explicitly empty `cache-dir:`, both resolving `defaultCacheDir()` = `os.UserCacheDir()/srr`), stores it on `fetchRun.cacheDir` (creation is **mandatory** ŌĆö never empty, since a command handed an empty working dir would run in SRR's cwd), and `Feed.fetchURL` hands it to **every** feed as both `Request.AssetDir` and the upload-step `cacheDir` ŌĆö no strategy gate, so a built-in fetcher may download into it too. Whichever fetcher ran owns the file layout inside the shared dir. The upload itself is the **automatic end-of-pipeline step** `fetchRun.uploadAssets` (run from `Feed.fetchURL` after `processItem`, which stays a pure, store-free transform; it fans marker-bearing items out concurrently, the peek/transcode/upload work bounded by the `assetFetcher`'s run-global `SRR_ASSET_WORKERS` pool (held by the singleflight leader job)): for each item it calls `mod.RewriteAttrs(content, fn)` (the generic walk over `assetAttrs` ŌĆö img/video/audio `src`, video `poster`, + `a href`), and `fn` scans for **`#`-markers** ŌĆö a value starting with `#` whose remainder names an existing regular file in `cacheDir` (e.g. `#/photo.jpg`) ŌĆö uploading each through `assetFetcher.UploadCacheRef` (content-hash key, upload-if-absent) and rewriting the marker to the returned key. `UploadCacheRef` classifies its guard failures with the `errNotAsset` sentinel ŌĆö references naming no regular file (an ordinary `#fragment` anchor) or resolving outside the cache dir (attacker-influenced content) are **declined** by `fn` (the marker stays untouched, one containment implementation lives in `UploadCacheRef` alone); any other upload failure (store error) returns the error from `fn`, which `RewriteAttrs` surfaces so `Feed.fetchURL` **hard-fails the whole feed** ŌĆö feed state (watermark/dedup/etag) is left untouched, so a transient failure self-heals next fetch. (Oversize is no longer one of these: the `--max-asset-size` cap is enforced **at download** by `#selfhost`/ingest, so an over-cap asset is dropped there ŌĆö keeping its remote URL ŌĆö and never reaches the upload step.) `<video poster>` is **not** a usable target *for ingest-injected markers*: `#sanitize`'s poster regex (`^(https?://|assets/)`) strips a `#`-poster before the upload step runs (an ingest strategy's markers ride the pipeline from the start, so they pass through `#sanitize`). The built-in `#selfhost` mod, by contrast, runs *after* `#sanitize` (placed after `#default`) and **does** self-host posters ŌĆö its markers are created post-sanitize and never re-sanitized. Download dedup is the command's (cache check); upload dedup is SRR's (content hash + store existence check). The cache is **self-maintaining**: after each cycle's Commit, `sweepAssetCache` (cmd_fetch.go) deletes files unused for longer than `--cache-max-age`/`SRR_CACHE_MAX_AGE` (default 72h; 0 disables). Relevance = **mtime**: `UploadCacheRef` touches every source file it consumes (before the memo/store-hit/upload branching, so all paths count), and the cache consumers also touch on reuse (`srr-telegram`'s size-validated cache check, `#selfhost`'s URL cache), so any file a live feed still references stays fresh ŌĆö including a big backlog warming up across failing cycles ŌĆö while orphans (dropped items' media, interrupted-run debris, consumed downloads nothing re-references) age out. A `FetchFunc` owns only I/O and parsing ŌĆö dedup, watermarking, pipeline modules, and storage all stay in `Feed.fetchURL`/`processItem` and operate uniformly on `Result.Items`. Built-in `FetchFunc`s and the shell-command path must be concurrent-safe (one `*Fetcher` is shared across workers; unlike `*mod.Module` it isn't pooled).
- **`feed.go`** ŌĆö Built-in `#feed` strategy (registered inline in `init`): the zero-config default. HTTP GET with `If-None-Match` / `If-Modified-Since` hints into the shared per-worker buffer, then `ParseFeed`. Returns `NotModified` on 304; populates `ETag` / `LastModified` for the next request. **Feed auto-discovery**: when `ParseFeed` reports `errNotFeed` (the document is not a recognized feed ŌĆö an HTML page, unknown XML root, or non-XML ŌĆö *as opposed to* a genuine fault parsing a real feed) and the response looks like HTML (`text/html` Content-Type or body starting with `<!doctype html`/`<html`), `discoverFeedLink` (`feed_discover.go`) scans for `<link rel="alternate" type="application/rss+xml|atom+xml|ŌĆ”">` in the HTML head, fetches the discovered URL, and sets `Result.ResolvedURL`; `feed.go` persists `ch.URL = ResolvedURL` without resetting dedup/etag/vitals. One-hop guard: a discovered page that is also HTML-without-feed returns the original error and does not recurse. **`errNotFeed`** is the unexported sentinel that classifies "not a feed" so the discovery branch (here, fetch-time) never fires on a broken-but-real feed. **`Resolve(ctx, client, rawURL, maxSize)`** is the subscribe-time wrapper: it runs the same `feedFetch` discovery once and returns the canonical feed URL (rawURL if it already serves a feed, the discovered target if it's an advertising HTML page, or an error if neither) ŌĆö used by `feed add`/`upd -u`/`import` (see `cmd_feeds.go`/`cmd_import.go`). Also houses the streaming RSS/Atom/RDF parser: `ParseFeed(data, fn) (title, err)` (returns the feed's own channel/feed-level title ŌĆö parent-checked so an `<item>`/`<image>` title never masquerades as it; surfaced as `Result.Title`, consumed by serve's `GET /api/resolve`; a non-feed document is reported wrapped in `errNotFeed`; a fault parsing a recognized feed is a plain error) and the package-private `hash(string)` used for GUID stability (the FNV-32a contract external fetchers replicate).
- **`feed_discover.go`** ŌĆö `discoverFeedLink(htmlData, baseURL)`: scans HTML for the first `<link rel="alternate">` with a recognised feed MIME type using `golang.org/x/net/html.NewTokenizer`; returns the href resolved against baseURL. Case-insensitive rel (including multi-valued), tolerant of malformed/partial HTML.

### Store (`store/`)

Low-level storage interface: `Get`/`Put`/`AtomicPut`/`Rm`/`Close`. Registry selects by URL scheme; local = empty scheme `""`. Config registry: backends call `RegisterConfig` in `init()` with a config struct pointer; `LoadConfigs` reads YAML sections into them. `main.go` also owns the write-once pack-name grammar: the `PackSeries` seriesŌåÆkinds table (the contract's single source ŌĆö emitted to the frontend as `PACK_SERIES_KINDS` by `srr gen-ts`) builds `packKeyRe`, which `cacheControlForKey` uses to stamp immutable Cache-Control on pack keys (strict per-series: `data/h3.gz` is not a pack name). `cacheControlForKey` also revalidates db.gz and every seen.gz sidecar object (`isSeenObject`: the `seen.0.gz`/`seen.1.gz` ping/pong slots plus the legacy bare `seen.gz` name), and classifies the self-hosted frontend shell at the store root (`srr frontend update`): `index.html`/`manifest.webmanifest`/`sitemap.txt` revalidate, content-hashed root assets (`feHashedRe` = `<name>.<8+ hex>.<ext>`, slash-free so it never matches a pack key) are immutable. Its sibling `contentTypeForKey` stamps the meta-unset Content-Type the same grammar-driven way: db.gz + seen sidecar objects (`isSeenObject`) + `packKeyRe` matches ŌåÆ `application/gzip` (honest type for SRR's own gzip objects; deliberately NO `Content-Encoding: gzip` ŌĆö the reader gunzips manually via `DecompressionStream`, so a transparently-decompressing CDN/browser would hand it pre-inflated bytes and break it), everything else `application/octet-stream`. S3 + HTTP backends only (local/SFTP ignore Cache-Control/Content-Type, deferring to the static server's mime-by-extension).

| Method | Signature | Behavior |
|---|---|---|
| `Get(ignoreMissing)` | returns `io.ReadCloser` | Controls error-on-missing; streaming read |
| `Stat` | returns `int64` | Object size without reading the body (os/sftp stat, S3 HeadObject ŌĆö its missing-key code is the bodyless `NotFound`, not `NoSuchKey` ŌĆö HTTP HEAD); missing key = `(0, nil)`, silent like Rm ŌĆö expiration's AssetBytes accounting treats absent as zero (a retried expire cycle re-stats keys an aborted predecessor already deleted), and the upload dedup probe (`resolveAndUpload`) trusts only `size > 0` as a hit, falling back to the `Get` probe on 0 since a HEAD can't tell absence from a zero-byte object or a missing Content-Length |
| `Put(ignoreExisting)` | accepts `io.Reader` | Controls overwrite vs exclusive create; streaming write |
| `AtomicPut` | accepts `io.Reader` | temp-then-rename (local/SFTP); overwrite (S3); streaming write. Local/SFTP also sweep stale `<key>.tmp.<pid>.<n>` staging leftovers in the target's directory (age-gated `tempSweepMaxAge` 24h via `staleTemp`, best-effort): a hard kill between temp-create and rename strands a uniquely-named file no GC speaks, so without the sweep each such crash leaked one store file forever. The sweep runs *after* the call's own staging file is created and ages leftovers against **that file's** mtime read from the same directory listing, never `time.Now()` ŌĆö both sides of the comparison must come from the store's clock, which on an SFTP server or an NFS/SMB mount is not this host's (and taking the reference from the listing costs no extra round-trip). Our own entry is skipped, and a missing reference or a zero mtime (a server omitting it in READDIR attrs) means no sweep at all: no reference, no judgement. The gate is what protects a live writer's in-flight temp, and concurrent asset uploads inside ONE process already put several in a shard |
| `Rm` | ŌĆö | Silent on missing (local/SFTP warn + return nil; S3 unconditional delete) |

- **`local.go`** ŌĆö Auto-creates subdirs via `os.MkdirAll`.
- **`s3.go`** ŌĆö `IfNoneMatch` precondition headers + CRC32 checksums. `S3Config`: region, endpoint, profile, static credentials.
- **`sftp.go`** ŌĆö Auto-creates subdirs via `client.MkdirAll`. Auth chain: URL password ŌåÆ config password ŌåÆ config/default private key ŌåÆ `~/.ssh/` keys ŌåÆ SSH agent ŌåÆ error. Host key verification via `~/.ssh/known_hosts` by default (`SFTPConfig.Insecure` to skip).
- **`http.go`** ŌĆö Plain HTTP object store (WebDAV-style / S3-compatible endpoint), registered for both `http` and `https` schemes (one shared `http` config section): `Get`=GET (404/410 ŌåÆ missing), `Put`/`AtomicPut`=PUT (AtomicPut is a plain overwrite like S3; metadata rides request headers ŌĆö Content-Type from meta, then `contentTypeForKey` (db.gz + seen slots + pack names ŌåÆ `application/gzip`), then the `application/octet-stream` default; Content-Encoding only when meta sets it; Cache-Control from `cacheControlForKey` ŌĆö servers that don't store metadata serve their own headers like local/SFTP), `Rm`=DELETE (404/410 ŌåÆ nil), `Stat`=HEAD (size from Content-Length, -1/absent counts as 0; 404/410 ŌåÆ `(0, nil)`). Exclusive create maps to `If-None-Match: *` with 412 wrapped in `os.ErrExist` (the lock/409 contract); a server that ignores conditionals makes `.locked` best-effort. Auth: URL userinfo (net/http basic auth), `HTTPConfig.Token` (bearer), or `HTTPConfig.Headers` (extra headers on every request, e.g. Cloudflare Access service tokens; an explicit `Authorization` entry wins over Token; both secret-tagged ŌĆö the map masks whole via `maskSecret`'s map case. `SRR_HTTP_HEADERS` env-overrides it through `loadEnv`'s `map[string]string` case (`parseEnvMap`: comma-separated `Name: value` entries, split on the first colon, trimmed; replaces the YAML map whole)); `Insecure` skips TLS verification. **Redirect guard**: `CheckRedirect` follows redirects only when the original request was a GET or HEAD (`Stat`) ŌĆö net/http otherwise silently replays a 301/302/303-redirected PUT/DELETE as a bodiless GET and reports the GET's status, turning a write that never happened into a false success (found by conformance audit; pinned by `TestHTTPWriteRedirectFailsLoudly`). No client timeout by design (uploads may run long; every op carries the caller's ctx); URLs log via `Redacted()` so userinfo passwords never leak. One shared `http` config section for both schemes (no `SRR_HTTPS_*` names, deliberate).

**Delta-segment tail** (2026-07-17, spec `docs/DELTA-TAIL-SPEC.md` ŌĆö read it before touching this): `PutArticles` is split into an **accounting preamble** (TotalArticles / per-feed TotalArt / ContentBytes, once per article at ingest) and a materialization path chosen by `shouldConsolidate` ŌĆö chain cap `--max-deltas`/`SRR_MAX_DELTAS` (default `maxDeltasDefault`=12, exported as `MAX_DELTAS`; **0 = kill switch, byte-identical pre-delta behavior** ŌĆö what the zero-value test globals run), byte cap `--max-delta-bytes` (`dby` tracks it), or a 5k-meta-stratum crossing (invariant I2). A **delta cycle** (`emitDelta`) first re-validates the consolidated tail idx (`checkTailIntact` ŌåÆ `checkLatestIdx` against `tc`; warn on a transient read, fatal on a structural mismatch ŌĆö restoring the per-cycle tail check the pre-delta writer ran, so a corrupt tail fails loud NOW instead of at the next consolidation ~maxDeltas cycles later), then publishes the batch as immutable `data/d<Seq+1>.gz` (`deltaKey`), bumps `Seq`/`nd`/`na`/`dby`, and maintains the head projection itself (`DBCore.extendHead` ŌĆö SyncMeta no-ops, `tailCovered` unmoved). A dormant feed with no chain entry (the common case) is removed with NO consolidation ŌĆö a reused id can't confuse a replay that holds none of the old incarnation's articles ŌĆö so membership is checked by only READING the chain (`loadDeltaChain`), and only a chain that can't be parsed at all (corrupt segment / unreachable store) blocks removal. `tailGen(core) = Seq ŌłÆ nd` anchors `latestKey`; `tailCovered(core) = TotalArticles ŌłÆ na` is the packŌåödelta seam (`idxKeyAndSize` returns the PHYSICAL tail size; callers extend with the chain). `GCLatest` cutoff `tailGen ŌłÆ keep ŌłÆ 1 ŌłÆ 2┬ĘMaxDeltas` is the grace boundary; the sweep is a **low-water drain** ŌĆö clears `(GCLatestSwept, cutoff]`, advances `GCLatestSwept` (`gcs`) only over generations actually cleared (per-run bound `gcMaxSweep`) ŌĆö so a missed/failed sweep or a lowered `--max-deltas` never permanently strands an `L<g>`/`d<g>` name below a fixed window (tailGen JUMPS at consolidation). Sweeps `L<g>` of all three series + `d<g>`. `BumpGen` deliberately does NOT touch `nd`/`na`/`dby`/`gcs`.

Backend-specific behavior:
- `PutArticles` and `savePack` manage the idx and data series. `SyncMeta` manages the meta/ series. Order: `PutArticles` ŌåÆ `SyncIdxSummary` ŌåÆ `SyncMeta` ŌåÆ `ExpireArticles` ŌåÆ `SyncOutFeeds` ŌåÆ `Commit` ŌåÆ GC sweeps.
- Consolidation writes binary idx and JSONL data directly; splits idx packs at every 50,000 articles (accounting ŌĆö each article's uncompressed JSONL line length charged to its feed's `ContentBytes` ŌĆö lives in `PutArticles`' preamble). It tracks the data-pack boundaries (where `NextPackID` advances) into a per-pack slice and emits them as the idx footer at each finalize and at the tail-pack save; on the append path it strips the loaded tail pack's old footer (keeping header+entries) and recovers its boundaries via `parseIdxFooter` before extending.
- `ArticleData` struct: `{ FeedID, FetchedAt, Published, Title, Link, Content }` ŌĆö serialized as JSONL with short keys `f/a/p/t/l/c` ŌĆö plus the in-memory-only `Lang` (`json:"-"`, from `Item.Lang`; see `RawItem.Lang` in the Module System section): present during the write cycle, never in the pack bytes, always empty on the read side (`parseDataPack`).
- `DBCore.Feeds` is `map[int]*Feed`; serialized as a JSON object keyed by feed ID. `AddFeed` assigns the first free ID in `[0, feedIDCeiling)` (`feedIDCeiling` = 65536, since `feed_id` is a u16 in each idx entry) and returns an error if all slots are taken; it sets `AddIdx = TotalArticles` and explicitly zeroes `Expired` (a fresh incarnation has expired nothing ŌĆö a reused id's legacy entries sit below the new `AddIdx`, keeping the id-reuse invariant local by construction). `NewDB` range-checks loaded ids and rejects a feed with an empty `URL` (a pre-merge `feeds[]`-only db.gz) with a delete-and-re-fetch message. `RemoveFeed(ctx, id)` first drains a live delta chain (`DrainDeltas` ŌĆö see the delta-tail paragraph above), then `delete`s and purges the seen sidecar state.
- `seq` names the newest tail OBJECT (delta segment `data/d<seq>.gz` or, at consolidation, the tail packs `idx|data|meta/L<seq>.gz`; the live tail packs are `L<tailGen>` = `L<seqŌłÆnd>`): each article-producing cycle writes generation `seq+1`, bumps `Seq` only after its saves succeed, and `Commit` publishes it ŌĆö names are write-once, so a generation can never be seen half-written or rewritten. `GCLatest(ctx, keep)` (called from `cmd_fetch` after `Commit` when the tail generation advanced, `latestKeep` = 2, warn-only) deletes expired generations for all three series plus `d<g>`, sweeping a `gcSweepWindow+MaxDeltas`-wide trailing window (tailGen JUMPS at consolidation) so crash-leaked packs self-heal.
- `hdrs` names the idx header summary (`idx/h<hdrs>.gz`): `cmd_fetch` calls `SyncIdxSummary` between `PutArticles` and `Commit` (warn-only ŌĆö a failed ~1KB summary write must not discard a durable batch; readers fall back to eager idx loading and the next run retries), then `GCSummaries` next to `GCLatest`. `srr gen --bump` resets `HdrPacks` to 0 (an in-place rebuild invalidates the copied headers).
- `mp`/`mt` name the meta shards and latest tail (`MetaPacks`/`MetaTail`): `cmd_fetch` calls `SyncMeta` between `SyncIdxSummary` and `Commit` (warn-only); `GCMetaSummaries` runs next to `GCSummaries`; `GCLatest` also sweeps `meta/L<g>` generations. `srr gen --bump` resets both `MetaPacks` and `MetaTail` to 0.
- `withDB(locked, fn)` / `withDBCtx(ctx, locked, fn)`: most cmd `Run()` methods are now a single `withDB` call wrapping the work.

### Module System (`mod/`)

Pipeline per-feed during fetch. Factory pattern: `New()` returns a fresh stateful processor; each built-in registers a `func() Processor` factory that runs once per `New()` so it can capture per-instance state (compiled policies, a shared HTTP client). A built-in processor is `mod.Processor` = `func(context.Context, Params, *RawItem) error` (the ctx is the per-fetch context, threaded through `Module.Process`); most built-ins ignore the ctx, but network-bound ones like `#readability` use it for cancellation/deadline.

**File layout** ŌĆö `main.go` holds the engine (registry + `Module`/`Process` dispatch), the `RawItem` wire types, and the external/shell-module exec. Support helpers carry a `helper_` filename prefix (`helper_params.go`, `helper_assets.go`, `helper_ssrf.go` ŌĆö the latter exports `SafeTransport`, the SSRF-guarded `*http.Transport` used by `#readability`) to set them apart from the built-in module files, which are named for the module itself (`sanitize.go`, `minify.go`, `readability.go`, each with a matching `_test.go`). `secrets.go` holds `SetSecrets`/`SubprocessEnv` ŌĆö the process-envŌŖĢ`srr.yaml`-secrets merge (secrets win, deduped, sorted) used as the `env` for both `mod.New()` and `ingest.New()`'s external commands.

**Module parameters** ŌĆö a pipeline token may carry `key=value` options after the name, e.g. `#readability timeout=30s maxbody=16MiB`. `Module.Process` splits the token on whitespace; when the first field is a registered built-in it parses the rest into `mod.Params` (see `helper_params.go`) and dispatches internally, otherwise the original string runs as a shell command (so shell commands with spaces/`=` are untouched). The param split is **quote-aware** (`splitParamFields`): a double-quoted span keeps its spaces and drops the quotes, so a value with spaces stays one param (`ua="Mozilla/5.0 (X11)"`); an unterminated quote is a hard error, and there is no escape for a literal `"`. Built-ins read options via typed accessors with a default for the absent case ŌĆö `Params.Duration(key, def)`, `Params.Bytes(key, def)` (sizes accept `B`/`KB`/`MB`/`GB` decimal or `KiB`/`MiB`/`GiB` binary; bare = bytes), `Params.String(key, def)` (verbatim value; explicitly empty = hard error) ŌĆö and call `Params.only(allowed...)` to reject unknown keys. A malformed value or unknown key is a hard error (loud config feedback), distinct from a module's per-article fail-open behaviour; modules that take no options reject any param. Parameters are parsed per-item but the cost is negligible. External (shell) modules don't get parsed params ŌĆö they receive their args on the command line.

- `#sanitize` ŌĆö bluemonday, content-only. Allows `<audio>` (`src`/`controls`/`preload`, mirroring `<video>`) so `#selfhost` can self-host audio.
- `#minify` ŌĆö tdewolff/minify, content-only.
- `#readability` ŌĆö for feeds that syndicate only a teaser/summary: HTTP-GETs the item's `Link` (honest keyword-free UA ŌĆö `+full-text-extractor` was dropped 2026-07-03, keyword-scanning WAFs 406 on "extractor"; default 20s timeout, 8 MiB body cap) and replaces `Content` with the article's main body extracted via `go-shiori/go-readability`. Tunable per pipeline position via `timeout=` (Go duration), `maxbody=` (byte size), `ua=` (request User-Agent ŌĆö quote it, real UAs have spaces; some WAFs 406 on UA keywords, e.g. blogdechollos.com blocks any UA containing "extractor") and `selector=` (CSS selector group via `cascadia`; when set, the matches' HTML ŌĆö every match in document order, inner HTML per match, childless matches like `<img>` render the element itself, matches nested in an earlier match are folded in ŌĆö replaces the readability heuristic entirely: the per-site escape hatch for pages whose short genuine body loses the candidate scoring to text-dense sidebar/widget chrome (deal blogs), and via comma groups also the way to stitch disjoint blocks (hero image + body) into one article; a selector miss keeps the original content and WARNs, it never falls back to the heuristic), e.g. `#readability ua="Mozilla/5.0 (compatible; SRR/1.0)" selector=div.entry-content`; the timeout is enforced via a per-call `context.WithTimeout` so the shared client can carry different budgets. **Fail-open**: empty/invalid link, non-2xx, fetch/parse error, or empty extraction all leave `Content` untouched and return nil (logged at WARN) ŌĆö one bad article never fails the fetch (but a bad `timeout=`/`maxbody=` value, or an unknown param, is a hard error). Place it **before** `#sanitize`/`#minify` (e.g. recipe pipe `["#readability", "#default"]`) so the fetched HTML gets clamped to the allowed element set. Network-bound, so unlike the other built-ins it honours the fetch `context` (cancellation/deadline).
- `#filter` ŌĆö content-based item dropping (`filter.go`). Params (all optional; item dropped if ANY condition fires): `drop_title=/regex/[i]`, `keep_title=/regex/[i]`, `drop_content=/regex/[i]`, `keep_content=/regex/[i]`, `min_words=N`, `keep_lang=en,es` (ISO 639-1 allowlist over the item's `i.Lang` ŌĆö the pre-pipe detection stamp or a declared value; **no detection of its own**, and **fail-open**: empty Lang keeps the item; codes validated against `validLangCode`, and BOTH sides folded through `normalizeLang` so `EN`/`en-US`/`en` are one code ŌĆö a wire-declared `"ES"`/`"es-ES"` compared raw against a lowercased set dropped every item in the feed. A **macrolanguage** code expands config-side via `langMacro` to every variety detection can report under it: `no` admits `no`/`nb`/`nn`, since whatlanggo only ever classifies the varieties, and folding `no` to one of them silently discarded a Norwegian feed's Nynorsk half). On a match sets `i.Drop=true; return nil`. Unknown params / bad regex / bad int / unknown language code are hard errors. **Every present param is parsed before ANY condition is evaluated**: `Module.Validate` validates by running the processor against an empty sentinel item, so a firing condition returning early used to let a malformed *later* param escape config validation entirely and only surface per-item at fetch ŌĆö where a pipeline error merely drops the article with a warning, so the feed silently ingested nothing. Does NOT mutate GUID/Published/Title/Content/Link/Lang.
- `#dedupmedia` ŌĆö duplicate-media removal (`dedupmedia.go`): an article never shows the same picture (or video/audio) twice ŌĆö the best copy survives. Groups `<img>`/`<video>`/`<audio>` by normalized file identity (`mediaFileID`: scheme + query/fragment ignored, WordPress `-WxH` size suffix stripped, `iN.wp.com` Photon proxy unwrapped; identity is per element type ŌĆö an `<img>` and a `<video>` off one URL never merge) and removes all but the richest copy per group (`dedupScore`: canonical un-suffixed/un-proxied URL > alt text > declared pixel area > attribute count; ties keep document order), pruning wrapper chains a removal leaves without text or keep-worthy elements (`<a>`/`<p>`/`<figure>` ŌåÆ gone; `dedupKeepTags` keep them alive). Inline-glyph images are exempt (`isMediaGlyph`) ŌĆö repetition is meaningful there: emoji classes (`emoji`/`wp-smiley`/`smiley`), `height: 1em`-style, `s.w.org/images/core/emoji` URLs, Ōēż2-rune emoji alt, both sides Ōēż 96px declared, blank/spacer/pixel/placeholder/loading file names. No params (`p.only()`). Pure CPU, never fails an item: unparseable content and no-op passes return verbatim (no re-render). **Place BEFORE `#sanitize`** (`["#dedupmedia", "#default"]`) so the glyph heuristics still see the class/style attributes the sanitizer strips. Motivating case: WordPress "featured image in feed" plugins prepending a copy of an image the body repeats (finofilipino.org ŌĆö every post shipped its image twice).
- `#unlazy` ŌĆö lazy-load recovery (`unlazy.go`): promotes the real URL out of `data-src`-style attributes (`lazySrcAttrs`, URL-shaped values only ŌĆö a boolean `data-lazyload="true"` never becomes a src) into `src` on `<img>`/`<video>`/`<audio>`; an `<img>` whose src is still missing or a placeholder (`lazyPlaceholderRe`: data URIs, about:blank, 1x1/spacer/loading names) takes the best `srcset`/`data-srcset` candidate (largest width descriptor, else density, else first) ŌĆö a genuine src is never overridden by srcset; `<noscript>` fallbacks contributing an image the document doesn't already show (by `mediaFileID` identity) are unwrapped in place, redundant ones dropped (the parser treats noscript children as raw text, so the payload is re-parsed). No params (`p.only()`). Pure CPU, fail-open, verbatim no-op. **Place BEFORE `#sanitize`** ŌĆö after it the `data-*` attributes holding the real URLs are gone. Essential in pipes using `#readability` (live article pages are where lazy-load markup comes from; without this they publish with blank/1-px images).
- `#embed` ŌĆö iframe-to-link conversion (`embed.go`): `#sanitize` deletes `<iframe>` wholesale, so a post built around an embed publishes with its point silently missing. Rewrites known providers (`classifyEmbed`: YouTube incl. shorts + nocookie + protocol-relative ŌĆö `/embed/videoseries` playlists excluded; Vimeo `player.vimeo.com/video/<digits>`; Dailymotion classic + `geo.dailymotion.com/player.html?video=`; Spotify `open.spotify.com/embed/*`) into a `<p>` holding the derivable thumbnail (YouTube `i.ytimg.com/vi/<id>/hqdefault.jpg`, Dailymotion `/thumbnail/video/<id>`) wrapped in a link, plus always a "Ō¢Č"-prefixed text link (labelled by the iframe `title` attr, else "Watch on <Provider>") ŌĆö the thumbnail may 404 and the frontend collapses broken images, so the target stays reachable. Unknown iframes (ads included) are deliberately left for `#sanitize` to strip. No params. Pure CPU, fail-open, verbatim no-op. **Place BEFORE `#sanitize`**; composes with `#selfhost` (self-hosts the injected thumbnail).
- `#enclosure` ŌĆö out-of-body media surfacing (`enclosure.go`): prepends media that rides in item metadata the body never mentions ŌĆö RSS `<enclosure>`, Atom `<link rel="enclosure">`, Media RSS `media:content`/`media:thumbnail` (directly or inside `media:group`, YouTube's layout), `<itunes:image>` ŌĆö as leading `<p>`-wrapped `<img>` / `<video controls>` / `<audio controls>` blocks. Reads the parsed entry from `RawItem.Raw` (type-asserts `RawFeedItem`; **no-ops for external-ingest items** and on url-less `content` entries, which are text bodies not Media RSS). Picks the best image (full images beat thumbnails, then largest declared area), first video, first audio; kind from MIME prefix, then `medium` attr, then URL extension (`enclosureExtKinds`) ŌĆö YouTube's `x-shockwave-flash` content entry falls out while its thumbnail survives. Skips candidates already visible in the body (exact URL, HTML-escaped URL, or `mediaFileID` identity ŌĆö a WordPress size variant counts as present); non-http(s) URLs ignored. No params. Pure CPU, fail-open, verbatim no-op. **Place BEFORE `#sanitize` and `#dedupmedia`** (`["#enclosure", "#dedupmedia", "#default"]`).
- `#untrack` ŌĆö tracking-freight removal (`untrack.go`): three cleanups. (1) Beacon images: an `<img>` matching `trackerPixelRe` (feedburner `~4`, Mailchimp `list-manage.com/track/open`, `pixel.wp.com`, WordPress.com stats, GA `collect`/`__utm.gif`, Facebook `tr`, scorecardresearch, quantserve, doubleclick, Yandex) or declaring both sides Ōēż 2px is removed + `pruneEmptyWrappers` ŌĆö the size rule skips any `<img>` still carrying a lazy `data-src` URL (a not-yet-promoted placeholder can declare 1x1). (2) Tracking query params (`utm_*` prefix + the `trackingParams` set: fbclid/gclid/mc_eid/ŌĆ”) stripped from `a href`, media `src`, `video poster` (`untrackAttrs`) ŌĆö remaining params keep order and encoding verbatim, non-http(s) URLs never touched, emptied query drops its `?` (fragment kept). (3) The trailing WordPress "The post ŌĆ” appeared first on ŌĆ”" footer (`wpTrailerRe`, matched whitespace-collapsed against the **last** element only ŌĆö the phrase mid-article survives). No params. Pure CPU, fail-open, verbatim no-op. **Place BEFORE `#sanitize`, AFTER `#unlazy`** (`["#unlazy", "#untrack", "#default"]`).
- `#selfhost` ŌĆö downloads remote `<img>`/`<video>`/`<audio>` media (src + `<video poster>`, not `<a href>`) into the run's shared cache dir and rewrites each to a `#`-marker, so the end-of-pipeline upload step peeks/converts (`SRR_ASSET_PROCESS`)/uploads it to `assets/`. Network-bound (SSRF-guarded `SafeTransport`, like `#readability`); reads the run cache dir from the fetch context (`mod.WithCacheDir`, set in `feed.go:Feed.Fetch`) and **no-ops when absent** (e.g. `srr preview`). Params: `timeout=` (default 120s), `maxbody=` (default 128MiB, **clamped to `--max-asset-size`** via the `mod.MaxAssetSize` package var so the self-host object cap is enforced here at download ŌĆö an over-cap body never lands in the cache); bad/unknown param is a hard error. **Fail-open per asset**: a bad URL / non-2xx / oversize / SSRF-blocked dial / write error leaves the remote URL in place (WARN). Download dedup is URLŌåÆcache-file; upload dedup is `UploadCacheRef`'s content hash. **Place AFTER `#default`** (`["#default", "#selfhost"]`): it then downloads only sanitizer-approved media and its markers (incl. `<video poster>`) never round-trip through `#sanitize`. Limitations: with `webify` as `SRR_ASSET_PROCESS`, audio is self-hosted but not transcoded (webify is image/video only); `<source>` children stay stripped; cross-run URL cache may serve stale bytes (clear the cache dir).
- External modules: `/bin/sh -c`, stdin/stdout JSON (`RawItem`), stderr passthrough. Emitting `{"drop":true}` on stdout sets `i.Drop=true` (via the existing `json.Unmarshal(out, i)` path ŌĆö no peek struct needed); subsequent pipeline steps are skipped and the item is not stored. Dropped items retain their GUID in the feed boundary. The author-facing protocol spec + a reference implementation live in `backend/README.md` (External mod protocol); the protocol tests are in `mod/main_test.go`.

#### Asset self-hosting (`assetFetcher.UploadCacheRef` + `RewriteAttrs`)

SRR self-hosts files an ingest strategy or the built-in `#selfhost` mod leaves in the run's shared cache dir: it stores them under a content-hash key (`assets/<2-hex>/<16-hex><ext>`) and rewrites the in-content reference to that key. This is **not** a pipeline module ŌĆö it is the automatic end-of-pipeline step `fetchRun.uploadAssets`, run from `Feed.fetchURL`.

- Implementation `assetFetcher` (`assets.go`): `UploadCacheRef(ctx, cacheDir, localname)` returns `(key, uploadedBytes, err)` ŌĆö uploadedBytes is the payload size of the Put THIS call led (0 on memo/store dedup hits and for singleflight followers, so summing it counts each stored object exactly once; it feeds `Feed.AssetBytes`). It resolves `localname` inside `cacheDir` (guards: containment via `EvalSymlinks`+`Rel`, reject non-regular/symlink), keys by `contentHashKey` (sha256 of the **original source file bytes**, with the source extension) and `Put`s only if the key isn't already in the store (`Backend.Stat` existence probe ŌĆö zero-body; `size == 0` falls back to the body-carrying `Get` check, which stays authoritative for the cases a HEAD can't prove: a zero-byte stored asset, an HTTP store omitting Content-Length). The `SRR_MAX_ASSET_SIZE` cap is **enforced at download** (`#selfhost` / external ingest via `Request.MaxAssetSize`), so `UploadCacheRef` trusts the cache file and does **not** re-check the source size. It does no outbound HTTP. Built once per `fetch` run in `cmd_fetch.go` from `db.Backend`, shared across workers (concurrent-safe). The external-ingest upload step (`fetchRun.uploadAssets` ŌåÆ `rewriteItemAssets`, run from `Feed.fetchURL`; see Ingest ┬¦) calls it once per `#`-marker against the run's shared cache dir (`fetchRun.cacheDir` = `assetCacheRoot()`).
- **Per-asset pre-upload processing (`SRR_ASSET_PROCESS` / `globals.AssetProcess`):** an optional operator-configured command (e.g. `webify -m 720`) run on **every** asset just before upload to process its bytes ŌĆö image/video transcoding being the motivating case (covers external-ingest and any internal-mod marker uniformly, since both funnel through this one step). It runs **only on a store miss**: the existence check keys on the **source** hash *before* processing, so an asset already uploaded is never re-processed (the common case for an image reused across articles/feeds). `runProcess` substitutes the cache file path for each `{input}` token in the command (per arg, e.g. `conv -i {input} --flags`), or appends it as the final arg when absent. With a `{output}` token the command runs in **file mode**: SRR substitutes a fresh staging path under `<cache-dir>/_processed/` (`assetFetcher.procDir`, set by cmd_fetch; OS temp dir fallback when unset ŌĆö big transcodes must not land in a tmpfs `/tmp`, and a crash-leaked output inside the cache tree ages out via the post-cycle sweep; success removes it immediately), the command writes its processed bytes there, and prints a `{mimetype,extension,encoding}` JSON to stdout ŌĆö the mimetype/encoding ride the upload as the stored object's `store.ObjectMeta` (Content-Type/-Encoding), the extension is for the peek-decided key (see asset-peek). Without `{output}`, processed bytes are read from stdout (no declared metadata, so the object gets the `application/octet-stream` default). stderr is **captured, not passed through** (asset commands narrate progress on stderr ŌĆö webify's tty progress bar ŌĆö and parallel jobs would garble srr's own output/status line): `mod.RunCommandTimeout` buffers the most recent 4 KiB (`tailBuffer`) and on failure folds the trailing lines (CRs count as line breaks) into the returned error, so the warn line carries the diagnostic; on success it is discarded. This is deliberately unlike `RunSubprocess` ŌĆö external ingest/mod commands keep passthrough stderr as their documented log channel. **Fail-soft** ŌĆö a non-zero exit, empty output, or (file mode) invalid metadata JSON uploads the original unchanged, so a processing hiccup or an unhandled file type never wedges a feed. The command is bounded by its **own** timeout `--asset-process-timeout` / `SRR_ASSET_PROCESS_TIMEOUT` (default **0 = unlimited**: no deadline, bounded only by run cancellation on SIGINT/SIGTERM, since media transcoding can run arbitrarily long), separate from the shared `--cmd-timeout` ŌĆö which governs ingest/mod commands only and **never** affects asset work: both `runProcess` and asset-peek's `runPeek` run through `mod.RunCommandTimeout(ctx, procTimeout, ŌĆ”)`, where a `procTimeout <= 0` adds no deadline. The dedup key always hashes the **source** bytes (the existence check must run pre-process), but its **extension** comes from `asset-peek` when configured (else the source extension); the asset-process **output** keeps a fail-soft size guard in **both** modes (`readProcOutput` for `{output}` mode, an inline `len(out)` check for stdout mode): an over-`SRR_MAX_ASSET_SIZE` output uploads the original instead ŌĆö that output is generated here and isn't download-bounded, unlike the source. Empty `SRR_ASSET_PROCESS` disables it (pure pass-through, the previous behaviour). Asset jobs run concurrently across a feed's items; the peek/transcode/upload work is bounded globally by `SRR_ASSET_WORKERS` (the `assetFetcher`'s `a.sem` pool, held by the leader job only). A marker referenced by several items (same feed or across feeds ŌĆö the run's `assetFetcher` is shared) is coalesced by `assetFetcher.flight` (a `singleflight.Group` keyed on the source content hash) via `DoChan`, so one concurrent wave peeks/transcodes/uploads the asset exactly once and the rest receive the leader's key without taking a slot ŌĆö a waiter whose own feed cancels bails on its caller ctx instead of blocking on the leader. The body runs under the run/shutdown ctx (`a.baseCtx`), not the leader feed's, so the leader's result is shared with the wave (a shared asset that genuinely fails to upload fails every referencing feed, since none can publish content pointing at a missing asset; each self-heals next fetch) WITHOUT one feed's cancellation poisoning a healthy follower feed. The `seen` memo still short-circuits the cheaper sequential/cross-call hit (and a wave's late arrivals) before `flight`, and the store-existence check stays the cross-run backstop. The matching out-of-repo media converter is `webify` (`~/ws/webify`, on PATH as `~/.local/bin/webify`): auto-detects imageŌåÆWebP / videoŌåÆWebM, `webify <input>` writes to stdout.
- **Per-asset identification (`SRR_ASSET_PEEK` / `globals.AssetPeek`, the `assetFetcher.peek` field set in `cmd_fetch.go`):** an optional probe run on **every** asset **before** the dedup check (so it runs even for already-uploaded assets ŌĆö the accepted cost). `runPeek` substitutes/append the cache path like `runProcess` and parses a `{mimetype,extension,supported,encoding}` JSON from stdout. The **extension** becomes the stored object's key extension (`contentHashKey(peek.extension, sha256(source))` ŌĆö so a transcoded `.gif`ŌåÆWebP rides a `.webp` key while dedup still keys on the source bytes); the **mimetype/encoding** become the object's `store.ObjectMeta`; **supported=false** hosts the original bytes and **skips** `asset-process` (peek reports the real source type for an unsupported asset). Precedence: in `{output}` mode `asset-process`'s declared mimetype/encoding (the actual result) override peek's; a `process.extension` disagreeing with the peek key only **warns** (the key is already chosen). **Fail-soft**: a peek error / invalid JSON falls back to the source extension and `supported=true`. Empty disables (the object then gets the source extension + `application/octet-stream` default). **Corrupt-media guard**: a peek that runs fine but identifies NOTHING (`supported=false`, empty/`octet-stream` mimetype, empty extension) on a source whose extension claims media (`isMediaExt`) classifies the file as broken bytes ŌĆö `UploadCacheRef` returns the `errCorruptAsset` sentinel, `rewriteItemAssets` declines the marker with a WARN (the article publishes without working media; the reader collapses the dead element) instead of hosting garbage (the chron-437 bug) or failing the feed (the bytes won't improve on retry ŌĆö a hard error would wedge it). Identified-but-unsupported formats and non-media extensions keep the host-as-is contract. **Cycle-level degradation warning**: per-run `assetFetcher.procFailed`/`corrupt` counters (bumped on each peek/process command failure and each corrupt decline) are surfaced by `runFetch` as one WARN per cycle ŌĆö a systemic cause (webify missing from the service PATH) otherwise drowns in per-asset noise while every asset silently ships unprocessed with `octet-stream`.
- `RewriteAttrs(content, fn)` (`mod/helper_assets.go`): the generic HTML attribute walk over `assetAttrs` (img/video/audio `src`, video `poster`, **plus `a href`**, the generic-file set) behind the upload step. The caller's `fn` returns `(newVal, ok, err)` and owns the `#`-marker upload policy ŌĆö which values are markers, how files are stored, and which failures are fatal (a non-nil `err` aborts the walk and fails the feed) ŌĆö keeping that policy out of `mod`.
- Idempotency: download dedup is the ingest strategy's (its cache check); upload dedup is SRR's ŌĆö `UploadCacheRef` skips the `Put` (and the `SRR_ASSET_PROCESS` run) when the source-hash key is already in the store (`Backend.Stat` probe, `Get` fallback on size 0), so a re-run re-uploads and re-processes nothing.
- `assets/` is a reserved store prefix in the writerŌåöreader contract (analogous to `idx/`/`data/`); the frontend (`fmt.ts`) resolves it against the pack base. The store's `AtomicPut` carries a `store.ObjectMeta{ContentType, ContentEncoding}`: `S3` stamps it (with meta unset, Content-Type falls to `contentTypeForKey` ŌĆö SRR's own gzip objects, db.gz + seen slots + pack-grammar names, declare `application/gzip`; anything else gets `application/octet-stream` ŌĆö an asset's type is never derived by extension or byte-sniffing, peek/process is its single source of truth); local/SFTP ignore it and rely on the static server's mime-by-extension at request time. So a peek-corrected key **extension** is what fixes local/SFTP rendering, while the explicit mimetype hardens S3.
- `GUID` and `Published` are immutable for all modules (built-in or external; change = error). Enforced in `processItem` after each pipeline step ŌĆö the captured value before the step is compared to the post-step value, attributing changes to the offending module.
- `RawItem.Raw` is set by the ingest parser (`ingest/feed.go`) to the parsed entry as `mod.RawFeedItem` (`map[string][]mod.RawField`); modules can type-assert it for typed access. External (shell) modules see the same data as JSON via the short keys `@`/`$`/`+` (text/attrs/children).
- `RawItem.Drop bool` (`json:"drop,omitempty"`) ŌĆö set to `true` by a pipeline step (built-in or external) to signal that the item should be discarded. `processItem` short-circuits on the first `Drop=true`: returns nil, skips remaining steps and post-loop normalization. Callers (`feed.go` `fetchURL`, `cmd_preview.go`) check `i.Drop` after `processItem` returns nil and `continue` to skip storing/rendering the item. The GUID is already in the boundary map by the time the second-pass loop runs, so a dropped item stays deduped on the next fetch without any extra handling.
- `RawItem.Lang string` (`json:"lang,omitempty"`) ŌĆö the article's ISO 639-1 language code, **backend-internal only**: stamped by the **always-on PRE-pipeline detection** in `processItem` (`mod.DetectLang` in `mod/helper_lang.go`: whatlanggo trigram detection over title + HTML-stripped content ŌĆö `extractText` in `helper_html.go`, **hard** 4 KiB cap on the extracted TEXT, a single whitespace-free token truncated on a rune boundary rather than written whole, which is what bounds no-space scripts like Japanese; the *document* is deliberately parsed in full ŌĆö a byte cut on markup is not a cut on text, and truncating let text-sparse markup either empty the extract or leave only a short foreign boilerplate line that classifies confidently, which `keep_lang` would then drop for good; fail-open gate Ōēź 24 runes at Ōēź 0.8 confidence, else Lang stays empty; an ingest-declared value is never clobbered), so every pipe step sees it ŌĆö `#filter keep_lang` decides from it, and a shell mod can read/override it on the external wire. A post-pipe re-attempt runs only when the pre-pipe pass came up empty AND `DetectLang`'s inputs changed. `langCode` is the one LangŌåÆ639-1 mapping ŌĆö whatlanggo's table corrected by `iso6391Extra` for Western Persian (`fa`) and Eastern Yiddish (`yi`), which it maps to `""` despite those languages having 639-1 codes; without the correction `keep_lang=fa` was an "unknown code" that failed `Validate` and took the whole feed down. Carried `RawItem` ŌåÆ `Item` ŌåÆ `ArticleData.Lang` (`json:"-"`) for same-cycle use ŌĆö never serialized to the data packs, invisible to readers and `gen-ts` (which skips `json:"-"`). Mutable, unlike GUID/Published.

### Recipes

Processing config lives in named `{ingest, pipe}` recipes (`DBCore.Recipes map[string]Recipe`), referenced by feeds via `Feed.Recipe`. The reserved `default` recipe (always present, seeded `["#sanitize","#minify"]` by `NewDB`) is the fallback.

- A feed with empty/absent `Recipe` resolves to `default` via `recipeFor`.
- Each axis falls back to `default` independently: a recipe with only `Ingest` set uses its own ingest and `default`'s pipe; only `Pipe` ŌćÆ its pipe and `default`'s ingest.
- **Feed-level overrides**: `Feed.Ingest` (`ingest`) and `Feed.Pipe` (`pipe`) sit on top of the recipe, each axis independently ŌĆö set, they win over the recipe's value; empty, they inherit it. Resolved at the start of `Feed.Fetch`: `resolvePipe(resolvePipe(default.Pipe, recipe.Pipe), feed.Pipe)` and `ingest.Select(feed.Ingest, recipe.Ingest, default.Ingest)` (variadic first-non-empty).
- `#default` (`pipeDefault`) expands inline to the next pipe down the fallback chain: in a recipe's pipe, the `default` recipe's pipe; in a feed's pipe, the feed's *effective recipe pipe* (already `#default`-expanded). The `default` recipe itself forbids `#default` (enforced by `validatePipe(allowDefault = name != defaultRecipeName)` in `RecipeSetCmd`); a feed pipe always allows it (`normalizeFeed` runs `filterPipe` + `validatePipe(ŌĆ”, true)`).
- CLI: `srr recipe set <name> [-i ingest] [-p step]ŌĆ”` (add/replace); `srr recipe ls/show/rm`; `srr feed add/upd/import -r <name>`; feed-level overrides via `srr feed add/upd [-i ingest] [-p step]ŌĆ”` (on `upd`, `-i ""` clears the ingest override and a lone `-p ""` clears the pipe override). Every `-p` flag is **repeatable, one step per flag** (`sep:"none"`).
- Clean break, amended: a pre-recipes db.gz's legacy **root** `pipe`/`ingest` still drop on load (DBCore has no such json tags) and `NewDB` re-seeds `default` if absent ŌĆö but the legacy **feed-level** keys are the same keys `Feed.Ingest`/`Feed.Pipe` use today, so those values revive as feed-level overrides (deliberate: the old per-feed meaning matches the new one; pinned by `TestNewDBLegacyPipeIngest`).

## Conventions

- **Imports**: stdlib ŌåÆ external ŌåÆ internal, blank-line separated, alphabetical within groups
- **Errors**: Always wrap with `fmt.Errorf("context: %w", err)`. No custom error types; the only package-level sentinels are `errNotFeed` (`ingest/feed.go`), `errNotAsset` and `errCorruptAsset` (`assets.go`), and `errExpireDone` (`db_expire.go` ŌĆö the expiration walk's early-stop channel, since `walkArticles`' callback contract has no other) ŌĆö don't add more without need.
