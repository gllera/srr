# CLAUDE.md

## Project

**SRR Backend** — Static RSS Reader Backend. Go CLI that fetches RSS/Atom/RDF feeds into gzip-compressed pack series (idx/, data/). Backends: local filesystem, S3, SFTP.

## Commands

`go build -o srr .`, `go test ./...`, `go test -run TestName .`, `go test -v ./store/`. Release: `CGO_ENABLED=0 go build -ldflags "-s -w"`. No Makefile/linter/Dockerfile.

## Architecture

- **`main.go`** — CLI via `alecthomas/kong` + YAML config. Source: `$SRR_CONFIG_INLINE` (raw YAML content, takes precedence) → `$SRR_CONFIG` (file path) → `$XDG_CONFIG_HOME/srr/srr.yaml`. `Globals` struct for flags. Command groups: `feed` (alias `ch`): `add`, `upd`, `rm`, `ls`, `show`, `edit`, `apply`, `import`; `art` (alias `a`): `fetch`, `ls`; `pipe`; `preview` (alias `p`); `config` (alias `c`); `inspect` (alias `i`); `version`.
- **`cmd_inspect.go` / `cmd_inspect_check.go` / `cmd_inspect_report.go`** — `InspectCmd` (`srr inspect`): pack consistency checker. Split across three files by responsibility: command entry + fetchers (`cmd_inspect.go`: `openFetcher`/`httpFetcher`/`loadCore`/`loadDataPack`), validation checks invoked by `--validate` (`cmd_inspect_check.go`), single-mode reports for `--chron`/`--filter`/`--list-tags`/`--from-hash` (`cmd_inspect_report.go`). The idx parse + addressing mirror itself lives in `idx_read.go` (shared with `srr art ls`). Modes: `--chron N` (single-entry inspection w/ recovered fetched_at vs stored), `--validate` (full sweep: bounds-vs-data, db-meta cross-check, feedCounts/fetchedAts continuity across pack boundaries, unknown feed_ids, latest-pack file presence, idx header-summary coverage/content vs `hdrs`, search coverage/consistency via `checkSearch` — overclaim, latest tail count vs `srcht`, shard sizes, every title's grams probe positive in its shard bloom, summary == concat of shard headers), `--filter <tag|feed_id>` (filter resolution + count, mirrors frontend filter math), `--from-hash '<hash>'` (replays `nav.fromHash` end-to-end), `--list-tags` (mirrors `groupFeedsByTag`). `--url` for HTTP CDN; otherwise uses `--store`. Diagnoses the `(reading 'f')` crash class in frontend `nav.ts` (`showFeed` accesses `data.db.feeds[article.f]`).
- **`cmd_fetch.go`** — `signal.NotifyContext` for graceful shutdown. `errgroup` (`golang.org/x/sync/errgroup`) with `SetLimit(globals.Workers)` and a `sync.Pool` for feed buffers. Articles sorted by published time (ascending) before storage. Order: `PutArticles` → `SyncIdxSummary` → `SyncSearch` → `Commit` → GC sweeps (all sync/GC steps warn-only; each GC sweep runs only when its counter — `Seq`/`HdrPacks`/`SearchPacks` — advanced this run, since most cycles fetch nothing and would just re-delete an already-empty window; a crash-leaked name is still swept by the next advancing run). `--interval` / `SRR_FETCH_INTERVAL` duration flag runs fetch in a loop. Error summary (fetched/failed counts) logged at end.
- **`cmd_feeds.go`** — One feed = one URL. `AddCmd` (`srr feed add`, strict-create; flags `-t/--title`, `-u/--url` (single, required), `-g/--tag`, `-p/--parsers`, `-i/--ingest`), `UpdCmd` (`srr feed upd ID`, feed-level fields plus `-u/--url` to repoint the source), `RmCmd` (`srr feed rm`), `LsCmd` (`srr feed ls`, filter by `-g/--tag`, yaml/json output), `ShowCmd` (`srr feed show ID`), `ApplyCmd` (`srr feed apply` reads feed JSON from `--file PATH` or stdin; accepts a single object or an array; whole-input atomic; create when `id` absent, full-replace update when `id` present, error if `id` references a missing feed), `EditCmd` (`srr feed edit ID` opens `$VISUAL`/`$EDITOR`/`vi` on the feed JSON; no-change is a no-op; id changes refused; tempfile preserved on parse or apply error). The per-feed fetch state (`ETag`/`LastModified`/`Watermark`/`BoundaryGUIDs`/`FetchError`) is preserved when the URL is unchanged across `feed upd -u …`, `feed apply` updates, and `feed edit` (`setFeedURL` no-ops on an unchanged URL); changing the URL resets it (a new source shares no dedup/cache history). `feedView` is the flat JSON/YAML shape `{id?, title, url, error?, tag?, pipe?, ingest?}` — `error` is the read-only last `FetchError`, never applied back. Empty-string convention on `-t`/`-g`/`-p`/`-i` clears the field where allowed (title and url cannot be cleared).
- **`cmd_art.go`** — `ArtCmd` (`srr art ls`): lists stored articles newest-first with cursor pagination (`-b/--before`), filter by feed ID (`-i`) or tag (`-g`), with `-l/--limit` (default 50). Outputs JSON. `readAllIdx` walks the packs parsed by the shared `idx_read.go` reader (`loadIdxPacks` + `getPackRef`), filtering by `db.Feeds()`/`AddIdx`; `loadContent` parses JSONL data packs to fill in the per-article `ArticleData` (title, link, published, content) from the data pack.
- **`cmd_import.go`** — OPML import with hierarchical ID selection (`-a` all, `-i` specific). Each OPML `xmlUrl` becomes one single-URL feed. OPML group hierarchy auto-resolves to hierarchical tags; `-g/--tag` overrides. Cross-folder URL dedup: a `seen` set on `importWalker` emits a URL once even when it is cross-listed in several folders (the first folder visited — case-insensitive name order — wins its tag). `-p/--parsers` and `--ingest` stamp every imported feed. `-n/--dry-run` lists resulting feeds without importing. `applyImportDefaults` centralises post-walk Tag/Pipe/Ingest stamping. (No merge mode — that was removed with the Feed concept.)
- **`opml.go`** — OPML XML parsing. `ParseOPMLTree` builds `OPMLNode` tree from file; each `OPMLNode.Feed` is a single-URL `Feed`. `normalizeGroupName` converts group names to tag-safe identifiers.
- **`cmd_preview.go`** — `preview` subcommand: dispatches a URL through the ingest engine (`-i/--ingest` picks the strategy, otherwise falls back to the db.gz root `Ingest` then `#rss` via `ingest.Select`), runs module pipeline (`-p`), serves rendered articles via local HTTP server (`-a/--addr`). Opens the DB read-only to read the root default.
- **`cmd_ingest.go`** — `IngestCmd` (`srr ingest [strategy]`): top-level command for the db.gz root `ingest` field. Mirrors `srr pipe`: no args prints, `""` alone clears, otherwise replaces.
- **`feed.go`** — `Feed` carries one source URL plus its own fetch state: Title, `URL`, ETag `etag`, LastModified `last_modified`, Watermark `wm`, BoundaryGUIDs `bg`, FetchError `ferr`, Tag, Pipe, `Ingest`, TotalArt, AddIdx (the `Feed` type was folded in — there is no longer a `Feeds []*Feed`). `Feed.Fetch` resolves the effective pipeline once via `resolvePipe(rootPipe, ch.Pipe)` and the effective ingest strategy once via `ingest.Select(ch.Ingest, rootIngest)`, `Validate`s the pipeline once, then calls `Feed.fetchURL` for the single URL, setting/clearing `ch.FetchError`. `fetchURL` (the former `feed.fetch`) dispatches I/O+parse through the shared `*ingest.Fetcher` engine, then applies the dedup/watermark/pipeline path on `ingest.Result.Items`. Dedup model: `Watermark` is the max published unix-second ever seen and `BoundaryGUIDs` is the union of (GUIDs at `Watermark` in the most recent fetch) and (dateless GUIDs in the most recent fetch). Repopulated each non-empty fetch from the current response (no carry-over) so its size stays bounded by what the publisher currently exposes, and hard-capped at `maxBoundaryGUIDs` (1024) so one misbehaving feed can't bloat db.gz — the kept set is the cap smallest hashes (deterministic per response) and over-cap items are skipped entirely rather than ingested-but-forgotten, which would duplicate them on every subsequent fetch; a `NotModified` result or a non-modified response with zero items preserves prior `Watermark`/`BoundaryGUIDs` so the dedup state survives a transient empty feed. An item is new iff its GUID isn't in the prior fetch's `BoundaryGUIDs` AND (`pub == 0` OR `pub >= Watermark`). Item `pub` is clamped to `fetchedAt` so a publisher CMS bug that ships a far-future date can't poison `Watermark`. Within-fetch dedup uses a per-GUID set checked first. Trade-off: items at `Watermark` or dateless items that disappear from the feed for one fetch and reappear later are re-ingested as duplicates (snapshot semantics over carry-over). Also hosts the `resolvePipe` helper, the `#base` token constant, the `maxBoundaryGUIDs` const, and `uint32Set`. (`feed.go` is gone — its logic moved here.)
- **`cmd_pipe.go`** — `PipeCmd` (`srr pipe [pipe...]`): top-level command for the db.gz root `pipe` field. No args prints; `""` alone clears; otherwise replaces. Also hosts `filterPipe` (drops empty strings; returns nil for all-empty input as the CLI "clear" sentinel).
- **`cmd_gents.go`** — `GenTSCmd` (`srr gen-ts`, hidden): generates `frontend/src/js/format.gen.ts` — the TS side of the writer↔reader data contract — by reflecting over the live Go declarations (the `tsConsts` table references the db.go format constants; `store.PackSeries` is emitted as the `PACK_SERIES_KINDS` grammar table; `tsTypes` walks the json tags of `ArticleData`/`SearchEntry`/`Feed`/`DBCore`, omitempty → `?`, nil-able without omitempty → `| null`). `IFeedWire` is now flat (`url`/`etag?`/`last_modified?`/`wm?`/`bg?`/`ferr?` directly on the feed); `IFeedWire` is no longer emitted. `//go:generate go run . gen-ts` (run via `make generate`); `--check` compares instead of writing and is wired into `make verify` (`verify-be`), so editing a format atom without regenerating fails loudly. A struct reachable from `tsTypes` but not listed in it is a hard generator error.
- **`cmd_config.go`** — `ConfigCmd` (`srr config`): prints resolved configuration. With no argument, prints all globals and active backend config; with a key argument, prints a single field. Reads from `store.Configs()` for backend-specific sections.
- **`processing.go`** — Per-item transformation utilities shared by `Feed.fetchURL` and `PreviewCmd`: `processItem` runs the module pipeline, enforces GUID/Published immutability, then normalises Title (via `bluemonday` strict policy + whitespace collapse) and strips control chars from Link/Content. Also hosts `validFeedURL` (used by `cmd_feeds.go` and `opml.go`).

### Ingest (`ingest/`)

An "ingest strategy" is the I/O+parse layer that produces `mod.RawItem`s from a URL. The default zero-config behaviour is the historical RSS/Atom/RDF path (`#rss`); alternative strategies — shipped out-of-repo as external shell commands — cover URLs that aren't feeds. Selection precedence per feed: `Feed.Ingest` (most specific) > `DBCore.Ingest` (db.gz root, managed by `srr ingest`) > built-in `#rss`. Empty strings fall through; persistence uses `omitempty`.

Built-in names start with `#` and resolve through the package's `Register` registry (init-time). Anything not starting with `#` is treated as a shell command per the external-ingest protocol. Only `#rss` ships built-in; source-specific strategies live out-of-repo as external commands.

Mirrors `mod/`'s architecture: a `Fetcher` struct (built by `New()`) holds the registered built-in `FetchFunc`s in a name→func map; `Fetcher.Fetch(ctx, args, ...)` dispatches by name — built-ins for matching keys, shell-command fallthrough for anything else.

- **`main.go`** — `Request` / `Result` structs, `FetchFunc` type (`func(ctx, *http.Client, buf, Request) (Result, error)`), factory registration `Register(name, func() FetchFunc)` (the outer factory runs once per `New()` so built-ins can capture per-instance state — same pattern as `mod.Register`), `Select(feed, global) string` precedence helper (final `#rss` fallback), the `Fetcher` struct + `New() *Fetcher`, and `Fetcher.Fetch(ctx, args, client, buf, req)` which routes `args` to a built-in if registered or executes it as a shell command otherwise. External wire schema is the JSON encoding of `Request` (stdin) and `Result` (stdout) — see struct tags; stderr is passthrough. A non-zero exit code or empty stdout is a hard error (fails just that feed's fetch); stdout is capped at 64 MiB. Items on the wire are `mod.RawItem` records, so external strategies emit `guid` as the FNV-32a hash (uint32) of any stable per-item string, matching the dedup contract used by built-ins. `published` is RFC3339 (null/absent for dateless). The author-facing protocol spec + a reference implementation live in `backend/README.md` (Ingest Strategies); the protocol tests are in `ingest/external_test.go`.

**External file self-hosting.** The ingest engine owns no asset machinery: it just sets the external command's working directory to `Request.AssetDir` (`cmd.Dir`) so the command stashes/references files with relative paths. The caller (`main`) owns the rest. `FetchCmd.fetch` creates **one cache dir shared by every feed this run** = `assetCacheRoot()` (`MkdirAll`, kept across runs), stores it on `fetchRun.cacheDir` (creation is **mandatory** — never empty, since a command handed an empty working dir would run in SRR's cwd), and `Feed.fetchURL` hands it to **every** feed as both `Request.AssetDir` and the upload-step `cacheDir` — no strategy gate, so a built-in fetcher may download into it too. Whichever fetcher ran owns the file layout inside the shared dir. The upload itself is the **automatic end-of-pipeline step** inlined in `Feed.fetchURL` (run after `processItem`, which stays a pure, store-free transform): for each item it calls `mod.RewriteAttrs(content, fn)` (the generic walk over `assetAttrs` — img/video `src`/`poster` + `a href`), and `fn` scans for **`#`-markers** — a value starting with `#` whose remainder names an existing regular file in `cacheDir` (e.g. `#/photo.jpg`) — uploading each through `assetFetcher.UploadCacheRef` (content-hash key, upload-if-absent) and rewriting the marker to the returned key. `UploadCacheRef` classifies its guard failures with the `errNotAsset` sentinel — references naming no regular file (an ordinary `#fragment` anchor) or resolving outside the cache dir (attacker-influenced content) are **declined** by `fn` (the marker stays untouched, one containment implementation lives in `UploadCacheRef` alone); any other upload failure (oversize, store error) returns the error from `fn`, which `RewriteAttrs` surfaces so `Feed.fetchURL` **hard-fails the whole feed** — feed state (watermark/dedup/etag) is left untouched, so a transient failure self-heals next fetch while a permanently-rejected asset wedges the feed until fixed. `<video poster>` is **not** a usable target: `#sanitize`'s poster regex (`^(https?://|assets/)`) strips a `#`-poster before the upload step runs. Download dedup is the command's (cache check); upload dedup is SRR's (content hash + store existence check). The cache is disposable, no eviction (ops-managed). A `FetchFunc` owns only I/O and parsing — dedup, watermarking, pipeline modules, and storage all stay in `Feed.fetchURL`/`processItem` and operate uniformly on `Result.Items`. Built-in `FetchFunc`s and the shell-command path must be concurrent-safe (one `*Fetcher` is shared across workers; unlike `*mod.Module` it isn't pooled).
- **`rss.go`** — Built-in `#rss` strategy (registered inline in `init`): the zero-config default. HTTP GET with `If-None-Match` / `If-Modified-Since` hints into the shared per-worker buffer, then `ParseFeed`. Returns `NotModified` on 304; populates `ETag` / `LastModified` for the next request. Also houses the streaming RSS/Atom/RDF parser: `ParseFeed(data, fn)`, `ErrStopFeed` sentinel, and the package-private `hash(string)` used for GUID stability (the FNV-32a contract external fetchers replicate).

### Store (`store/`)

Low-level storage interface: `Get`/`Put`/`AtomicPut`/`Rm`/`Close`. Registry selects by URL scheme; local = empty scheme `""`. Config registry: backends call `RegisterConfig` in `init()` with a config struct pointer; `LoadConfigs` reads YAML sections into them. `main.go` also owns the write-once pack-name grammar: the `PackSeries` series→kinds table (the contract's single source — emitted to the frontend as `PACK_SERIES_KINDS` by `srr gen-ts`) builds `packKeyRe`, which `cacheControlForKey` uses to stamp immutable Cache-Control on pack keys (strict per-series: `data/h3.gz` is not a pack name).

| Method | Signature | Behavior |
|---|---|---|
| `Get(ignoreMissing)` | returns `io.ReadCloser` | Controls error-on-missing; streaming read |
| `Put(ignoreExisting)` | accepts `io.Reader` | Controls overwrite vs exclusive create; streaming write |
| `AtomicPut` | accepts `io.Reader` | temp-then-rename (local/SFTP); overwrite (S3); streaming write |
| `Rm` | — | Silent on missing (local/SFTP warn + return nil; S3 unconditional delete) |

- **`local.go`** — Auto-creates subdirs via `os.MkdirAll`.
- **`s3.go`** — `IfNoneMatch` precondition headers + CRC32 checksums. `S3Config`: region, endpoint, profile, static credentials.
- **`sftp.go`** — Auto-creates subdirs via `client.MkdirAll`. Auth chain: URL password → config password → config/default private key → `~/.ssh/` keys → SSH agent → error. Host key verification via `~/.ssh/known_hosts` by default (`SFTPConfig.Insecure` to skip).

### Pack Storage (`db.go` + `db_pack.go` + `db_search.go`)

See root `CLAUDE.md` Data Contract for pack format spec (idx/, data/, search/ series), db.gz schema, CDN layout, and file-based locking.

Split across four files by concern:
- **`db.go`** — `DB`, `DBCore`, lifecycle (`NewDB`/`Close`/`Commit`), feed CRUD (`AddFeed`/`RemoveFeed`/`FeedByID`/`Feeds`), `withDB`/`withDBCtx` boilerplate wrappers, and shared utilities (`gunzip`, `readGz`, `jsonEncode`). `Commit` serializes `DBCore` via `AtomicPut`. `BumpGen` owns the bump-implies-reset invariant (Gen++ zeroes `HdrPacks`/`SearchPacks`/`SearchTail`; `srr gen --bump` is a thin caller). Also holds the format atom constants including `searchGram`/`searchBloomBytes`/`searchBloomK` (exported to TS as `SEARCH_GRAM`/`SEARCH_BLOOM_BYTES`/`SEARCH_BLOOM_K` by `cmd_gents.go`).
- **`db_pack.go`** — Binary idx + JSONL data pack writer. Contains `ArticleData` and `Item` types (plus `Item.articleData`, the one Item→ArticleData mapping), the `pack` struct (`newPack`/`writeIdx` — a 3-byte entry `feed_id:u16 LE + packed:u8` — /`writeIdxHeader` — the variable-length header: 3 state u32s + a `numSlots` u32 prefix, then `numSlots` cumulative-count u32s sized to the high-water feed_id present /`writeArticle`), `loadPack` (returns the decompressed bytes so `checkLatestIdx` can verify the latest idx pack's entry count against db.gz), `savePack`/`savePackFinal`/`checkLatestIdx` (`savePackFinal` publishes finalized — immutable, downloaded-forever — names recompressed by `gzipBest`: zopfli-grade deflate via `foobaz/go-zopfli`, still plain gzip to readers, measured −4% data / −11% idx; the candidate must round-trip byte-for-byte through the stdlib gzip reader or the save hard-fails — no silent fallback, a finalized name is cached forever so corrupt bytes must never publish. Latest packs, summaries, and db.gz keep fast stdlib gzip. `finalGzip` is the test seam — `setupTestDB` stubs it to identity so 50k-boundary tests skip the CPU), and the `PutArticles` driver (returns the `[]ArticleData` it persisted, in pack order — `SyncSearch` consumes that slice so derived search entries can't drift from the packs). Also `dataKeyFor` and `parseDataPack` (used by `cmd_art.go` and `cmd_inspect.go` for read-side access), and the idx header-summary machinery: `summaryKey`/`numFinalizedIdx`/`idxKeyAndSize`/`readPackHeader` (fixed-size header reader, used by search's bloom header)/`readIdxHeader` (variable idx header: reads the prefix, then `numSlots`×4 count bytes), `saveSummary` (publishes a count-named concatenation of finalized-pack headers; takes a per-pack header-bytes callback so idx passes its variable-header reader and search passes its fixed bloom reader), `SyncIdxSummary` (rebuilds `idx/h<N>.gz` from the finalized packs' headers whenever `HdrPacks` lags — covers normal finalization, pre-summary store migration, and post-`gen --bump` resets; sets `HdrPacks` only after the save succeeds), and `GCSummaries` (sweeps superseded `h<g>` names, same grace window as `GCLatest`).
- **`db_search.go`** — Search pack writer and folding contract. `SearchEntry` type (`s`/`w`/`t` JSONL wire — the struct tags own the format; every encode and decode goes through it) with its buffer-level decoder `parseSearchEntries` (used by `srr inspect`'s `checkSearch` and the tests). `foldSearchText` (NFD → strip Mn → lowercase → ς→σ → non-letter/number = word break, space-joined; mirrored by `search.ts fold()`). `bloomBits`/`bloomAdd`/`bloomHas` (FNV-1a-64 double-hash, 4 probes). `walkArticles` (streams `[from, to)` chron range from idx+data packs). `SyncSearch(ctx, written)` (called by `cmd_fetch` between `SyncIdxSummary` and `Commit` with the `[]ArticleData` slice `PutArticles` returned, warn-only): reconciles `search/` with the store — reads back `search/L<Seq-1>` (trusted iff entry count == `SearchTail`), fills the missing chron range from `written` when it covers the gap exactly (the common cycle — no pack re-read seconds after the write) else via `walkArticles`, finalizes bloom-headed shards at `idxPackSize` boundaries (`saveSearchShard` decodes titles for the bloom there, once per 50k articles), writes `search/L<Seq>`, rebuilds the bloom summary from streaming `readPackHeader` reads when `SearchPacks` lags `numFinalized`; sets `SearchPacks`/`SearchTail` only after every save succeeds. `finalizedSearchKey`/`searchSummaryKey`. `GCSearchSummaries` (sweeps superseded `s<g>` names, same grace window as `GCSummaries`). `golang.org/x/text` (unicode/norm) is a direct dependency.
- **`idx_read.go`** — the single Go read-side mirror of the binary idx format: `keyGetter` (fetch+gunzip a store key, local or HTTP), `idxPack`/`idxBound`, `parseIdxPack` (byte-for-byte mirror of frontend `idx.ts` `makeIdxPack().parse()`), `getPackRef` (mirrors `data.ts`), `packIdxFor`, `loadIdxPacks`. Every read-side command (`srr inspect`, `srr art ls`) goes through this one parser; the only other implementations of the format are the writer (`db_pack.go`) and the frontend reader.

Backend-specific behavior:
- `PutArticles` and `savePack` manage the idx and data series. `SyncSearch` manages the search/ series. Order: `PutArticles` → `SyncIdxSummary` → `SyncSearch` → `Commit`.
- `PutArticles` writes binary idx and JSONL data directly; splits idx packs at every 50,000 articles; sets `FirstFetchedAt` on first run that produces articles.
- Per-entry `delta_fetched_at` is computed against `prevFetchedTS = FirstFetchedAt/28800 + FetchedAtCursor` so the first entry of each batch records the elapsed time since the previous fetch (clamped to 7 bits with carry).
- `ArticleData` struct: `{ FeedID, FetchedAt, Published, Title, Link, Content }` — serialized as JSONL with short keys `s/a/p/t/l/c`.
- `DBCore.Feeds` is `map[int]*Feed`; serialized as a JSON object keyed by feed ID. `AddFeed` assigns the first free ID in `[0, feedIDCeiling)` (`feedIDCeiling` = 65536, since `feed_id` is a u16 in each idx entry) and returns an error if all slots are taken. `NewDB` range-checks loaded ids and rejects a feed with an empty `URL` (a pre-merge `feeds[]`-only db.gz) with a delete-and-re-fetch message. `RemoveFeed` uses `delete`.
- `seq` names the latest packs (`idx/L<seq>.gz`, `data/L<seq>.gz`, `search/L<seq>.gz`): each article-producing batch writes generation `seq+1` (idx + data series; SyncSearch adds search), bumps `Seq` only after idx+data saves succeed, and `Commit` publishes it — names are write-once, so a generation can never be seen half-written or rewritten. `GCLatest(ctx, keep)` (called from `cmd_fetch` after `Commit`, `latestKeep` = 2, warn-only) deletes expired generations for all three series, sweeping a 4-wide trailing window so crash-leaked packs self-heal.
- `hdrs` names the idx header summary (`idx/h<hdrs>.gz`): `cmd_fetch` calls `SyncIdxSummary` between `PutArticles` and `Commit` (warn-only — a failed ~1KB summary write must not discard a durable batch; readers fall back to eager idx loading and the next run retries), then `GCSummaries` next to `GCLatest`. `srr gen --bump` resets `HdrPacks` to 0 (an in-place rebuild invalidates the copied headers).
- `srch`/`srcht` name the search shards and latest tail: `cmd_fetch` calls `SyncSearch` between `SyncIdxSummary` and `Commit` (warn-only); `GCSearchSummaries` runs next to `GCSummaries`; `GCLatest` also sweeps `search/L<g>` generations. `srr gen --bump` resets both `SearchPacks` and `SearchTail` to 0.
- `withDB(locked, fn)` / `withDBCtx(ctx, locked, fn)`: most cmd `Run()` methods are now a single `withDB` call wrapping the work.

### Module System (`mod/`)

Pipeline per-feed during fetch. Factory pattern: `New()` returns a fresh stateful processor; each built-in registers a `func() Processor` factory that runs once per `New()` so it can capture per-instance state (compiled policies, a shared HTTP client). A built-in processor is `mod.Processor` = `func(context.Context, Params, *RawItem) error` (the ctx is the per-fetch context, threaded through `Module.Process`); most built-ins ignore the ctx, but network-bound ones like `#readability` use it for cancellation/deadline.

**File layout** — `main.go` holds the engine (registry + `Module`/`Process` dispatch), the `RawItem` wire types, and the external/shell-module exec. Support helpers carry a `helper_` filename prefix (`helper_params.go`, `helper_assets.go`, `helper_ssrf.go` — the latter exports `SafeTransport`, the SSRF-guarded `*http.Transport` used by `#readability`) to set them apart from the built-in module files, which are named for the module itself (`sanitize.go`, `minify.go`, `readability.go`, each with a matching `_test.go`).

**Module parameters** — a pipeline token may carry `key=value` options after the name, e.g. `#readability timeout=30s maxbody=16MiB`. A bare key with no `=` (e.g. `verbose`) is shorthand for `verbose=true`, so boolean flags need no value. `Module.Process` splits the token on whitespace; when the first field is a registered built-in it parses the rest into `mod.Params` (see `helper_params.go`) and dispatches internally, otherwise the original string runs as a shell command (so shell commands with spaces/`=` are untouched). Built-ins read options via typed accessors with a default for the absent case — `Params.Duration(key, def)`, `Params.Bytes(key, def)` (sizes accept `B`/`KB`/`MB`/`GB` decimal or `KiB`/`MiB`/`GiB` binary; bare = bytes), `Params.Bool(key, def)` (a bare flag reads as true; explicit values take the `strconv.ParseBool` set) — and call `Params.only(allowed...)` to reject unknown keys. A malformed value or unknown key is a hard error (loud config feedback), distinct from a module's per-article fail-open behaviour; modules that take no options reject any param. Parameters are parsed per-item but the cost is negligible. External (shell) modules don't get parsed params — they receive their args on the command line.

- `#sanitize` — bluemonday, content-only.
- `#minify` — tdewolff/minify, content-only.
- `#readability` — for feeds that syndicate only a teaser/summary: HTTP-GETs the item's `Link` (browser-like UA, default 20s timeout, 8 MiB body cap) and replaces `Content` with the article's main body extracted via `go-shiori/go-readability`. Tunable per pipeline position via `timeout=` (Go duration) and `maxbody=` (byte size), e.g. `#readability timeout=30s maxbody=16MiB`; the timeout is enforced via a per-call `context.WithTimeout` so the shared client can carry different budgets. **Fail-open**: empty/invalid link, non-2xx, fetch/parse error, or empty extraction all leave `Content` untouched and return nil (logged at WARN) — one bad article never fails the fetch (but a bad `timeout=`/`maxbody=` value, or an unknown param, is a hard error). Place it **before** `#sanitize`/`#minify` (e.g. feed pipe `["#readability", "#base"]`) so the fetched HTML gets clamped to the allowed element set. Network-bound, so unlike the other built-ins it honours the fetch `context` (cancellation/deadline).
- External modules: `/bin/sh -c`, stdin/stdout JSON (`RawItem`), stderr passthrough. The author-facing protocol spec + a reference implementation live in `backend/README.md` (External mod protocol); the protocol tests are in `mod/main_test.go`.

#### Asset self-hosting (`assetFetcher.UploadCacheRef` + `RewriteAttrs`)

SRR self-hosts files an ingest strategy leaves in the run's shared cache dir: it stores them under a content-hash key (`assets/<2-hex>/<16-hex><ext>`) and rewrites the in-content reference to that key. This is **not** a pipeline module — it is the automatic end-of-pipeline step inlined in `Feed.fetchURL`.

- Implementation `assetFetcher` (`assets.go`): `UploadCacheRef(ctx, cacheDir, localname)` resolves `localname` inside `cacheDir` (guards: containment via `EvalSymlinks`+`Rel`, reject non-regular/symlink), keys by `contentHashKey` (sha256 of the **original source file bytes**, with the source extension) and `Put`s only if the key isn't already in the store (`Backend.Get` existence check; the stored object — possibly filtered, below — must also be under the `SRR_MAX_MEDIA_SIZE` cap). It does no outbound HTTP. Built once per `fetch` run in `cmd_fetch.go` from `db.Backend`, shared across workers (concurrent-safe). The external-ingest upload step (inlined in `Feed.fetchURL`, see Ingest §) calls it once per `#`-marker against the run's shared cache dir (`fetchRun.cacheDir` = `assetCacheRoot()`).
- **Per-asset pre-upload filter (`SRR_ASSET_FILTER` / `globals.AssetFilter`):** an optional operator-configured command (e.g. `webify -m 720`) run on **every** asset just before upload to process its bytes — image/video transcoding being the motivating case (covers external-ingest and any internal-mod marker uniformly, since both funnel through this one step). It runs **only on a store miss**: the existence check keys on the **source** hash *before* filtering, so an asset already uploaded is never re-processed (the common case for an image reused across articles/feeds). `runFilter` appends the cache file path as the command's final arg and reads processed bytes from its stdout (stderr passes through); **fail-soft** — a non-zero exit or empty output uploads the original unchanged, so a filter hiccup or an unhandled file type never wedges a feed. The store key keeps the **source extension** even when the filter changes the format (the existence check must run pre-filter), so filtered bytes ride the source key/extension; the size cap applies to the (possibly smaller) filtered output. Empty `SRR_ASSET_FILTER` disables it (pure pass-through, the previous behaviour). The matching out-of-repo media converter is `webify` (`~/ws/webify`, on PATH as `~/.local/bin/webify`): auto-detects image→WebP / video→WebM, `webify <input>` writes to stdout.
- `RewriteAttrs(content, fn)` (`mod/helper_assets.go`): the generic HTML attribute walk over `assetAttrs` (img/video src/poster **plus `a href`**, the generic-file set) behind the upload step. The caller's `fn` returns `(newVal, ok, err)` and owns the `#`-marker upload policy — which values are markers, how files are stored, and which failures are fatal (a non-nil `err` aborts the walk and fails the feed) — keeping that policy out of `mod`.
- Idempotency: download dedup is the ingest strategy's (its cache check); upload dedup is SRR's — `UploadCacheRef` skips the `Put` (and the `SRR_ASSET_FILTER` run) when the source-hash key is already in the store (`Backend.Get` existence check), so a re-run re-uploads and re-processes nothing.
- `assets/` is a reserved store prefix in the writer↔reader contract (analogous to `idx/`/`data/`); the frontend (`fmt.ts`) resolves it against the pack base. `S3.Put` stamps `Content-Type` by extension so assets render in-browser; local/SFTP rely on the static server's mime-by-extension.
- `GUID` and `Published` are immutable for all modules (built-in or external; change = error). Enforced in `processItem` after each pipeline step — the captured value before the step is compared to the post-step value, attributing changes to the offending module.
- `RawItem.Raw` is set by the ingest parser (`ingest/rss.go`) to the parsed entry as `mod.RawFeedItem` (`map[string][]mod.RawField`); modules can type-assert it for typed access. External (shell) modules see the same data as JSON via the short keys `@`/`$`/`+` (text/attrs/children).

### Pipe Hierarchy

Two levels store an optional mod pipeline: root (`DBCore.Pipe`) and feed (`Feed.Pipe`, JSON `pipe`). Resolved once per feed at the start of `Feed.Fetch` via `resolvePipe` in `feed.go`:

- An empty feed `Pipe` (nil/absent or empty slice) inherits the root pipe; only a non-empty `Pipe` overrides.
- The `#base` token in a feed override expands inline to the root pipe; can appear multiple times. Non-token entries pass through verbatim.
- CLI: `srr pipe` (root), `srr feed add -p` / `srr feed upd -p` / `srr feed import -p` (feed). `-p ""` clears (reverts to inherit-root). The `-p` flag is **repeatable, one step per flag** and carries `sep:"none"` so kong never comma- or space-splits it — a step keeps its spaces (module parameters) and any commas (e.g. `-p "#readability timeout=30s maxbody=16MiB" -p "#base"`). `srr pipe` (root) is a variadic positional, one arg per step (also `sep:"none"`).
- `NewDB` substitutes `defaultRootPipe()` (`["#sanitize", "#minify"]`) when the loaded `DBCore.Pipe` is nil — fresh DBs and existing DBs predating this feature both pick up the default. Persisted on the next `Commit`.

## Conventions

- **Imports**: stdlib → external → internal, blank-line separated, alphabetical within groups
- **Errors**: Always wrap with `fmt.Errorf("context: %w", err)`. Single sentinel: `ErrStopFeed` (ingest/rss.go). No custom error types.

- File-based DB lock (`.locked`) with `--force` override
