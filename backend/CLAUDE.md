# CLAUDE.md

## Project

**SRR Backend** — Static RSS Reader Backend. Go CLI that fetches RSS/Atom/RDF feeds into gzip-compressed pack series (idx/, data/). Backends: local filesystem, S3, SFTP.

## Commands

The root `Makefile` is the unified entry point for both halves of the monorepo; prefer it over raw `go` commands:

- `make build-be` — build the binary to `dist/srrb`
- `make test-be` — `go test ./...`
- `make vet-be` — `go vet ./...`
- `make format-be` / `make format-check-be` — `gofmt -w .` / gofmt gate (the latter runs inside `make verify-be`)
- `make lint-be` — `golangci-lint run ./...` (opt-in; not yet gate-clean, so not in `verify-be`)
- `make generate` / `make generate-check` — regenerate / freshness-check `frontend/src/js/format.gen.ts` (`srr gen-ts`)
- `make verify-be` — vet + gofmt check + build + test + generate-check
- `make release VERSION=…` — cross-compile all platforms (`CGO_ENABLED=0 -ldflags "-s -w"`)

Direct equivalents when working inside `backend/`: `go build -o srr .`, `go test ./...`, `go test -run TestName .`, `go test -v ./store/`.

## Architecture

- **`main.go`** — CLI via `alecthomas/kong` + YAML config. Source: `$SRR_CONFIG_INLINE` (raw YAML content, takes precedence) → `$SRR_CONFIG` (file path) → `$XDG_CONFIG_HOME/srr/srr.yaml`. `Globals` struct for flags. Command groups: `feed` (alias `ch`): `add`, `upd`, `rm`, `ls`, `show`, `edit`, `apply`, `import`; `art` (alias `a`): `fetch`, `ls`; `recipe`; `syndicate`; `preview` (alias `p`); `config` (alias `c`); `inspect` (alias `i`); `version`; `serve`. The optional top-level `secrets:` YAML section (a flat env-var name→value map) is parsed by `parseSecrets` (`secrets.go`) and pushed via `mod.SetSecrets` into `mod.SubprocessEnv`, which merges it into every external ingest/mod command's environment (secrets win over the ambient env; NOT applied to asset-process/asset-peek). An empty name or a name containing `=` is a hard error; values stay plaintext in `srr.yaml`.
- **`cmd_inspect.go` / `cmd_inspect_check.go` / `cmd_inspect_report.go`** — `InspectCmd` (`srr inspect`): pack consistency checker. Split across three files by responsibility: command entry + fetchers (`cmd_inspect.go`: `openFetcher`/`httpFetcher`/`loadCore`/`loadDataPack`), validation checks invoked by `--validate` (`cmd_inspect_check.go`), single-mode reports for `--chron`/`--filter`/`--list-tags`/`--from-hash` (`cmd_inspect_report.go`). The idx parse + addressing mirror itself lives in `idx_read.go` (shared with `srr art ls`). Modes: `--chron N` (single-entry inspection: feed_id + resolved data packId/offset + the stored fetched_at), `--validate` (full sweep: bounds-vs-data, db-meta cross-check, feedCounts continuity across pack boundaries, unknown feed_ids, latest-pack file presence, idx header-summary coverage/content vs `hdrs`, meta coverage/consistency via `checkMeta` — overclaim, latest tail count vs `mt`, shard sizes, every title's grams probe positive in its shard bloom, summary == concat of shard headers), `--filter <tag|feed_id>` (filter resolution + count, mirrors frontend filter math), `--from-hash '<hash>'` (replays `nav.fromHash` end-to-end), `--list-tags` (mirrors `groupFeedsByTag`). `--url` for HTTP CDN; otherwise uses `--store`. Diagnoses the `(reading 'f')` crash class in frontend `nav.ts` (`showFeed` accesses `data.db.feeds[article.f]`).
- **`cmd_fetch.go`** — `signal.NotifyContext` for graceful shutdown. `errgroup` (`golang.org/x/sync/errgroup`) with `SetLimit(globals.Workers)` and a `sync.Pool` for feed buffers. Articles sorted by published time (ascending) before storage. Order: `PutArticles` → `SyncIdxSummary` → `SyncMeta` → `SyncOutFeeds` → `Commit` → GC sweeps (all sync/GC steps warn-only; each GC sweep runs only when its counter — `Seq`/`HdrPacks`/`MetaPacks` — advanced this run, since most cycles fetch nothing and would just re-delete an already-empty window; a crash-leaked name is still swept by the next advancing run). `--interval` / `SRR_FETCH_INTERVAL` duration flag runs fetch in a loop; the loop body is `FetchCmd.fetchLoop(ctx, client)` (single cycle when interval ≤ 0, else cycle→sleep→repeat until ctx cancel, returning nil on clean shutdown and logging — not propagating — a failed cycle), shared with `srr serve --interval`. Error summary (fetched/failed counts) logged at end. The end-of-pipeline asset-upload step is parallel too: `Feed.fetchURL` defers self-hosting to `fetchRun.uploadAssets`, which fans out marker-bearing items over a per-feed `errgroup`. The **run-global** asset worker pool lives on the shared `assetFetcher` (`a.sem`, sized by `SRR_ASSET_WORKERS`/`--asset-workers`, default nproc, independent of `--workers`) and is held by the singleflight **leader** job only — the goroutine that actually peeks/transcodes/uploads a unique asset — so followers coalescing on the same bytes never pin a slot, and total concurrent peek/transcode/upload jobs across all feeds is capped regardless of how assets distribute. The singleflight body runs under the **run/shutdown ctx** (`a.baseCtx`), not the per-feed errgroup ctx, so one feed's cancellation can't poison a follower feed sharing an asset (only run shutdown aborts it; a follower whose own feed cancels bails via the `DoChan` caller-select). Marker-less items (`mod.HasAssetMarkers` is false) skip the fan-out; `FetchCmd.fetch` sizes the pool (cap ≥ 1).
- **`cmd_feeds.go`** — One feed = one URL. `AddCmd` (`srr feed add`, strict-create; flags `-t/--title`, `-u/--url` (single, required), `-g/--tag`, `-r/--recipe`), `UpdCmd` (`srr feed upd ID`, feed-level fields plus `-u/--url` to repoint the source), `RmCmd` (`srr feed rm`), `LsCmd` (`srr feed ls`, filter by `-g/--tag`, yaml/json output), `ShowCmd` (`srr feed show ID`), `ApplyCmd` (`srr feed apply` reads feed JSON from `--file PATH` or stdin; accepts a single object or an array; whole-input atomic; create when `id` absent, full-replace update when `id` present, error if `id` references a missing feed), `EditCmd` (`srr feed edit ID` opens `$VISUAL`/`$EDITOR`/`vi` on the feed JSON; no-change is a no-op; id changes refused; tempfile preserved on parse or apply error). The per-feed fetch state (`ETag`/`LastModified`/`Watermark`/`BoundaryGUIDs`/`FetchError`) is preserved when the URL is unchanged across `feed upd -u …`, `feed apply` updates, and `feed edit` (`setFeedURL` no-ops on an unchanged URL); changing the URL resets it (a new source shares no dedup/cache history). `feedView` is the flat JSON/YAML shape `{id?, title, url, error?, tag?, recipe?}` — `error` is the read-only last `FetchError`, never applied back. Empty-string convention on `-t`/`-g`/`-r` clears the field where allowed (title and url cannot be cleared). `normalizeFeed` validates the recipe name exists (eager error on unknown recipe). **Subscribe-time discovery (automatic, networked):** `feed add` and `feed upd -u` (only when the URL actually changes) resolve the URL through `ingest.Resolve` (the `resolveFeedURL` package var — a test seam) when the feed's effective ingest strategy is the built-in `#feed` (`resolvesFeed` gates on `ingest.Select` == `ingest.Builtin` using the feed's resolved recipe; external-ingest feeds are stored as-is). A homepage URL is repointed to its discovered feed; an unresolvable URL is a **hard error** that rejects the add/repoint. `feed apply` / `feed edit` deliberately stay offline. So `add`/`upd -u` are no longer purely offline — they make one network probe.
- **`cmd_art.go`** — `ArtCmd` (`srr art ls`): lists stored articles newest-first with cursor pagination (`-b/--before`), filter by feed ID (`-i`) or tag (`-g`), with `-l/--limit` (default 50). Outputs JSON. `readAllIdx` walks the packs parsed by the shared `idx_read.go` reader (`loadIdxPacks` + `getPackRef`), filtering by `db.Feeds()`/`AddIdx`; `loadContent` parses JSONL data packs to fill in the per-article `ArticleData` (title, link, published, content) from the data pack.
- **`cmd_import.go`** — OPML import with hierarchical ID selection (`-a` all, `-i` specific). Each OPML `xmlUrl` becomes one single-URL feed. OPML group hierarchy auto-resolves to hierarchical tags; `-g/--tag` overrides. Cross-folder URL dedup: a `seen` set on `importWalker` emits a URL once even when it is cross-listed in several folders (the first folder visited — case-insensitive name order — wins its tag). `-r/--recipe` stamps every imported feed. `-n/--dry-run` lists resulting feeds without importing. `applyImportDefaults` centralises post-walk Tag/Recipe stamping. (No merge mode — that was removed with the Feed concept.) **Subscribe-time discovery (automatic, networked):** after stamping, `resolveImportFeeds` probes every `#feed` feed concurrently (bounded by `globals.Workers`; external-ingest feeds pass through untouched) via the `resolveFeedURL` seam, repointing homepage URLs to their discovered feed. Import is **partial-success**, not atomic on resolution: resolvable feeds are imported, unresolvable ones are skipped and reported (`reportImportFailures` → the test-overridable `stdout`). `--dry-run` probes too, so its preview shows resolved URLs and the skip report. `importRecipes` reads the db.gz recipes map (read-only) to gate the `#feed` decision per each feed's resolved recipe.
- **`opml.go`** — OPML XML parsing. `ParseOPMLTree` builds `OPMLNode` tree from file; each `OPMLNode.Feed` is a single-URL `Feed`. `normalizeGroupName` converts group names to tag-safe identifiers.
- **`cmd_preview.go`** — `preview` subcommand: loads a recipe (`-r/--recipe`, default `default`) from db.gz, applies optional ad-hoc overrides (`-p` pipe steps, `-i` ingest strategy), dispatches through the ingest engine, runs the module pipeline, and serves rendered articles via local HTTP server (`-a/--addr`). Opens the DB read-only.
- **`cmd_serve.go` / `serve_overview.go` / `serve_feeds.go` / `serve_recipes.go` / `serve_syndicate.go` / `serve_fetch.go` / `serve_tools.go`** — `ServeCmd` (`srr serve`): a localhost web admin GUI. Serves an embedded `webui/` bundle (`//go:embed`) at `/` and a JSON API under `/api/*`. Each handler opens its own `withDB` scope per request — read-only (no lock) for GET, `withDB(true)` (acquire→commit→release) for mutations; no persistent lock or DB handle. A Host/Origin loopback allowlist (`hostGuard`) blocks cross-origin access to the mutating API; binds `localhost:8088` by default (`--addr`/`SRR_SERVE_ADDR`). Optional `--interval`/`SRR_SERVE_INTERVAL` (default 0 = off) runs the same all-feeds fetch loop in-process — a background goroutine driving `FetchCmd.fetchLoop` under the server's signal context, so one Ctrl-C/SIGTERM stops both; a running cycle holds the store lock for its duration, so a concurrent GUI mutation gets the same 409 contract as a separate fetch process. Reuses CLI logic via shared helpers: `saveFeed` (feed upsert + subscribe-time discovery), `setRecipe`/`removeRecipe` (also back `srr recipe`), `setOutFeed`/`removeOutFeed` (also back `srr syndicate`), `renderPreview` (also backs `srr preview`), and `FetchCmd.runFetch(filter, onFeed)` (also backs `srr art fetch`; the SSE `/api/fetch` streams per-feed progress). Lock contention → 409 (`os.ErrExist`); a triggered fetch holds the lock for its duration like the cron loop. `/api/inspect` captures the existing `srr inspect` report. **`GET /api/overview`** (`serve_overview.go`) is the single-read store snapshot every read-tab renders from: feeds projected to the UI list shape (map-key id + friendly `error`, omitting the per-feed dedup/cache internals bg/etag/wm), the derived tag buckets, `db.core.Recipes` directly (no defensive copy — the map is consumed read-only for JSON marshaling), the syndication `out` list, and `gen`/`fetched_at`/`total_art` — one db.gz read per request. (All projection logic is inlined into `getOverview`; the granular `GET /api/feeds`/`/api/tags`/`/api/recipes`/`/api/syndicate`/`/api/gen` read endpoints were removed — the webui only ever called `/api/overview`.)
- **`cmd_recipe.go`** — `RecipeGroup` (`srr recipe`): sub-commands `ls` (print all recipes as JSON/YAML), `show <name>` (print one; errors if missing), `set <name> [-i ingest] [-p step]…` (add or replace — full-replace semantics; each axis falls back to `default` at resolution time, not at write time; `#default` is validated and forbidden in `'default'` itself via `validatePipe(allowDefault = name != defaultRecipeName)`), `rm <name>` (refuses `'default'` and any name still referenced by a feed). Also hosts `filterPipe` (drops empty strings; returns nil for all-empty input as the CLI "clear" sentinel) and `validatePipe(steps, allowDefault bool)` (rejects `#default` when `allowDefault` is false; validates built-in names; unknown `#`-tokens error).
- **`cmd_syndicate.go`** — `SyndicateGroup` (`srr syndicate`): manages the db.gz `out[]` list. Sub-commands: `ls` (print current Out as JSON), `set <name> -f rss|json [-t title] [-g tags] [-i feed-ids] [-l limit]` (upsert by name; validates format, requires ≥1 selector, checks feed ids exist, defaults limit to `outDefaultLimit`=50), `rm <name>` (remove entry + best-effort delete `out/<name>.<ext>` from store). Also hosts `outFileKey`, `outTitle`, `joinURL` helpers used by `db_out.go`.
- **`db_out.go`** — `SyncOutFeeds(ctx)`: writes `out/<name>.rss` (RSS 2.0) or `out/<name>.json` (JSON Feed 1.1) for each `OutFeed` entry. No-op when `Out` is empty or `SRR_CDN_URL` is unset (warn-only skip). For each output: builds feed-id set from Tags∪FeedIDs union; walks a tail window (limit×10, widened to full store if needed) via `walkArticles`; rewrites relative asset refs to absolute CDN URLs via `rewriteAssetURLs` (a dedicated HTML walk over `outAssetAttrs` — `RewriteAttrs` in `mod/` only fires on `#`-prefixed markers, not arbitrary relative refs, so a separate rewriter is used here); serialises via `marshalRSS`/`marshalJSONFeed`; writes via `Backend.Put(…, true)` (overwrite). Never returns a non-nil error (each per-feed failure is logged at WARN). The `IOutFeedWire` TS type is generated by `cmd_gents.go` and appears in `IDBWire.out?`, but **the frontend/service-worker ignores the `out` field** — it is backend-only config and output.
- **`feed.go`** — `Feed` carries one source URL plus its own fetch state: Title, `URL`, ETag `etag`, LastModified `last_modified`, Watermark `wm`, BoundaryGUIDs `bg`, FetchError `ferr`, Tag, Recipe `recipe`, TotalArt, AddIdx (the `Feed` type was folded in — there is no longer a `Feeds []*Feed`). `Feed.Fetch` resolves the recipe and `default` recipe from `run.recipes` via `recipeFor`, resolves effective pipeline via `resolvePipe(default.Pipe, recipe.Pipe)` and effective ingest via `ingest.Select(recipe.Ingest, default.Ingest)`, validates the pipeline, then calls `Feed.fetchURL` for the single URL, setting/clearing `ch.FetchError`. `fetchURL` dispatches I/O+parse through the shared `*ingest.Fetcher` engine, then applies the dedup/watermark/pipeline path on `ingest.Result.Items`. Dedup model: `Watermark` is the max published unix-second ever seen and `BoundaryGUIDs` is the union of (GUIDs at `Watermark` in the most recent fetch) and (dateless GUIDs in the most recent fetch). Repopulated each non-empty fetch from the current response (no carry-over) so its size stays bounded by what the publisher currently exposes, and hard-capped at `maxBoundaryGUIDs` (1024) so one misbehaving feed can't bloat db.gz — already-stored GUIDs (from the prior fetch's `bg`) are retained first, then the smallest-hash new GUIDs fill the remaining slots, so over-cap items are skipped entirely rather than ingested-but-forgotten, which would duplicate them on every subsequent fetch; a `NotModified` result or a non-modified response with zero items preserves prior `Watermark`/`BoundaryGUIDs` so the dedup state survives a transient empty feed. An item is new iff its GUID isn't in the prior fetch's `BoundaryGUIDs` AND (`pub == 0` OR `pub >= Watermark`). Item `pub` is clamped to `fetchedAt` so a publisher CMS bug that ships a far-future date can't poison `Watermark`. Within-fetch dedup uses a per-GUID set checked first. Trade-off: items at `Watermark` or dateless items that disappear from the feed for one fetch and reappear later are re-ingested as duplicates (snapshot semantics over carry-over). Also hosts the `resolvePipe` helper, the `#default` token constant (`pipeDefault`), the `maxBoundaryGUIDs` const, the `fetchRun` struct (run-scoped deps including `recipes map[string]Recipe`), and `uint32Set`. (`feed.go` is gone — its logic moved here.)
- **`cmd_gents.go`** — `GenTSCmd` (`srr gen-ts`, hidden): generates `frontend/src/js/format.gen.ts` — the TS side of the writer↔reader data contract — by reflecting over the live Go declarations (the `tsConsts` table references the db.go format constants; `store.PackSeries` is emitted as the `PACK_SERIES_KINDS` grammar table; `tsTypes` walks the json tags of `ArticleData`/`MetaEntry`/`Feed`/`Recipe`/`OutFeed`/`DBCore`, omitempty → `?`, nil-able without omitempty → `| null`; emits `IArticleWire`/`IMetaWire`/`IFeedWire`/`IRecipeWire`/`IOutFeedWire`/`IDBWire`). `IRecipeWire`/`IOutFeedWire` are listed in `tsTypes` before `IDBWire` so the generator resolves the nested types; the frontend/service-worker ignores both. `//go:generate go run . gen-ts` (run via `make generate`); `--check` compares instead of writing and is wired into `make verify` (`verify-be`), so editing a format atom without regenerating fails loudly. A struct reachable from `tsTypes` but not listed in it is a hard generator error.
- **`cmd_config.go`** — `ConfigCmd` (`srr config`): prints resolved configuration. With no argument, prints all globals plus **every** registered backend section (s3, sftp — sorted, regardless of the active store scheme, so unset/inactive-backend configs are discoverable). A key argument prints a single field: a global name, a whole backend section by scheme (`s3`), or one backend field (`s3.region`) — all resolved against `store.Configs()`, not just the active store's scheme. Each line annotates its `[SRR_…]` env-override var **only when it deviates** from the conventional derivation (`expectedEnv`: `SRR_[<SCHEME>_]<SCREAMING_SNAKE(name)>`); a name the reader can derive from the field name is omitted. Backend env names are conventional by construction (`store.EnvName`), so they are never printed. When the `secrets:` section is set it is appended as a `secrets:` block (and resolvable via `secrets` / `secrets.<NAME>`), always **masked** through the shared `maskSecret` helper (`printSecretEntries`) so values never leak.
- **`processing.go`** — Per-item transformation utilities shared by `Feed.fetchURL` and `PreviewCmd`: `processItem` runs the module pipeline, enforces GUID/Published immutability, then normalises Title (via `bluemonday` strict policy + whitespace collapse) and strips control chars from Link/Content. When a pipeline step sets `i.Drop=true`, `processItem` returns nil immediately and skips both subsequent steps and post-loop normalization. Also hosts `validFeedURL` (used by `cmd_feeds.go` and `opml.go`).

### Ingest (`ingest/`)

An "ingest strategy" is the I/O+parse layer that produces `mod.RawItem`s from a URL. The default zero-config behaviour is the historical RSS/Atom/RDF path (`#feed`); alternative strategies — shipped out-of-repo as external shell commands — cover URLs that aren't feeds. Selection per feed: `ingest.Select(recipe.Ingest, default.Ingest)` — the feed's resolved recipe's ingest, falling back to the `default` recipe's ingest, falling back to `#feed`. Empty strings fall through; persistence uses `omitempty`.

Built-in names start with `#` and resolve through the package's `Register` registry (init-time). Anything not starting with `#` is treated as a shell command per the external-ingest protocol. Only `#feed` ships built-in; source-specific strategies live out-of-repo as external commands.

Mirrors `mod/`'s architecture: a `Fetcher` struct (built by `New()`) holds the registered built-in `FetchFunc`s in a name→func map; `Fetcher.Fetch(ctx, args, ...)` dispatches by name — built-ins for matching keys, shell-command fallthrough for anything else.

- **`main.go`** — `Request` / `Result` structs, `FetchFunc` type (`func(ctx, *http.Client, buf, Request) (Result, error)`), direct registration `Register(name, FetchFunc)` (built-ins are stateless, so `New()` shares the registry rather than instantiating one per call), `Select(feed, global) string` precedence helper (final `#feed` fallback, exported as the `Builtin` const — the token callers compare against to gate subscribe-time discovery), the `Fetcher` struct + `New() *Fetcher`, and `Fetcher.Fetch(ctx, args, client, buf, req)` which routes `args` to a built-in if registered or executes it as a shell command otherwise. External wire schema is the JSON encoding of `Request` (stdin) and `Result` (stdout) — see struct tags; stderr is passthrough. A non-zero exit code or empty stdout is a hard error (fails just that feed's fetch); stdout is capped at 64 MiB. Items on the wire are `mod.RawItem` records, so external strategies emit `guid` as the FNV-32a hash (uint32) of any stable per-item string, matching the dedup contract used by built-ins. `published` is RFC3339 (null/absent for dateless). The author-facing protocol spec + a reference implementation live in `backend/README.md` (Ingest Strategies); the protocol tests are in `ingest/external_test.go`.

**External file self-hosting.** The ingest engine owns no asset machinery: it just sets the external command's working directory to `Request.AssetDir` (`cmd.Dir`) so the command stashes/references files with relative paths. The caller (`main`) owns the rest. `FetchCmd.fetch` creates **one cache dir shared by every feed this run** = `assetCacheRoot()` (`MkdirAll`, kept across runs), stores it on `fetchRun.cacheDir` (creation is **mandatory** — never empty, since a command handed an empty working dir would run in SRR's cwd), and `Feed.fetchURL` hands it to **every** feed as both `Request.AssetDir` and the upload-step `cacheDir` — no strategy gate, so a built-in fetcher may download into it too. Whichever fetcher ran owns the file layout inside the shared dir. The upload itself is the **automatic end-of-pipeline step** `fetchRun.uploadAssets` (run from `Feed.fetchURL` after `processItem`, which stays a pure, store-free transform; it fans marker-bearing items out concurrently, the peek/transcode/upload work bounded by the `assetFetcher`'s run-global `SRR_ASSET_WORKERS` pool (held by the singleflight leader job)): for each item it calls `mod.RewriteAttrs(content, fn)` (the generic walk over `assetAttrs` — img/video `src`/`poster` + `a href`), and `fn` scans for **`#`-markers** — a value starting with `#` whose remainder names an existing regular file in `cacheDir` (e.g. `#/photo.jpg`) — uploading each through `assetFetcher.UploadCacheRef` (content-hash key, upload-if-absent) and rewriting the marker to the returned key. `UploadCacheRef` classifies its guard failures with the `errNotAsset` sentinel — references naming no regular file (an ordinary `#fragment` anchor) or resolving outside the cache dir (attacker-influenced content) are **declined** by `fn` (the marker stays untouched, one containment implementation lives in `UploadCacheRef` alone); any other upload failure (store error) returns the error from `fn`, which `RewriteAttrs` surfaces so `Feed.fetchURL` **hard-fails the whole feed** — feed state (watermark/dedup/etag) is left untouched, so a transient failure self-heals next fetch. (Oversize is no longer one of these: the `--max-asset-size` cap is enforced **at download** by `#selfhost`/ingest, so an over-cap asset is dropped there — keeping its remote URL — and never reaches the upload step.) `<video poster>` is **not** a usable target *for ingest-injected markers*: `#sanitize`'s poster regex (`^(https?://|assets/)`) strips a `#`-poster before the upload step runs (an ingest strategy's markers ride the pipeline from the start, so they pass through `#sanitize`). The built-in `#selfhost` mod, by contrast, runs *after* `#sanitize` (placed after `#default`) and **does** self-host posters — its markers are created post-sanitize and never re-sanitized. Download dedup is the command's (cache check); upload dedup is SRR's (content hash + store existence check). The cache is disposable, no eviction (ops-managed). A `FetchFunc` owns only I/O and parsing — dedup, watermarking, pipeline modules, and storage all stay in `Feed.fetchURL`/`processItem` and operate uniformly on `Result.Items`. Built-in `FetchFunc`s and the shell-command path must be concurrent-safe (one `*Fetcher` is shared across workers; unlike `*mod.Module` it isn't pooled).
- **`feed.go`** — Built-in `#feed` strategy (registered inline in `init`): the zero-config default. HTTP GET with `If-None-Match` / `If-Modified-Since` hints into the shared per-worker buffer, then `ParseFeed`. Returns `NotModified` on 304; populates `ETag` / `LastModified` for the next request. **Feed auto-discovery**: when `ParseFeed` reports `errNotFeed` (the document is not a recognized feed — an HTML page, unknown XML root, or non-XML — *as opposed to* a genuine fault parsing a real feed) and the response looks like HTML (`text/html` Content-Type or body starting with `<!doctype html`/`<html`), `discoverFeedLink` (`feed_discover.go`) scans for `<link rel="alternate" type="application/rss+xml|atom+xml|…">` in the HTML head, fetches the discovered URL, and sets `Result.ResolvedURL`; `feed.go` persists `ch.URL = ResolvedURL` without resetting dedup/etag/vitals. One-hop guard: a discovered page that is also HTML-without-feed returns the original error and does not recurse. **`errNotFeed`** is the unexported sentinel that classifies "not a feed" so the discovery branch (here, fetch-time) never fires on a broken-but-real feed. **`Resolve(ctx, client, rawURL, maxSize)`** is the subscribe-time wrapper: it runs the same `feedFetch` discovery once and returns the canonical feed URL (rawURL if it already serves a feed, the discovered target if it's an advertising HTML page, or an error if neither) — used by `feed add`/`upd -u`/`import` (see `cmd_feeds.go`/`cmd_import.go`). Also houses the streaming RSS/Atom/RDF parser: `ParseFeed(data, fn)` (a non-feed document is reported wrapped in `errNotFeed`; a fault parsing a recognized feed is a plain error) and the package-private `hash(string)` used for GUID stability (the FNV-32a contract external fetchers replicate).
- **`feed_discover.go`** — `discoverFeedLink(htmlData, baseURL)`: scans HTML for the first `<link rel="alternate">` with a recognised feed MIME type using `golang.org/x/net/html.NewTokenizer`; returns the href resolved against baseURL. Case-insensitive rel (including multi-valued), tolerant of malformed/partial HTML.

### Store (`store/`)

Low-level storage interface: `Get`/`Put`/`AtomicPut`/`Rm`/`Close`. Registry selects by URL scheme; local = empty scheme `""`. Config registry: backends call `RegisterConfig` in `init()` with a config struct pointer; `LoadConfigs` reads YAML sections into them. `main.go` also owns the write-once pack-name grammar: the `PackSeries` series→kinds table (the contract's single source — emitted to the frontend as `PACK_SERIES_KINDS` by `srr gen-ts`) builds `packKeyRe`, which `cacheControlForKey` uses to stamp immutable Cache-Control on pack keys (strict per-series: `data/h3.gz` is not a pack name).

| Method | Signature | Behavior |
|---|---|---|
| `Get(ignoreMissing)` | returns `io.ReadCloser` | Controls error-on-missing; streaming read |
| `Put(ignoreExisting)` | accepts `io.Reader` | Controls overwrite vs exclusive create; streaming write |
| `AtomicPut` | accepts `io.Reader` | temp-then-rename (local/SFTP); overwrite (S3); streaming write |
| `Rm` | — | Silent on missing (local/SFTP warn + return nil; S3 unconditional delete) |

- **`local.go`** — Auto-creates subdirs via `os.MkdirAll`.
- **`s3.go`** — `IfNoneMatch` precondition headers + CRC32 checksums. `S3Config`: region, endpoint, profile, static credentials.
- **`sftp.go`** — Auto-creates subdirs via `client.MkdirAll`. Auth chain: URL password → config password → config/default private key → `~/.ssh/` keys → SSH agent → error. Host key verification via `~/.ssh/known_hosts` by default (`SFTPConfig.Insecure` to skip).

### Pack Storage (`db.go` + `db_pack.go` + `db_meta.go`)

See root `CLAUDE.md` Data Contract for pack format spec (idx/, data/, meta/ series), db.gz schema, CDN layout, and file-based locking.

Split across four files by concern:
- **`db.go`** — `DB`, `DBCore`, `Recipe`, `OutFeed`, lifecycle (`NewDB`/`Close`/`Commit`), feed CRUD (`AddFeed`/`RemoveFeed`/`FeedByID`/`Feeds`), `withDB`/`withDBCtx` boilerplate wrappers, and shared utilities (`gunzip`, `readGz`, `jsonEncode`). `Commit` serializes `DBCore` via `AtomicPut`. `BumpGen` owns the bump-implies-reset invariant (Gen++ zeroes `HdrPacks`/`MetaPacks`/`MetaTail`; `srr gen --bump` is a thin caller). `DBCore.Recipes map[string]Recipe` is the named-recipe map (backend-only config; `omitempty`); `DBCore.Out []OutFeed` is the syndication config (default nil → no-op; `omitempty`). `defaultRecipeName = "default"` is the reserved fallback; `recipeFor(recipes, name) Recipe` resolves an empty or unknown name to `default`. `NewDB` seeds the `default` recipe (`["#sanitize","#minify"]`) when absent. Also holds the format atom constants including `searchGram`/`searchBloomBytes`/`searchBloomK` (exported to TS as `SEARCH_GRAM`/`SEARCH_BLOOM_BYTES`/`SEARCH_BLOOM_K` by `cmd_gents.go`) and `metaPackSize` (exported as `META_PACK_SIZE`).
- **`db_pack.go`** — Binary idx + JSONL data pack writer. Contains `ArticleData` and `Item` types (plus `Item.articleData`, the one Item→ArticleData mapping), the `pack` struct (`newPack`/`writeIdx` — a 2-byte entry `feed_id:u16 LE` — /`writeIdxFooter`+`parseIdxFooter` — the data-pack boundary footer, u16 LE local indices where the packId advances; `parseIdxFooter` recovers it on the append path — /`writeIdxHeader` — the variable-length header: 2 state u32s (`packId_base`, `packOff_base`) + a `numSlots` u32 prefix, then `numSlots` cumulative-count u32s sized to the high-water feed_id present /`writeArticle`), `loadPack` (returns the decompressed bytes; `packFromBytes` wraps them into an appendable pack, guarding the empty case so a fresh store keeps `Len()==0`), `savePack`/`savePackFinal`/`checkLatestIdx` (`checkLatestIdx` verifies the latest idx pack's entry count against db.gz and returns the entries-end offset the append path splits the footer at; `savePackFinal` publishes finalized — immutable, downloaded-forever — names recompressed by `gzipBest`: zopfli-grade deflate via `foobaz/go-zopfli`, still plain gzip to readers, measured −4% data / −11% idx; the candidate must round-trip byte-for-byte through the stdlib gzip reader or the save hard-fails — no silent fallback, a finalized name is cached forever so corrupt bytes must never publish. Latest packs, summaries, and db.gz keep fast stdlib gzip. `finalGzip` is the test seam — `setupTestDB` stubs it to identity so 50k-boundary tests skip the CPU), and the `PutArticles` driver (returns the `[]ArticleData` it persisted, in pack order — `SyncMeta` consumes that slice so derived meta cards can't drift from the packs). Also `dataKeyFor` and `parseDataPack` (used by `cmd_art.go` and `cmd_inspect.go` for read-side access), and the idx header-summary machinery: `summaryKey`/`numFinalizedIdx`/`idxKeyAndSize`/`readPackHeader` (fixed-size header reader, used by meta's bloom header)/`readIdxHeader` (variable idx header: reads the prefix, then `numSlots`×4 count bytes), `saveSummary` (publishes a count-named concatenation of finalized-pack headers; takes a per-pack header-bytes callback so idx passes its variable-header reader and meta passes its fixed bloom reader), `SyncIdxSummary` (rebuilds `idx/h<N>.gz` from the finalized packs' headers whenever `HdrPacks` lags — covers normal finalization, pre-summary store migration, and post-`gen --bump` resets; sets `HdrPacks` only after the save succeeds), and `GCSummaries` (sweeps superseded `h<g>` names, same grace window as `GCLatest`).
- **`db_meta.go`** — Meta pack writer and folding contract. `MetaEntry` type (`f`/`w`/`t` JSONL wire — the struct tags own the format; every encode and decode goes through it) with its buffer-level decoder `parseMetaEntries` (used by `srr inspect`'s `checkMeta` and the tests). `foldSearchText` (NFD → strip Mn → lowercase → ς→σ → non-letter/number = word break, space-joined; mirrored by `search.ts fold()`). `bloomBits`/`bloomAdd`/`bloomHas` (FNV-1a-64 double-hash, 7 probes, mask `& (2^15-1)` matching `searchBloomBytes`=4096). `walkArticles` (streams `[from, to)` chron range from idx+data packs). `SyncMeta(ctx, written)` (called by `cmd_fetch` between `SyncIdxSummary` and `Commit` with the `[]ArticleData` slice `PutArticles` returned, warn-only): reconciles `meta/` with the store — reads back `meta/L<Seq-1>` (trusted iff entry count == `MetaTail`), fills the missing chron range from `written` when it covers the gap exactly (the common cycle) else via `walkArticles`, finalizes bloom-headed shards at `metaPackSize`=5000 boundaries (`saveMetaShard`), writes `meta/L<Seq>`, rebuilds the bloom summary from streaming `readPackHeader` reads when `MetaPacks` lags `numFinalized`; sets `MetaPacks`/`MetaTail` only after every save succeeds. The meta/ series feeds BOTH the home list (with a data/ fallback when `metaReady()` is false, so warn-only is safe even though the list depends on meta) and search. `finalizedMetaKey`/`metaSummaryKey`. `GCMetaSummaries` (sweeps superseded `s<g>` names, same grace window as `GCSummaries`). `golang.org/x/text` (unicode/norm) is a direct dependency.
- **`idx_read.go`** — the single Go read-side mirror of the binary idx format: `keyGetter` (fetch+gunzip a store key, local or HTTP), `idxPack`/`idxBound`, `parseIdxPack` (byte-for-byte mirror of frontend `idx.ts` `makeIdxPack().parse()`), `getPackRef` (mirrors `data.ts`), `packIdxFor`, `loadIdxPacks`. Every read-side command (`srr inspect`, `srr art ls`) goes through this one parser; the only other implementations of the format are the writer (`db_pack.go`) and the frontend reader.

Backend-specific behavior:
- `PutArticles` and `savePack` manage the idx and data series. `SyncMeta` manages the meta/ series. Order: `PutArticles` → `SyncIdxSummary` → `SyncMeta` → `Commit`.
- `PutArticles` writes binary idx and JSONL data directly; splits idx packs at every 50,000 articles. It tracks the data-pack boundaries (where `NextPackID` advances) into a per-pack slice and emits them as the idx footer at each finalize and at the latest-pack save; on the append path it strips the loaded latest pack's old footer (keeping header+entries) and recovers its boundaries via `parseIdxFooter` before extending.
- `ArticleData` struct: `{ FeedID, FetchedAt, Published, Title, Link, Content }` — serialized as JSONL with short keys `f/a/p/t/l/c`.
- `DBCore.Feeds` is `map[int]*Feed`; serialized as a JSON object keyed by feed ID. `AddFeed` assigns the first free ID in `[0, feedIDCeiling)` (`feedIDCeiling` = 65536, since `feed_id` is a u16 in each idx entry) and returns an error if all slots are taken. `NewDB` range-checks loaded ids and rejects a feed with an empty `URL` (a pre-merge `feeds[]`-only db.gz) with a delete-and-re-fetch message. `RemoveFeed` uses `delete`.
- `seq` names the latest packs (`idx/L<seq>.gz`, `data/L<seq>.gz`, `meta/L<seq>.gz`): each article-producing batch writes generation `seq+1` (idx + data series; SyncMeta adds meta), bumps `Seq` only after idx+data saves succeed, and `Commit` publishes it — names are write-once, so a generation can never be seen half-written or rewritten. `GCLatest(ctx, keep)` (called from `cmd_fetch` after `Commit`, `latestKeep` = 2, warn-only) deletes expired generations for all three series, sweeping a 4-wide trailing window so crash-leaked packs self-heal.
- `hdrs` names the idx header summary (`idx/h<hdrs>.gz`): `cmd_fetch` calls `SyncIdxSummary` between `PutArticles` and `Commit` (warn-only — a failed ~1KB summary write must not discard a durable batch; readers fall back to eager idx loading and the next run retries), then `GCSummaries` next to `GCLatest`. `srr gen --bump` resets `HdrPacks` to 0 (an in-place rebuild invalidates the copied headers).
- `mp`/`mt` name the meta shards and latest tail (`MetaPacks`/`MetaTail`): `cmd_fetch` calls `SyncMeta` between `SyncIdxSummary` and `Commit` (warn-only); `GCMetaSummaries` runs next to `GCSummaries`; `GCLatest` also sweeps `meta/L<g>` generations. `srr gen --bump` resets both `MetaPacks` and `MetaTail` to 0.
- `withDB(locked, fn)` / `withDBCtx(ctx, locked, fn)`: most cmd `Run()` methods are now a single `withDB` call wrapping the work.

### Module System (`mod/`)

Pipeline per-feed during fetch. Factory pattern: `New()` returns a fresh stateful processor; each built-in registers a `func() Processor` factory that runs once per `New()` so it can capture per-instance state (compiled policies, a shared HTTP client). A built-in processor is `mod.Processor` = `func(context.Context, Params, *RawItem) error` (the ctx is the per-fetch context, threaded through `Module.Process`); most built-ins ignore the ctx, but network-bound ones like `#readability` use it for cancellation/deadline.

**File layout** — `main.go` holds the engine (registry + `Module`/`Process` dispatch), the `RawItem` wire types, and the external/shell-module exec. Support helpers carry a `helper_` filename prefix (`helper_params.go`, `helper_assets.go`, `helper_ssrf.go` — the latter exports `SafeTransport`, the SSRF-guarded `*http.Transport` used by `#readability`) to set them apart from the built-in module files, which are named for the module itself (`sanitize.go`, `minify.go`, `readability.go`, each with a matching `_test.go`). `secrets.go` holds `SetSecrets`/`SubprocessEnv` — the process-env⊕`srr.yaml`-secrets merge (secrets win, deduped, sorted) used as the `env` for both `mod.New()` and `ingest.New()`'s external commands.

**Module parameters** — a pipeline token may carry `key=value` options after the name, e.g. `#readability timeout=30s maxbody=16MiB`. `Module.Process` splits the token on whitespace; when the first field is a registered built-in it parses the rest into `mod.Params` (see `helper_params.go`) and dispatches internally, otherwise the original string runs as a shell command (so shell commands with spaces/`=` are untouched). Built-ins read options via typed accessors with a default for the absent case — `Params.Duration(key, def)`, `Params.Bytes(key, def)` (sizes accept `B`/`KB`/`MB`/`GB` decimal or `KiB`/`MiB`/`GiB` binary; bare = bytes) — and call `Params.only(allowed...)` to reject unknown keys. A malformed value or unknown key is a hard error (loud config feedback), distinct from a module's per-article fail-open behaviour; modules that take no options reject any param. Parameters are parsed per-item but the cost is negligible. External (shell) modules don't get parsed params — they receive their args on the command line.

- `#sanitize` — bluemonday, content-only. Allows `<audio>` (`src`/`controls`/`preload`, mirroring `<video>`) so `#selfhost` can self-host audio.
- `#minify` — tdewolff/minify, content-only.
- `#readability` — for feeds that syndicate only a teaser/summary: HTTP-GETs the item's `Link` (browser-like UA, default 20s timeout, 8 MiB body cap) and replaces `Content` with the article's main body extracted via `go-shiori/go-readability`. Tunable per pipeline position via `timeout=` (Go duration) and `maxbody=` (byte size), e.g. `#readability timeout=30s maxbody=16MiB`; the timeout is enforced via a per-call `context.WithTimeout` so the shared client can carry different budgets. **Fail-open**: empty/invalid link, non-2xx, fetch/parse error, or empty extraction all leave `Content` untouched and return nil (logged at WARN) — one bad article never fails the fetch (but a bad `timeout=`/`maxbody=` value, or an unknown param, is a hard error). Place it **before** `#sanitize`/`#minify` (e.g. recipe pipe `["#readability", "#default"]`) so the fetched HTML gets clamped to the allowed element set. Network-bound, so unlike the other built-ins it honours the fetch `context` (cancellation/deadline).
- `#filter` — content-based item dropping (`filter.go`). Params (all optional; item dropped if ANY condition fires): `drop_title=/regex/[i]`, `keep_title=/regex/[i]`, `drop_content=/regex/[i]`, `keep_content=/regex/[i]`, `min_words=N`. On a match sets `i.Drop=true; return nil`. Unknown params / bad regex / bad int are hard errors. Does NOT mutate GUID/Published/Title/Content/Link.
- `#selfhost` — downloads remote `<img>`/`<video>`/`<audio>` media (src + `<video poster>`, not `<a href>`) into the run's shared cache dir and rewrites each to a `#`-marker, so the end-of-pipeline upload step peeks/converts (`SRR_ASSET_PROCESS`)/uploads it to `assets/`. Network-bound (SSRF-guarded `SafeTransport`, like `#readability`); reads the run cache dir from the fetch context (`mod.WithCacheDir`, set in `feed.go:Feed.Fetch`) and **no-ops when absent** (e.g. `srr preview`). Params: `timeout=` (default 120s), `maxbody=` (default 128MiB, **clamped to `--max-asset-size`** via the `mod.MaxAssetSize` package var so the self-host object cap is enforced here at download — an over-cap body never lands in the cache); bad/unknown param is a hard error. **Fail-open per asset**: a bad URL / non-2xx / oversize / SSRF-blocked dial / write error leaves the remote URL in place (WARN). Download dedup is URL→cache-file; upload dedup is `UploadCacheRef`'s content hash. **Place AFTER `#default`** (`["#default", "#selfhost"]`): it then downloads only sanitizer-approved media and its markers (incl. `<video poster>`) never round-trip through `#sanitize`. Limitations: with `webify` as `SRR_ASSET_PROCESS`, audio is self-hosted but not transcoded (webify is image/video only); `<source>` children stay stripped; cross-run URL cache may serve stale bytes (clear the cache dir).
- External modules: `/bin/sh -c`, stdin/stdout JSON (`RawItem`), stderr passthrough. Emitting `{"drop":true}` on stdout sets `i.Drop=true` (via the existing `json.Unmarshal(out, i)` path — no peek struct needed); subsequent pipeline steps are skipped and the item is not stored. Dropped items retain their GUID in the feed boundary. The author-facing protocol spec + a reference implementation live in `backend/README.md` (External mod protocol); the protocol tests are in `mod/main_test.go`.

#### Asset self-hosting (`assetFetcher.UploadCacheRef` + `RewriteAttrs`)

SRR self-hosts files an ingest strategy or the built-in `#selfhost` mod leaves in the run's shared cache dir: it stores them under a content-hash key (`assets/<2-hex>/<16-hex><ext>`) and rewrites the in-content reference to that key. This is **not** a pipeline module — it is the automatic end-of-pipeline step `fetchRun.uploadAssets`, run from `Feed.fetchURL`.

- Implementation `assetFetcher` (`assets.go`): `UploadCacheRef(ctx, cacheDir, localname)` resolves `localname` inside `cacheDir` (guards: containment via `EvalSymlinks`+`Rel`, reject non-regular/symlink), keys by `contentHashKey` (sha256 of the **original source file bytes**, with the source extension) and `Put`s only if the key isn't already in the store (`Backend.Get` existence check). The `SRR_MAX_ASSET_SIZE` cap is **enforced at download** (`#selfhost` / external ingest via `Request.MaxAssetSize`), so `UploadCacheRef` trusts the cache file and does **not** re-check the source size. It does no outbound HTTP. Built once per `fetch` run in `cmd_fetch.go` from `db.Backend`, shared across workers (concurrent-safe). The external-ingest upload step (`fetchRun.uploadAssets` → `rewriteItemAssets`, run from `Feed.fetchURL`; see Ingest §) calls it once per `#`-marker against the run's shared cache dir (`fetchRun.cacheDir` = `assetCacheRoot()`).
- **Per-asset pre-upload processing (`SRR_ASSET_PROCESS` / `globals.AssetProcess`):** an optional operator-configured command (e.g. `webify -m 720`) run on **every** asset just before upload to process its bytes — image/video transcoding being the motivating case (covers external-ingest and any internal-mod marker uniformly, since both funnel through this one step). It runs **only on a store miss**: the existence check keys on the **source** hash *before* processing, so an asset already uploaded is never re-processed (the common case for an image reused across articles/feeds). `runProcess` substitutes the cache file path for each `{input}` token in the command (per arg, e.g. `conv -i {input} --flags`), or appends it as the final arg when absent. With a `{output}` token the command runs in **file mode**: SRR substitutes a fresh temp path, the command writes its processed bytes there, and prints a `{mimetype,extension,encoding}` JSON to stdout — the mimetype/encoding ride the upload as the stored object's `store.ObjectMeta` (Content-Type/-Encoding), the extension is for the peek-decided key (see asset-peek). Without `{output}`, processed bytes are read from stdout (no declared metadata, so the object gets the `application/octet-stream` default). stderr passes through; **fail-soft** — a non-zero exit, empty output, or (file mode) invalid metadata JSON uploads the original unchanged, so a processing hiccup or an unhandled file type never wedges a feed. The command is bounded by its **own** timeout `--asset-process-timeout` / `SRR_ASSET_PROCESS_TIMEOUT` (default **60m**), separate from the shared `--cmd-timeout` because media transcoding can outlast a feed/ingest command: `runProcess` runs through `mod.RunCommandTimeout(ctx, procTimeout, …)` (a zero `procTimeout` — tests/unset — falls back to `SubprocessTimeout`), while `asset-peek` stays on the shared `--cmd-timeout`. The dedup key always hashes the **source** bytes (the existence check must run pre-process), but its **extension** comes from `asset-peek` when configured (else the source extension); the asset-process **output** keeps a fail-soft size guard in **both** modes (`readProcOutput` for `{output}` mode, an inline `len(out)` check for stdout mode): an over-`SRR_MAX_ASSET_SIZE` output uploads the original instead — that output is generated here and isn't download-bounded, unlike the source. Empty `SRR_ASSET_PROCESS` disables it (pure pass-through, the previous behaviour). Asset jobs run concurrently across a feed's items; the peek/transcode/upload work is bounded globally by `SRR_ASSET_WORKERS` (the `assetFetcher`'s `a.sem` pool, held by the leader job only). A marker referenced by several items (same feed or across feeds — the run's `assetFetcher` is shared) is coalesced by `assetFetcher.flight` (a `singleflight.Group` keyed on the source content hash) via `DoChan`, so one concurrent wave peeks/transcodes/uploads the asset exactly once and the rest receive the leader's key without taking a slot — a waiter whose own feed cancels bails on its caller ctx instead of blocking on the leader. The body runs under the run/shutdown ctx (`a.baseCtx`), not the leader feed's, so the leader's result is shared with the wave (a shared asset that genuinely fails to upload fails every referencing feed, since none can publish content pointing at a missing asset; each self-heals next fetch) WITHOUT one feed's cancellation poisoning a healthy follower feed. The `seen` memo still short-circuits the cheaper sequential/cross-call hit (and a wave's late arrivals) before `flight`, and the store-existence check stays the cross-run backstop. The matching out-of-repo media converter is `webify` (`~/ws/webify`, on PATH as `~/.local/bin/webify`): auto-detects image→WebP / video→WebM, `webify <input>` writes to stdout.
- **Per-asset identification (`SRR_ASSET_PEEK` / `globals.AssetPeek`, the `assetFetcher.peek` field set in `cmd_fetch.go`):** an optional probe run on **every** asset **before** the dedup check (so it runs even for already-uploaded assets — the accepted cost). `runPeek` substitutes/append the cache path like `runProcess` and parses a `{mimetype,extension,supported,encoding}` JSON from stdout. The **extension** becomes the stored object's key extension (`contentHashKey(peek.extension, sha256(source))` — so a transcoded `.gif`→WebP rides a `.webp` key while dedup still keys on the source bytes); the **mimetype/encoding** become the object's `store.ObjectMeta`; **supported=false** hosts the original bytes and **skips** `asset-process` (peek reports the real source type for an unsupported asset). Precedence: in `{output}` mode `asset-process`'s declared mimetype/encoding (the actual result) override peek's; a `process.extension` disagreeing with the peek key only **warns** (the key is already chosen). **Fail-soft**: a peek error / invalid JSON falls back to the source extension and `supported=true`. Empty disables (the object then gets the source extension + `application/octet-stream` default).
- `RewriteAttrs(content, fn)` (`mod/helper_assets.go`): the generic HTML attribute walk over `assetAttrs` (img/video src/poster **plus `a href`**, the generic-file set) behind the upload step. The caller's `fn` returns `(newVal, ok, err)` and owns the `#`-marker upload policy — which values are markers, how files are stored, and which failures are fatal (a non-nil `err` aborts the walk and fails the feed) — keeping that policy out of `mod`.
- Idempotency: download dedup is the ingest strategy's (its cache check); upload dedup is SRR's — `UploadCacheRef` skips the `Put` (and the `SRR_ASSET_PROCESS` run) when the source-hash key is already in the store (`Backend.Get` existence check), so a re-run re-uploads and re-processes nothing.
- `assets/` is a reserved store prefix in the writer↔reader contract (analogous to `idx/`/`data/`); the frontend (`fmt.ts`) resolves it against the pack base. The store's `AtomicPut` carries a `store.ObjectMeta{ContentType, ContentEncoding}`: `S3` stamps it (defaulting Content-Type to `application/octet-stream` when unset — it no longer derives a type by extension or byte-sniffing); local/SFTP ignore it and rely on the static server's mime-by-extension at request time. So a peek-corrected key **extension** is what fixes local/SFTP rendering, while the explicit mimetype hardens S3.
- `GUID` and `Published` are immutable for all modules (built-in or external; change = error). Enforced in `processItem` after each pipeline step — the captured value before the step is compared to the post-step value, attributing changes to the offending module.
- `RawItem.Raw` is set by the ingest parser (`ingest/feed.go`) to the parsed entry as `mod.RawFeedItem` (`map[string][]mod.RawField`); modules can type-assert it for typed access. External (shell) modules see the same data as JSON via the short keys `@`/`$`/`+` (text/attrs/children).
- `RawItem.Drop bool` (`json:"drop,omitempty"`) — set to `true` by a pipeline step (built-in or external) to signal that the item should be discarded. `processItem` short-circuits on the first `Drop=true`: returns nil, skips remaining steps and post-loop normalization. Callers (`feed.go` `fetchURL`, `cmd_preview.go`) check `i.Drop` after `processItem` returns nil and `continue` to skip storing/rendering the item. The GUID is already in the boundary map by the time the second-pass loop runs, so a dropped item stays deduped on the next fetch without any extra handling.

### Recipes

Processing config lives in named `{ingest, pipe}` recipes (`DBCore.Recipes map[string]Recipe`), referenced by feeds via `Feed.Recipe`. The reserved `default` recipe (always present, seeded `["#sanitize","#minify"]` by `NewDB`) is the fallback.

- A feed with empty/absent `Recipe` resolves to `default` via `recipeFor`.
- Each axis falls back to `default` independently: a recipe with only `Ingest` set uses its own ingest and `default`'s pipe; only `Pipe` ⇒ its pipe and `default`'s ingest. Resolved at the start of `Feed.Fetch`: `resolvePipe(default.Pipe, recipe.Pipe)` and `ingest.Select(recipe.Ingest, default.Ingest)`.
- `#default` (`pipeDefault`) in a recipe's pipe expands inline to the `default` recipe's pipe; the `default` recipe itself forbids `#default` (enforced by `validatePipe(allowDefault = name != defaultRecipeName)` in `RecipeSetCmd`).
- CLI: `srr recipe set <name> [-i ingest] [-p step]…` (add/replace); `srr recipe ls/show/rm`; `srr feed add/upd/import -r <name>`. The `-p` flag on `recipe set` is **repeatable, one step per flag** (`sep:"none"`), just like feed `-p` on `preview`.
- Clean break: a pre-recipes db.gz that carries legacy root/feed `pipe`/`ingest` fields drops them on load (Go `omitempty` + absent json tags); every feed reverts to `default`. `NewDB` re-seeds `default` if absent.

## Conventions

- **Imports**: stdlib → external → internal, blank-line separated, alphabetical within groups
- **Errors**: Always wrap with `fmt.Errorf("context: %w", err)`. No custom error types or sentinels.
