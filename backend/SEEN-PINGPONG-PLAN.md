# Ping/Pong seen.gz — relocate `bg` out of db.gz Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Move each feed's `BoundaryGUIDs` (`bg`) out of the hot no-cache `db.gz` and into the `seen.gz` sidecar, made crash-atomic with the article commit via a double-buffered (ping/pong) two-slot scheme, so removing `bg` doesn't weaken dedup.

**Architecture:** `seen.gz` becomes two fixed slots `seen.0.gz` / `seen.1.gz`. A boolean `sf` flag in `db.gz` names the active slot. Each dirty cycle writes the *inactive* slot, flips `sf`, then commits `db.gz` — so the article batch and the pointer to its matching dedup state become durable in one atomic `db.gz` write. On load, a corrupt/missing active slot falls back to the sibling slot (≤1 stale cycle) instead of mass-duplicating. `wm` **stays** in `db.gz` (it is reader-displayed and only 3.9% of the object; it also remains a partial dedup floor if the sidecar is ever lost).

**Tech Stack:** Go (backend), the existing `backend/seen.go` binary sidecar format (bumped v1→v2), `srr gen-ts` → `frontend/src/js/format.gen.ts`.

**Measured payoff (prod db.gz, 71 feeds, 2026-07-13):** db.gz 13,801 B compressed → **~6,150 B (−56%)** with `bg` removed. `bg` is 41% of the uncompressed JSON but 56% of the *compressed* bytes because it is incompressible FNV-32 hashes. The 1024-entry cap never binds (max 50/feed, median 17).

**Scope decision — `bg` only, not `wm`:** `frontend/src/js/picker.ts:510` renders `ch.wm` as the "Latest published" card, so `wm` is reader-consumed. Moving it would break that card and require a frontend change for only 3 extra percentage points. Keep `wm` in `db.gz`. This also means a *total* seen loss degrades to `wm`-flooring (dated items ≤ watermark still suppressed) — only re-promotion + dateless dupes leak — the same benign class as "seen.gz missing today".

**Corruption safety (the "Guarded" choice):** gzip's trailer CRC32 + `parseSeen`'s structural checks already *detect* corruption (a bad slot fails `gunzip`/`parseSeen`). The new safety is *recovery*: on active-slot failure, read the sibling slot (the previous generation the ping/pong keeps alive). No separate app-level checksum is added (YAGNI — gzip already covers integrity). Both-slots-corrupt degrades to an empty pool + WARN, where `wm`-in-db.gz remains the floor.

**Invariants preserved:** `db.gz` is the atomic commit point that names companion-object generations (exactly like `seq`/`hdrs`/`mp`/`mt`). `wm`/`db.gz`-atomicity unchanged. `SyncSeen` write-if-dirty preserved.

- [ ] **Step 3: Rewrite `SyncSeen` for ping/pong (write inactive, flip on success)**

Replace `SyncSeen` (line 432) with:

```go
// SyncSeen persists the pool to the INACTIVE seen slot, then flips
// core.SeenFlag so the caller's Commit publishes the pointer to it — making the
// dedup state (pool + bg) atomic with the article batch. It first pulls every
// live feed's current validators + bg into the pool (snapshotHTTP). Write-if-
// dirty: an idle cycle writes nothing and does NOT flip, so db.gz keeps naming
// the still-valid active slot. Unlike the old post-Commit contract, this runs
// BEFORE Commit and its failure is fatal to the cycle (returned, not warned):
// bg is load-bearing now, so a committed article batch must never outrun the
// slot that dedups its GUIDs. On failure the flag is NOT flipped (nothing to
// point at) and the caller aborts the commit.
func (o *DB) SyncSeen(ctx context.Context) error {
	if o.seen == nil {
		return nil
	}
	o.seen.snapshotHTTP(o.core.Feeds)
	if !o.seen.dirty {
		return nil
	}
	inactive := seenSlotKey(!o.core.SeenFlag)
	var buf bytes.Buffer
	gz := gzip.NewWriter(&buf)
	if _, err := gz.Write(o.seen.marshal()); err != nil {
		return err
	}
	if err := gz.Close(); err != nil {
		return err
	}
	if err := o.AtomicPut(ctx, inactive, &buf, store.ObjectMeta{}); err != nil {
		return err
	}
	o.core.SeenFlag = !o.core.SeenFlag // now names the slot we just wrote
	o.seen.dirty = false
	return nil
}
```

```go
// loadSeen reads the active seen slot named by core.SeenFlag. A missing/corrupt
// active slot falls back to the sibling slot (the previous ping/pong
// generation, at most one cycle stale — a bounded re-fetch/re-dup, never a mass
// duplicate), then to the pre-ping-pong legacy seen.gz (migration bridge), then
// to an empty pool + WARN. gzip's trailer CRC32 + parseSeen's structural checks
// are the corruption detector; the sibling slot is the recovery. Always returns
// a non-nil, clean pool.
func (o *DB) loadSeen(ctx context.Context) *seenPool {
	active := seenSlotKey(o.core.SeenFlag)
	sibling := seenSlotKey(!o.core.SeenFlag)
	for _, attempt := range []struct {
		key      string
		fallback bool
	}{{active, false}, {sibling, true}, {seenLegacyKey, true}} {
		p, ok := o.tryLoadSeen(ctx, attempt.key)
		if ok {
			if attempt.fallback {
				slog.Warn("active seen slot unreadable; recovered from fallback", "active", active, "used", attempt.key)
			}
			return p
		}
	}
	slog.Warn("no readable seen slot; using empty pool (bg-only dedup floored by wm)")
	return newSeenPool()
}

// tryLoadSeen reads and parses one seen key. ok=false on missing/read/gzip/parse
// error so the caller can fall through to the next slot. A genuinely-absent
// object (rc == nil) returns (empty, false) so load falls through rather than
// treating "absent" as "successfully empty" (which would mask the sibling).
func (o *DB) tryLoadSeen(ctx context.Context, key string) (*seenPool, bool) {
	rc, err := o.Get(ctx, key, true)
	if err != nil || rc == nil {
		return nil, false
	}
	data, err := gunzip(rc)
	rc.Close()
	if err != nil {
		return nil, false
	}
	p, err := parseSeen(data)
	if err != nil {
		return nil, false
	}
	return p, true
}
```

Note: on a truly fresh store all three attempts miss and we log the "no readable seen slot" WARN once. To avoid a noisy WARN on first-ever run, guard it — only warn when `o.core.Seq > 0` (a store that has committed at least one batch should have a slot). Add that condition to the final WARN.
